Section 9 — Decision Properties of Regular Sets

Decision Properties answer YES/NO questions about Regular Languages. There exist algorithms to decide these properties for Regular Languages.

1. Membership Problem

Is a string w ∈ L?
Given a Regular Language L over Σ and a string w ∈ Σ*, is there an algorithm to determine whether or not w is in L?
Proof: Let L be accepted by a DFA M. For input string w, start from initial state q₀ of M and follow path by accepting one character at a time. If we reach a final state after accepting last character of w → w is accepted (YES). Otherwise → rejected (NO).

2. Emptiness, Finiteness and Infiniteness

Is L empty? Is L finite? Is L infinite?
There exists an algorithm for determining whether a Regular Language is empty, finite, or infinite.
Proof: Let M be a DFA accepting L.
• Check if there is a direct or indirect path from initial state to any final state in the transition graph.
• If NO path exists → L is empty.
• If path exists but no cycle in the path → L is finite.
• If there is a cycle on any path from initial to final → L is infinite.

3. Equivalence of Two Regular Languages

Is L₁ = L₂?
For any two Regular Languages L₁ and L₂, there exists an algorithm to determine whether L₁ = L₂.
Proof: Let L = (L₁ ∩ L̄₂) ∪ (L̄₁ ∩ L₂)  [symmetric difference]
Clearly L is a Regular Language by closure properties.
There exists a DFA which accepts L.
Now, as per the Second Theorem (emptiness): we can determine whether L is empty or not.
L is empty iff L₁ = L₂.

4. Key Decision Results for Regular Languages

  • Membership: Decidable — run string on FSA in O(n) time
  • Emptiness: Decidable — reachability in transition graph
  • Finiteness: Decidable — check for cycles on accepting paths
  • Equivalence: Decidable — check if symmetric difference is empty
  • L∩L': Decidable — product construction, check emptiness
★ Important: All these decision problems are decidable (solvable by an algorithm) for Regular Languages. This is NOT the case for some higher-level languages (Context-Sensitive, Recursively Enumerable).

5. Decision Algorithm for Membership

Algorithm for determining whether a string w is in RL L:

Input: FSA M accepting L, string w
Step 1: Start from initial state q₀ of M
Step 2: Follow transitions, accepting one character of w at a time
Step 3: After reading last character:
   • If current state is a FINAL state → w is ACCEPTED (YES)
   • Otherwise → w is REJECTED (NO)

6. Test Whether Language is Regular or Not

Key Rules:
★ If a language is finite → it is Regular (always)
★ If a language is infinite and we can design a DFA or RE for it → it is Regular
★ If Pumping Lemma FAILS → it is NOT Regular

7. Proof: L is empty iff L₁ = L₂

L₁ = L₂ means: every string in L₁ is also in L₂ and vice versa.

Consider symmetric difference: L = (L₁ ∩ L̄₂) ∪ (L̄₁ ∩ L₂)
• L₁ ∩ L̄₂ = strings in L₁ but NOT in L₂
• L̄₁ ∩ L₂ = strings in L₂ but NOT in L₁

L is empty ↔ no strings exist in only one of them ↔ L₁ = L₂   ✓

Section 10 — Regular vs Non-Regular: Quick Reference

Complete Regularity Test Table

Language Regular? Reason / Method
L = { aⁿ | n ≥ 1 } NOT RL Pumping lemma — can't track n
L = { aⁿbᵐ | n, m ≥ 1 } RL RE = a⁺b⁺ — no balance needed
L = { aⁿbⁿ | n < 10¹⁰ } RL Finite language → always RL
L = { aⁿbⁿ | n ≥ 1 } NOT RL Classic pumping lemma proof
L = { wwᴿ | |w| = 2, Σ={a,b} } RL Finite: {abba, aabbaa, …} → RL
L = { wwᴿ | w ∈ {a,b}* } NOT RL Pumping lemma — palindromes
L = { aⁿbᵐcᵏ | n, m, k ≥ 1 } RL RE = a⁺b⁺c⁺ — independent counts
L = { aⁿbⁿcⁿ | n ≥ 1 } NOT RL Not even CFL — pumping fails
L = { aⁱb²ʲ | i, j ≥ 1 } RL RE = a⁺(bb)⁺ — ratio fixed, DFA possible
L = { aⁿ | n is even } RL RE = (aa)* — DFA with 2 states
L = { aⁿ | n is odd } RL RE = a(aa)* — DFA with 2 states
L = { aⁿ | n is prime } NOT RL Pumping lemma: p(1+k) not prime
L = { aⁿ² | n ≥ 1 } NOT RL Pumping lemma: n²+k not a perfect square

★ Decision Rules (Memorise These!)

1. Finite language → ALWAYS Regular (build DFA with finite states)
2. Infinite language with independent counts (like aⁿbᵐ) → Regular
3. Infinite language with equal/balanced counts (like aⁿbⁿ) → NOT Regular
4. Mathematical property of n (prime, square, etc.) in exponent → NOT Regular
5. Palindromes wwᴿ (unbounded) → NOT Regular
6. Ratio fixed (like aⁿb²ⁿ... wait, this has equal n — NOT RL; but aⁱb²ʲ where i,j independent → RL)

Section 11 — Limitations of Finite State Automata (FSA)

To understand the power of FSA, we must also understand their limitations. FSA are powerful but cannot recognise all languages.

Four Major Limitations of FSA

Limitation 1 — Cannot count arbitrarily
FSA cannot recognise languages that require counting beyond a fixed bound. It cannot recognise certain sets like:
  L = { aⁿbⁿ | n ≥ 1 } — requires remembering count of a's
  L = { wwᴿ | w ∈ {a,b}* } — requires unbounded memory
  L = { aⁿ² | n ≥ 1 } — requires computing perfect squares
  L = { aⁿbⁿcⁿ | n ≥ 1 } — requires matching three counts
Limitation 2 — Can only count finite input
FSA can only process finite input strings. It has a finite number of states and therefore only finite memory. It cannot handle unbounded counting or nesting structures.
Limitation 3 — Cannot recognise balanced structures (parentheses)
There is NO FSA that recognises the set of strings over '(' and ')' that have balanced parentheses.
Example: ((())), ()(), ((())) — require stack-like memory (need PDA/CFL, not FSA).
This is because matching opening and closing parentheses requires remembering the count, which is unbounded.
Limitation 4 — Head movement is one-directional
The head movement (reading) in an FSA is restricted to one direction only — either left→right or right→left. It cannot move back and forth on the input tape. (Turing Machines can move in both directions.)

Summary — What FSA CAN and CANNOT do

FSA CAN recognise FSA CANNOT recognise
✓ Any finite language
✓ Languages with independent counts (aⁿbᵐ)
✓ Languages with fixed ratios (aa)*, (aaa)*
✓ Union, intersection, concat of RLs
✓ Periodic patterns (even/odd length) 		✗ aⁿbⁿ (balanced strings)
✗ wwᴿ (palindromes, unbounded)
✗ Balanced parentheses
✗ Languages requiring stack memory
✗ Languages with mathematical properties (prime, square)
✗ aⁿbⁿcⁿ (multiple balanced counts)
★ Key Insight: FSA has no memory beyond its current state. Once it moves to the next character, it "forgets" the past — except what is encoded in the state. This finite memory is both its strength (simplicity, efficiency) and its fundamental limitation.
★ Remember the Hierarchy:
Regular Languages (FSA/RE)  ⊂  Context-Free Languages (PDA/CFG)  ⊂  Context-Sensitive  ⊂  Recursively Enumerable (Turing Machine)
Each higher level can recognise more languages than the one below it.
