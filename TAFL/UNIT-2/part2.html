Section 5 — Arden's Theorem (FSA → RE)

1. What is Arden's Theorem?

Arden's theorem is used to find the Regular Expression recognised by a Transition System (conversion from FSA → RE).

It solves equations of the form:  R = Q + RP
where R, P, Q are regular expressions.
R = Q + RP   has unique solution   R = QP*

2. Proof

Given:   R = Q + RP     — (1)
Substitute R = QP* in both sides:
QP* = Q + QP*·P
QP* = Q + Q(P*·P)
QP* = Q(ε + P*·P)
QP* = QP*     ← by Identity I₉: ε + P*P = P*
∴ R = QP* is the solution to equation (1). ✓

3. Assumptions for Arden's Method

  • Transition graph does NOT have ε-moves
  • It has only one initial state
  • To get strings recognised: take union of all equations for final states

4. How to Apply — Step-by-Step

  • Write one equation per state of the FSA
  • Each equation: qᵢ = Σ (qⱼ · symbol) for all transitions into qᵢ, plus ε if qᵢ is initial
  • Solve equations by substitution (use Arden's rule: R = Q + RP → R = QP*)
  • The final answer RE = equation(s) of the final state(s)

5. Solved Examples

Q1 — Find RE for FSA with states q₁, q₂, q₃ over Σ = {a, b}
Write equations:
q₁ = ε + q₁·a + q₂·b   —(i)   [q₁ is initial; self-loop a; q₂→q₁ on b]
q₂ = q₁·a + q₂·b + q₃·a  —(ii)
q₃ = q₂·a                 —(iii)
Solve:
From (ii): q₂ = q₁a + q₂b + q₂a·a = q₁a + q₂(b+aa)
Apply Arden: q₂ = q₁·a · (b+aa)* ← R=Q+RP → R=QP*, Q=q₁a, P=(b+aa)
From (i): q₁ = ε + q₁·a + q₁·a(b+aa)*·b
= ε + q₁[a + a(b+aa)*b] = ε + q₁·a[ε+(b+aa)*b]
Apply Arden: q₁ = (a + a(b+aa)*b)* = (a(ε+(b+aa)*b))*
q₂ = (a(ε+(b+aa)*b))* · a(b+aa)*
q₃ (final) = q₂·a = [(a+a(b+aa)*b)* · a(b+aa)*] · a
RE = (a+a(b+aa)*b)* · a(b+aa)* · a

q₁ = ε + q₁·0                       —(i)
q₂ = q₁·1 + q₂·1                 —(ii)

Q3 — Find RE for: q₁(init+final) ↔ q₂, transitions: ab and ba
q₁ = ε + q₂·b + q₃·a
q₂ = q₁·a
q₃ = q₁·b
→ q₁ = ε + q₁·ab + q₁·ba
→ q₁ = ε + q₁(ab+ba)    [Q=ε, P=ab+ba]
→ q₁ = (ab+ba)*          ← Arden: R=QP*, Q=ε
RE = (ab + ba)*
★ Quick Reference — Arden Substitution Pattern:
  • Equation form: R = Q + RP  →  Solution: R = QP*
  • If Q = ε:  R = ε + RP → R = P*
  • Multiple transitions out: combine using union (+) before applying Arden
  • Always solve non-final states first, substitute into final state equations

Section 6 — Closure Properties of Regular Languages

Regular Languages are closed under an operation if applying that operation to RL(s) always gives a Regular Language. The Regular Languages are closed under the following 8 operations:

1. Union

Closed under Union ✓
If L₁ and L₂ are RLs over Σ, then L₁ ∪ L₂ is also RL.
Construction: Create a new start state → ε to both FSA start states; accept if either machine accepts.
Union construction: new start → ε → L₁ FSA and L₂ FSA

2. Concatenation

Closed under Concatenation ✓
If L₁ and L₂ are RLs, then L₁·L₂ (concatenation) is also RL.
Construction: Connect final states of L₁ to start of L₂ via ε-transitions. Remove final status from L₁ states.

3. Complement

Closed under Complement ✓
If L is RL over Σ, then L̄ (complement) is also RL.
Construction: In the complete DFA for L, swap final ↔ non-final states.

L = {strings containing 'a'} → L̄ = {strings NOT containing 'a'}

4. Reversal

Closed under Reversal ✓
If L is RL, then L^R (reversal of all strings) is also RL.
Construction: Reverse all edge directions in the FSA. Swap initial ↔ final states.

If L = {ab, abab, a---b} then L^R = {ba, baba, b---a}

Reversal: swap initial↔final, reverse all arrows

5. Difference

Closed under Difference ✓
If L and M are RLs, then L − M is also RL.
L − M = L ∩ M̄   (intersection of L with complement of M)

6. Kleene's Closure

Closed under Kleene Closure ✓
If L is RL, then L* (zero or more concatenations of L) is also RL.
L = {a} → L* = {ε, a, aa, aaa, …}
Construction: Add ε-transition from all final states back to initial state. Add new start (also final).

7. Intersection

Closed under Intersection ✓
If L₁ and L₂ are RLs, then L₁ ∩ L₂ is also RL.
Proof: L₁ ∩ L₂ = L̄₁ ∪ L̄₂  (De Morgan's law applied to RL closures)
Also proved by product construction of two DFAs — final states = pairs where BOTH components are final.

8. Homomorphism

Closed under Homomorphism ✓
A string homomorphism h is a function on strings that works by substituting a particular string for each symbol.
Formal definition: h(L) = { h(w) | w ∈ L }
where h : Σ → Γ* is called a homomorphism.
Example: Σ = {0,1}, Γ = {a,b} — h(0)='aa', h(1)='bb'

If L = {00, 101} then h(L) = { h(0)h(0), h(1)h(0)h(1) } = { aaaa, bbaabb }

9. Inverse Homomorphism

Closed under Inverse Homomorphism ✓
If L is RL over Γ and h : Σ → Γ* is a homomorphism, then h⁻¹(L) is also RL over Σ.
h⁻¹(L) = { w ∈ Σ* | h(w) ∈ L }

Example: h(0)=a, h(1)=b, h(2)=ab  |  L = {abab}
h⁻¹(L) = {0101, 22, 012, 201}   — strings that map INTO L via h

Check: h(0101) = abab ✓, h(22) = abab ✓, h(012) = aabb ✗ — wait: h(012) = h(0)h(1)h(2) = a·b·ab = abab ✓

Section 7 — Pigeonhole Principle & Pumping Lemma

1. Pigeonhole Principle

Theorem: If n pigeons are assigned to m pigeonholes and m < n, then at least one pigeonhole contains two or more pigeons.

★ In FSA context: if a string of length ≥ n is processed by a DFA with n states, at least one state must be revisited (repeated). This creates a loop — the basis of the Pumping Lemma.

2. Pumping Lemma for Regular Languages

Statement: All Regular Languages have a special property called the pumping property. Every string in the language can be "pumped" (a section repeated) if it is at least as long as a certain special value called the pumping length, with the result string still remaining in the language.
★ Formal Statement

Let L be a regular language. Then there exists a constant n such that for any string w ∈ L with |w| ≥ n, we can write w = xyz such that:

Condition 1 |xy| ≤ n    (x and y together are not too long)
Condition 2 |y| ≥ 1     (y is not empty — it has at least 1 symbol)
Condition 3 xy^i z ∈ L    for all i ≥ 0   (pumping y any number of times keeps string in L)

★ Note: n is no greater than the number of states in the smallest DFA accepting L.

4. Proof of Pumping Lemma

Suppose w is any string in L represented as:  w = a₁a₂a₃...aₘ  where m ≥ n

The string w requires (m+1) states to process:  q₀, q₁, ..., qₘ

Since m ≥ n states and DFA has only n states → by Pigeonhole, at least one state repeats.

The repeated state creates a loop — the part of the string that goes around the loop is y.

Since the loop can be taken 0, 1, 2, ... times → xyiz ∈ L for all i ≥ 0  ✓  Hence Proved

5. 4-Step Method to DISPROVE Regularity

Step 1: Assume L is a Regular Language.
Step 2: If L is RL, let n be the number of states in minimised FSA accepting L.
Step 3: Consider a string w ∈ L such that |w| ≥ n. By Pumping Lemma, w = xyz with |xy| ≤ n, |y| ≥ 1, and xyiz ∈ L for all i ≥ 0.
Step 4: Show that for some i, xyiz ∉ L → Contradiction! Hence L is NOT Regular.
★ Exam Tip: Always choose w carefully — pick the string that will give a clear contradiction when y is pumped. Usually pick w = anbn or similar balanced structure. The contradiction is always about violating the balance condition.

Section 8 — Applications of Pumping Lemma

Template for ALL proofs: Step 1: Assume RL → Step 2: Let n = states → Step 3: Choose w ∈ L, |w|≥n, apply lemma → Step 4: Contradiction → Hence NOT RL

Q1 — Show L = { aⁿbⁿ | n ≥ 1 } is NOT a Regular Language

Proof by Pumping Lemma
1
Suppose L is a Regular Language.
2
If L is RL, let n be the number of states in the minimised FSA accepting L.
3
Consider string w = aⁿbⁿ ∈ L, and |w| = 2n ≥ n.  ✓
By Pumping Lemma, we can write w = xyz such that |xy| ≤ n, |y| ≥ 1, xyiz ∈ L for all i ≥ 0.

Since |xy| ≤ n and w starts with n a's, the substring xy lies entirely within the a-part.
So:   x = aˢ   y = aᵏ   z = aⁿ⁻ˢ⁻ᵏ bⁿ   where k ≥ 1

Take i = 2:  xy²z = aˢ · a²ᵏ · aⁿ⁻ˢ⁻ᵏ bⁿ = aⁿ⁺ᵏ bⁿ
4
xy²z = aⁿ⁺ᵏ bⁿ where k ≥ 1 → number of a's ≠ number of b's → xy²z ∉ L
This contradicts Pumping Lemma (which says xyiz ∈ L for all i ≥ 0).
∴ L = { aⁿbⁿ | n ≥ 1 } is NOT a Regular Language. ✓

Q2 — Show L = { wwᴿ | w ∈ {a,b}* } is NOT a RL

Proof by Pumping Lemma
1
Suppose L is a RL. If RL, let n = no. of states in minimised FSA.
3
Choose w = aᵐbᵐ · bᵐaᵐ where m = 2, so w = aabbbbaa for simplicity.
Actually: take w = aⁿbⁿbⁿaⁿ ∈ L (which is (aⁿbⁿ) concatenated with its reverse).
|w| = 4n ≥ n ✓

By Pumping Lemma: w = xyz, |xy| ≤ n, |y| ≥ 1.
Since |xy| ≤ n and w starts with n a's: y = aᵏ for some k ≥ 1

For i = 2:  xy²z = aⁿ⁺ᵏ bⁿ bⁿ aⁿ
4
aⁿ⁺ᵏbⁿbⁿaⁿ — first half has more a's than second half → NOT of form wwᴿ → ∉ L
∴ L = { wwᴿ | w ∈ {a,b}* } is NOT a Regular Language. ✓

Q3 — Show L = { aᵖ | p is a prime number } is NOT a RL

Proof by Pumping Lemma
1
Suppose L is a RL. Let n = no. of states in minimised FSA.
2
Consider string w = aᵖ where p is a prime, p ≥ n. So |w| = p ≥ n ✓
3
By Pumping Lemma: w = xyz with |xy| ≤ n, |y| ≥ 1.
Let |y| = k, where 1 ≤ k ≤ n
Then |xyz| = p, and |xz| = p − k

Pump y (p+1) times (i = p+1 — a tricky choice!):
|xy^(p+1)z| = |xz| + (p+1)|y| = (p−k) + (p+1)k = p + pk = p(1 + k)

Since p(1+k) = p × (1+k) → this is a product of two numbers both > 1 (since p ≥ 2, k ≥ 1 → 1+k ≥ 2)
4
p(1+k) is NOT prime (multiplication of two numbers > 1 can never be prime) → xy^(p+1)z ∉ L
∴ L = { aᵖ | p is prime } is NOT a Regular Language. ✓

Q4 — Show L = { w ∈ {a,b}* | nₐ(w) < n_b(w) } is NOT a RL

L = strings where count of a's is strictly less than count of b's

Proof by Pumping Lemma
1
Suppose L is RL. Let n = no. of states in minimised FSA.
3
Choose w = aᵐ b^(m+1) where m ≥ n. So nₐ(w)=m < n_b(w)=m+1 ✓ → w ∈ L
By PL: w = xyz, |xy| ≤ n, |y| ≥ 1.
Since |xy| ≤ n and w starts with m a's: y = aᵏ for some k ≥ 1

For i = 2:  xy²z = a^(m+k) b^(m+1)
Now: nₐ = m+k  and  n_b = m+1
Is m+k < m+1? → Only if k < 1. But k ≥ 1, so m+k ≥ m+1 → nₐ ≥ n_b
4
xy²z = a^(m+k) b^(m+1) where k≥1 → nₐ(xy²z) ≥ n_b(xy²z) → xy²z ∉ L
∴ L is NOT a Regular Language. ✓

Q5 — Show L = { 0ⁿ | n is a perfect square } is NOT a RL

Proof by Pumping Lemma
1
Suppose L is RL. Let n = no. of states in minimised FSA.
3
Choose w = 0^(n²) ∈ L (n² is a perfect square)    |w| = n² ≥ n ✓
By PL: w = xyz, |xy| ≤ n, |y| ≥ 1. Let |y| = k, 1 ≤ k ≤ n.

For i = 2:  |xy²z| = n² + k

Need to check: is n² + k a perfect square?
We know:  n² < n² + k ≤ n² + n < n² + 2n + 1 = (n+1)²
So:  n² < n²+k < (n+1)²
4
n² + k lies strictly between two consecutive perfect squares n² and (n+1)² → n²+k cannot be a perfect square → xy²z ∉ L
∴ L = { 0ⁿ | n is a perfect square } is NOT a Regular Language. ✓
