§5 Ambiguity in CFG

1. Definition of Ambiguous Grammar

A CFG denoted by G is said to be ambiguous if there exists some string w ∈ L(G) that has at least two distinct LMDs (or equivalently, two distinct parse trees).

L(G) = language generated by Grammar G.
In simple terms: A grammar is ambiguous if the same string can be derived in MORE THAN ONE WAY (different parse trees). This means the string has multiple structural interpretations.

2. Classic Ambiguity Example

Grammar: S → SbS | a    String w = ababа — show grammar is ambiguous
LMD Path 1:
S ⇒ SbS ⇒ abS ⇒ abSbS ⇒ ababS ⇒ ababa
LMD Path 2:
S ⇒ SbS ⇒ SbSbS ⇒ abSbS ⇒ ababS ⇒ ababa
Two distinct LMDs for same string w = ababa → Grammar is ambiguous ✓

3. Ambiguity in Arithmetic Expressions

Grammar: E→E+E | E*E | id    w = id+id*id
This grammar is ambiguous because it doesn't define:
Parse tree 1: (id+id)*id → E = (E+E)*E → (+) has higher precedence
Parse tree 2: id+(id*id) → E = E+(E*E) → (*) has higher precedence

Reasons for ambiguity:
① Associativity of + is not properly defined
② Precedence of operators is not properly defined

4. Unambiguous Grammar for Arithmetic

To resolve ambiguity in E→E+E|E*E|id, use operator precedence and associativity:

Unambiguous Grammar:
E → E + T | T
T → T * F | F
F → id

Here: * has higher precedence than + (handled by grammar structure), and both + and * are left-associative (left recursion in E and T).
Derive w = id+id*id using unambiguous grammar
E ⇒ E+T ⇒ T+T ⇒ F+T ⇒ id+T ⇒ id+T*F ⇒ id+F*F ⇒ id+id*F ⇒ id+id*id ✓
Only ONE parse tree possible → Grammar is unambiguous ✓

5. Inherently Ambiguous Languages

A CFL L is said to be Inherently Ambiguous if ALL grammars generating L are ambiguous (no unambiguous grammar exists for it).

Examples of Inherently Ambiguous CFLs:
L₁ = { aⁱbʲcᵏ | i,j,k ≥ 1, i=j or j=k }
L₂ = { aⁿbᵐcᵖdᵍ | n,m,p,q ≥ 1, n=p or m=q }
★ A context-free language can be generated by many grammars. A language is unambiguous if there is at least one unambiguous grammar for it. If ALL grammars are ambiguous → Inherently Ambiguous Language.

Q2 — G: bExp→bExp OR bExp | bExp AND bExp | NOT bExp | True | False. Show ambiguous for w = T AND F OR T
Two parse trees possible: ((T AND F) OR T) and (T AND (F OR T))
Path 1: bExp → bExp OR bExp → (bExp AND bExp) OR T → (T AND F) OR T
  → Interprets as ((T AND F) OR T)

Path 2: bExp → bExp AND bExp → T AND (bExp OR bExp) → T AND (F OR T)
  → Interprets as (T AND (F OR T))

Two parse trees → grammar is ambiguous ✓
Unambiguous version: bExp→bExp OR A|A; A→A AND F|F; F→NOT B|B; B→True|False

§6 Regular Grammars (Right & Left Linear)

1. Right Linear Grammar

A grammar G is Right Linear if every production is of the form:

A → xB
A → x
where x ∈ Σ* (string of terminals) and B ∈ V_N (single non-terminal at RIGHT end).

2. Left Linear Grammar

A grammar G is Left Linear if every production is of the form:

A → Bx
A → x
where x ∈ Σ* and B ∈ V_N (single non-terminal at LEFT end).

3. Regular Grammar = Right Linear XOR Left Linear

A grammar is Regular if it is EITHER Right Linear OR Left Linear — but not both in the same grammar.

Example (NOT Regular): A → Ba | a  and  A → aB | a in SAME grammar
→ Mixes left and right linear → NOT regular (but IS linear grammar).

Example (Regular): A → Ba | a (ALL left linear) → Regular ✓

4. Connection to FSA

  • Right Linear Grammar ↔ NFA/DFA (state = current NT, terminal = input symbol)
  • The non-terminal represents the current state of the FSA
  • A → aB means: read 'a', move to state B
  • A → a means: read 'a', move to final state (accepting)
  • Regular Languages are exactly the Type-3 languages

5. Writing Regular Grammars

Write Right Linear Grammar for L = { aⁿbᵐ | n,m ≥ 1 }
Strategy: Generate a's with one NT, then switch to generating b's with another NT.
S → aS | aB   (generate a's, then switch to B)
B → bB | b    (generate b's, end with terminal)
Write Right Linear Grammar for L = { (a+b)*b }
Strings ending with 'b'. Can have anything before.
S → aS | bS | b
Write Right Linear Grammar for L = { a*b+ } = strings with any a's then one or more b's
S → aS | bA | b
A → bA | b
S handles leading a's and transition to b's. A handles more b's.
Write Left Linear Grammar for L = { aⁿbᵐ | n,m ≥ 1 }
In left linear, non-terminal is at the LEFT side of production.
S → Sa | Ab
A → Ab | Aa | a
Note: Left linear generates strings right-to-left (generates in reverse order conceptually).

6. Conversion Between Right and Left Linear

Any Right Linear Grammar can be converted to an equivalent Left Linear Grammar (and vice versa) because both represent Regular Languages. The conversion goes through FA:
  • Right Linear Grammar → NFA → Reverse NFA → Left Linear Grammar

7. Practice Questions — §6

Q1 — Write Right Linear Grammar for L = {0ⁿ1ᵐ | n ≥ 1, m ≥ 0}
Must have at least one 0. May have zero or more 1's after.
S → 0S | 0A | 0
A → 1A | 1
OR more simply:
S → 0S | 0B
B → 1B | ε   (if ε-productions allowed)
Without ε: S → 0S | 0 | 0B; B → 1B | 1
Q2 — Write Right Linear Grammar for L = strings over {a,b} that do NOT contain substring 'aa'
After reading 'a', the next must be 'b' (not another 'a'). Use states: q₀ (no a last), q₁ (last was a), dead.
S → aA | bS | ε   (S = no 'a' just seen, or start)
A → bS | b        (A = just saw one 'a' — must take 'b' or end)
A cannot go to another 'a' (that would create 'aa')

§7 Conversion: FA → Regular Grammar

1. Method: FSA to Regular Grammar

Given FSA M = (Q, Σ, δ, q₀, F), construct Regular Grammar G = (V_N, Σ, P, S) where:

  • Each FSA state qᵢ becomes a Non-Terminal in V_N
  • Start state q₀ becomes Start Symbol S
Conversion Rules
If δ(q, a) = p and p is NOT a final state:
    q → aP    (right linear production)

If δ(q, a) = p and p IS a final state:
    q → aP | a   (can stop here OR continue)

2. Worked Example 1

FSA: q₀ →(a)→ q₀ self-loop, q₀ →(b)→ q₁ (final), q₁ →(a,b)→ q₁ self-loop
States: A=q₀ (initial), B=q₁ (final)
Transitions: δ(q₀,a)=q₀, δ(q₀,b)=q₁(final), δ(q₁,a)=q₁(final), δ(q₁,b)=q₁(final)
A → aA | bB | b    [δ(q₀,a)=q₀ not final → A→aA; δ(q₀,b)=q₁ final → A→bB|b]
B → aB | a | bB | b  [δ(q₁,a)=q₁ final → B→aB|a; δ(q₁,b)=q₁ final → B→bB|b]

3. Worked Example 2 — Three-state DFA

DFA: q₀(A) →b→ q₀, q₀ →a→ q₁(B), q₁ →a→ q₁, q₁ →b→ q₂(C,final), q₂ →b→ q₂, q₂ →a→ q₂
States: A=q₀(init), B=q₁, C=q₂(final). Apply conversion rules:
A (q₀): δ(q₀,b)=q₀(not final) → A→bA; δ(q₀,a)=q₁(not final) → A→aB
  ⇒ A → bA | aB

B (q₁): δ(q₁,a)=q₁(not final) → B→aB; δ(q₁,b)=q₂(final) → B→bC | b
  ⇒ B → aB | bC | b

C (q₂): δ(q₂,b)=q₂(final) → C→bC|b; δ(q₂,a)=q₂(final) → C→aC|a
  ⇒ C → bC | b | aC | a

4. Important Notes on FSA → Grammar

  • Each STATE of FSA → one NON-TERMINAL in grammar
  • INITIAL state → START SYMBOL (S)
  • If destination state p is FINAL: add TWO rules: q → ap AND q → a (can terminate)
  • If destination state p is NOT FINAL: add only: q → ap (must continue)
  • The grammar produced is always Right Linear

5. Practice Questions — §7

Q1 — Convert FSA to Regular Grammar: q₀ →(a)→ q₁, q₁ →(b)→ q₂(final). Σ={a,b}. Missing transitions go to dead state.
States: q₀=A(init), q₁=B, q₂=C(final). Dead state=D(non-final).
A → aB | bD     [δ(q₀,a)=q₁(not final) → A→aB; δ(q₀,b)=dead]
B → bC | b | aD  [δ(q₁,b)=q₂(final) → B→bC|b; a→dead]
C → aD | bD     [q₂ is final but all transitions go to dead]
D → aD | bD     [dead state loops]
Language: strings matching exactly 'ab'. Grammar simplification: A → aB; B → b (removing dead state paths)
Q2 — FSA: q₀(init+final) →(a)→ q₁, →(b)→ q₀; q₁(final) →(a)→ q₁, →(b)→ q₀. Convert to grammar.
q₀ is BOTH initial AND final. q₁ is also final.
S (q₀, final): δ(q₀,a)=q₁(final) → S→aA|a; δ(q₀,b)=q₀(final) → S→bS|b
A (q₁, final): δ(q₁,a)=q₁(final) → A→aA|a; δ(q₁,b)=q₀(final) → A→bS|b

Grammar:
S → aA | a | bS | b
A → aA | a | bS | b

§8 Conversion: Regular Grammar → FA

1. Method: Regular Grammar to FSA

Given Right Linear Grammar G = (V_N, T, P, S), construct NFA M = (Q, Σ, δ, q₀, F) where:

  • Each Non-Terminal Aᵢ → one STATE in FSA
  • Start Symbol S → Initial State q₀
  • Create a new Final State (e.g., qf)
Conversion Rules
Rule: Aᵢ → a  (just terminal)
   → Create transition: δ(Aᵢ, a) = qf  (move to final state)

Rule: Aᵢ → aAⱼ  (terminal + non-terminal)
   → Create transition: δ(Aᵢ, a) = Aⱼ

2. Worked Example 1

Grammar: S → bS | aA  |  A → bA | aB | a  |  B → bB | b
NTs: S, A, B. Create states S, A, B, + new final state F.
S→bS: δ(S,b)=S    S→aA: δ(S,a)=A
A→bA: δ(A,b)=A   A→aB: δ(A,a)=B   A→a: δ(A,a)=F
B→bB: δ(B,b)=B   B→b: δ(B,b)=F
Initial state: S. Final states: {F}. This gives the NFA for the language.

3. Worked Example 2 — Complete Conversion with Diagram

Grammar: A → bA | aB | ε  |  B → aC | bB  |  C → aA | bC | a
Start = A (initial). Note: A→ε means A itself is a final state (it can generate empty string).
From A: δ(A,b)=A, δ(A,a)=B; A→ε means A is also final
From B: δ(B,a)=C, δ(B,b)=B
From C: δ(C,a)=A, δ(C,b)=C; C→a: δ(C,a)=F

States: {A, B, C, F}. Initial: A. Final: {A, F}

Q1 — Convert to FSA: S → aS | bA  |  A → bA | b
States: S (initial), A, F (final). Create transitions:
S→aS: δ(S,a)=S     S→bA: δ(S,b)=A
A→bA: δ(A,b)=A    A→b: δ(A,b)=F

δ(S,b) = B

δ(B,b) = {B, F} (B→bB and B→b)
