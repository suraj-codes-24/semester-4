Section 1 — Introduction to Theory of Computation

1. Theory of Computation — Overview

Theory of Computation is the branch of computer science that deals with how efficiently problems can be solved on a model of computation, using an algorithm. It is divided into three major areas:
Automata Theory
Study of abstract machines (automata) and computational problems solvable using them.

Deals with: DFA, NFA, PDA, Turing Machines
Computability Theory
Determines what problems CAN be solved (decidable) vs what CANNOT be solved (undecidable) by any algorithm.

Deals with: Halting Problem, Decidability
Complexity Theory
Classifies problems by the resources (time, space) needed to solve them.

Deals with: P, NP, NP-Complete, NP-Hard

2. Basic Terminology

Symbol (σ)

A symbol (also called a character) is the most basic, indivisible unit of a language. It is a primitive entity — a letter, digit, or any character.
Examples: a, b, 0, 1, #, $

Alphabet (Σ)

An alphabet is a finite, non-empty set of symbols. It is usually denoted by Σ (sigma).

Examples:Σ = {a, b}  |  Σ = {0, 1}  |  Σ = {a, b, c}  |  Σ = {0, 1, 2, ..., 9}
Note:Σ must be finite and non-empty.

String (w)

A string (also called a word) is a finite sequence of symbols chosen from an alphabet Σ.
The length of a string w is written as |w| = number of symbols in w.

Example:Σ = {a,b}  →  w = abba  →  |w| = 4
Empty string:ε (epsilon) — string with zero symbols, |ε| = 0

4. Formal Language

A Formal Language L over an alphabet Σ is any subset of Σ*.
i.e., L ⊆ Σ*

It is a well-defined set of strings — each string either belongs to L or does not.

6. Automata

An automaton (plural: automata) is an abstract computational model — a mathematical machine that reads input and changes states according to transition rules. It is used to decide whether a string belongs to a language.

Components of an Automaton

  • Input Tape: A linear sequence of cells holding input symbols, read left to right
  • Read Head: Points to the current symbol being read
  • Finite Control: The current state — the "memory" of the machine
  • State Set (Q): Finite set of all possible states
  • Transition Function (δ): Rules for moving between states on input symbols
  • Initial State (q₀): The starting state
  • Final/Accept States (F): States that indicate acceptance

Section 2 — Deterministic Finite Automaton (DFA)

1. Definition of DFA

A Deterministic Finite Automaton (DFA) is a 5-tuple: M = (Q, Σ, δ, q₀, F)

where:
Q= Finite non-empty set of states
Σ= Finite non-empty input alphabet (set of input symbols)
δ= Transition function:  δ: Q × Σ → Q  (maps state+symbol to exactly ONE next state)
q₀= Initial state (start state),  q₀ ∈ Q
F= Set of final (accepting) states,  F ⊆ Q
★ The "Deterministic" property: In a DFA, for every state and every input symbol, there is exactly ONE transition. No ambiguity, no multiple choices, no missing transitions (in complete DFA, dead/trap state handles missing transitions).

2. Representations of DFA

A DFA can be represented in two ways:

① Transition Diagram (State Diagram)

  • States → circles
  • Initial state → arrow (→) pointing to it
  • Final/accepting state → double circle
  • Transitions → directed edges labelled with input symbol

② Transition Table (δ-table)

  • Rows = states  |  Columns = input symbols
  • Entry = next state: δ(current state, symbol)
  • Initial state marked with →
  • Final state marked with *

4. Extended Transition Function δ̂

The extended transition function δ̂ (delta hat) extends δ to handle entire strings (not just one symbol):
δ̂ : Q × Σ* → Q

Base case: δ̂(q, ε) = q   (reading empty string keeps same state)
Recursive case: δ̂(q, wa) = δ(δ̂(q, w), a)   (process w first, then apply a)

5. Acceptability — String and Language

String Acceptance: A string w is accepted by DFA M if δ̂(q₀, w) ∈ F (final state).

Language of a DFA: L(M) = { w ∈ Σ* | δ̂(q₀, w) ∈ F } — the set of ALL strings accepted by M.

7. Language Acceptance by DFA

The language accepted by DFA M = (Q, Σ, δ, q₀, F) is:
L(M) = { w ∈ Σ* | δ̂(q₀, w) ∈ F }

A string w is accepted if and only if the computation starting from q₀ ends in a final state after reading all of w.

8. Dead / Trap State

A dead state (trap state) is a non-accepting state from which there is no way to reach any accepting state. Once entered, it can never exit. All transitions from trap state loop back to itself.
★ Used to make a DFA complete (total/defined for every state-symbol pair).

  • Draw transitions for each symbol from each state

  • Add dead/trap state for any undefined transitions (makes DFA complete)
