Section 3 — NFA & Equivalence with DFA

1. Definition of NFA

A Non-Deterministic Finite Automaton (NFA) is a 5-tuple: M = (Q, Σ, δ, q₀, F)

Same components as DFA, BUT the transition function is different:
δ : Q × Σ → 2^Q   (maps state+symbol to a SET of states, possibly empty)
DFA
→ For each (state, symbol): exactly 1 next state
→ No ambiguity in transitions
→ Dead state needed if no transition
→ δ : Q × Σ → Q
→ Harder to design, easy to simulate
NFA
→ For each (state, symbol): 0, 1, or many next states
→ Multiple paths possible simultaneously
→ Missing transition = implicit dead state
→ δ : Q × Σ → 2^Q (power set)
→ Easier to design, harder to simulate
★ Acceptance in NFA: A string w is accepted by an NFA if there EXISTS AT LEAST ONE computation path that ends in a final state. Even if some paths reject, if any path accepts → string is accepted.

2. NFA Example — Strings ending in 'ab'

L = { w ∈ {a,b}* | w ends with 'ab' }

NFA — 3 states

★ q₀ on 'a' goes to BOTH q₀ and q₁ simultaneously (non-determinism). If ANY path reaches q₂, string is accepted.

3. Equivalence of NFA and DFA

Theorem: For every NFA, there exists an equivalent DFA that accepts the same language.

i.e., NFA and DFA are equivalent in expressive power — both recognise exactly the class of Regular Languages. NFA does NOT recognise more languages than DFA.

4. Subset Construction — NFA to DFA Conversion

The key idea: each state in the DFA = a SET of NFA states (a subset of Q in NFA).

Step 1 — Start state
The start state of the DFA = {q₀} (the set containing only the NFA's start state)
Step 2 — Compute transitions
For each DFA state (set of NFA states S) and each symbol a: DFA δ(S, a) = ∪ δ_NFA(q, a) for all q in S
Step 3 — Add new states
Any new set produced by Step 2 that hasn't been seen yet → add as a new DFA state. Repeat Step 2 for each new state.
Step 4 — Mark final states
A DFA state (set S) is a final state if S contains at least one NFA final state.
Step 5 — Dead state
The empty set ∅ becomes the dead/trap state in the DFA (if reached, stays there, non-accepting).

5. Why NFA is useful despite being equivalent to DFA?

  • NFA is easier and more compact to design for many languages
  • NFA may have exponentially fewer states than equivalent DFA
  • DFA conversion can lead to 2ⁿ states (worst case) from an n-state NFA
  • NFA is used as intermediate step: RE → NFA → DFA
  • Both accept exactly the class of Regular Languages
★ State complexity: An NFA with n states may require up to 2ⁿ states in the equivalent DFA (subset construction). However, in practice many subsets are unreachable and the actual DFA is much smaller.

Section 4 — NFA with ε-Transition

1. NFA with ε-Transitions (ε-NFA)

An ε-NFA is an NFA that also allows ε-transitions — transitions that occur without consuming any input symbol. The machine can spontaneously move to another state for "free".

Transition function: δ : Q × (Σ ∪ {ε}) → 2^Q
ε-NFA Example: L = { aⁿ | n ≥ 0 } ∪ { bᵐ | m ≥ 0 } = strings of only a's OR only b's

2. ε-Closure

The ε-closure of a state q, written ε-closure(q), is the set of all states reachable from q using zero or more ε-transitions (including q itself).

ε-closure({q₁, q₂, ...}) = union of ε-closures of each individual state.
ε-closure Example
Given: q₀ →ε→ q₁ →ε→ q₂, and q₃ →ε→ q₃ (only itself)

ε-closure(q₀) = {q₀, q₁, q₂}  (q₀ reaches q₁ via ε, q₁ reaches q₂ via ε)
ε-closure(q₁) = {q₁, q₂}
ε-closure(q₂) = {q₂}
ε-closure(q₃) = {q₃}

3. Converting ε-NFA to DFA (ε-closure + Subset Construction)

Step 1 — Start state of DFA
DFA start state = ε-closure(q₀) of the ε-NFA
Step 2 — Transition computation
For DFA state S and symbol a:   DFA δ(S,a) = ε-closure( ∪ δ_NFA(q,a) for all q ∈ S )
Step 3 — Repeat until stable
Add each new set produced as a new DFA state. Compute its transitions. Repeat until no new states.
Step 4 — Final states
A DFA state (set S) is final if S contains at least one ε-NFA final state.

4. Equivalence of NFA with and without ε-Transitions

Theorem: For every ε-NFA, there exists an equivalent NFA (without ε-transitions) that accepts the same language.

Consequence: DFA = NFA = ε-NFA in expressive power — all three recognise exactly the Regular Languages.

Method to convert ε-NFA to NFA (remove ε-transitions)

  • Compute ε-closure for every state
  • For new NFA: δ'(q, a) = ε-closure( δ(ε-closure(q), a) )
  • New start state = ε-closure(old start state)
  • State q is final in new NFA if ε-closure(q) contains any old final state
Q2 — Full Worked Example: ε-NFA to NFA without ε
ε-NFA: q₀ →ε→ q₁, q₀ →ε→ q₂, q₁ →a→ q₁, q₂ →b→ q₂. F={q₁,q₂}.
ε-closures:
ε-closure(q₀) = {q₀, q₁, q₂}
ε-closure(q₁) = {q₁}
ε-closure(q₂) = {q₂}

New NFA transitions:
δ'(q₀, a) = ε-closure(δ({q₀,q₁,q₂}, a)) = ε-closure({q₁}) = {q₁}
δ'(q₀, b) = ε-closure(δ({q₀,q₁,q₂}, b)) = ε-closure({q₂}) = {q₂}
δ'(q₁, a) = ε-closure(δ({q₁}, a)) = ε-closure({q₁}) = {q₁}
δ'(q₁, b) = ε-closure(∅) = ∅
δ'(q₂, a) = ε-closure(∅) = ∅
δ'(q₂, b) = ε-closure({q₂}) = {q₂}

Final states in new NFA: q₀ (since ε-closure(q₀) contains q₁,q₂ which are final), q₁, q₂

5. Summary — Three-way Equivalence

DFA  ≡  NFA  ≡  ε-NFA   ⟺   Regular Languages
  • All three models have equal expressive power
  • DFA → NFA: trivial (DFA is a special case of NFA)
  • NFA → DFA: subset construction (may exponentially increase states)
  • ε-NFA → NFA: ε-closure computation + redefine transitions
  • ε-NFA → DFA: ε-closure + subset construction (combined)
★ Important Flow: RE → ε-NFA (Thompson's) → NFA → DFA (Subset) → Minimised DFA. Each conversion is well-defined and the resulting automaton accepts the same language.
