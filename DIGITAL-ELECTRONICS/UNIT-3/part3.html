§8 Half Adder & Full Adder

Half Adder (HA)

A Half Adder is a combinational circuit that adds two single-bit binary numbers and produces a Sum and a Carry. It is called "half" because it cannot accept a carry-in from a previous addition.
ABSum (S)Carry (C)
0000
0110
1010
1101
Boolean Expressions:
S = A ⊕ B (XOR gate)
C = A · B (AND gate)

Gate Count: 1 XOR + 1 AND = 2 gates total
Alternative implementation: Half adder using NAND gates only (universal gate):
S = [(A·B)' · (A·(AB)')' · (B·(AB)')']' — uses 5 NAND gates
Or more simply: Can be built with 4 NAND gates using XOR from NAND identity.

Full Adder (FA)

A Full Adder adds three 1-bit binary numbers: two operand bits (A, B) and a Carry-in (Cin) from the previous stage. It produces a Sum and a Carry-out (Cout). Full adders can be chained to add multi-bit numbers.
ABCinSum (S)Cout
00000
00110
01010
01101
10010
10101
11001
11111

Full Adder — Boolean Expressions

From K-map simplification:
S = A ⊕ B ⊕ Cin
Cout = A·B + B·Cin + A·Cin
Or equivalently:
Cout = A·B + Cin·(A ⊕ B)

Minterms: S = Σm(1,2,4,7), Cout = Σm(3,5,6,7)

Full Adder using Two Half Adders

Construction:
Stage 1 — First Half Adder
HA₁ inputs: A and B
HA₁ outputs: S₁ = A⊕B, C₁ = A·B
Stage 2 — Second Half Adder
HA₂ inputs: S₁ and Cin
HA₂ outputs: S = S₁⊕Cin = A⊕B⊕Cin, C₂ = S₁·Cin
Stage 3 — OR gate
Cout = C₁ + C₂ = A·B + Cin·(A⊕B)
Why can't two HAs alone replace an FA? Because the two HAs might both generate a carry simultaneously, and you need one OR gate to combine them. FA = HA + HA + OR gate. Total: 2 XOR + 2 AND + 1 OR = 5 gates.

Full Adder — Carry Propagation

The carry output Cout has two terms:
Carry Generate (G): G = A·B — carry generated even if Cin=0
Carry Propagate (P): P = A⊕B — carry propagated from Cin
So: Cout = G + P·Cin and Sum = P ⊕ Cin
This G/P concept is fundamental to Carry Lookahead Adder (CLA)!

🧪 SOLVED EXAMPLE — Add binary numbers 1011 + 0110 using Full Adders
Numbers: A = 1011 (11 in decimal), B = 0110 (6 in decimal)
Column-by-column (right to left):
Bit 0: A₀=1, B₀=0, Cin=0 → S₀=1, Cout=0
Bit 1: A₁=1, B₁=1, Cin=0 → S₁=0, Cout=1
Bit 2: A₂=0, B₂=1, Cin=1 → S₂=0, Cout=1
Bit 3: A₃=1, B₃=0, Cin=1 → S₃=0, Cout=1
Final carry: 1 (overflow into bit 4)
Result = 1 0001 = 17 in decimal ✓ (11+6=17)
For n-bit addition: you need n Full Adders chained together, with the Cout of each stage feeding the Cin of the next. The LSB FA can have Cin=0 (or be replaced with a Half Adder if no carry-in is ever needed).

§9 Subtractors

Half Subtractor (HS)

A Half Subtractor subtracts two 1-bit numbers A and B, producing a Difference (D) and a Borrow (B_out). Like the half adder, it cannot accept a borrow-in from a previous stage.
ABDiff (D)Borrow (Bo)
0000
0111
1010
1100
Boolean Expressions:
D = A ⊕ B (XOR gate — same as Half Adder Sum!)
Bo = A'·B (AND gate with A complemented)

Key difference from HA: Borrow = A'·B (not A·B like carry)

Full Subtractor (FS)

A Full Subtractor computes A − B − Bin, where Bin is the Borrow-in from the previous stage. Produces a Difference (D) and Borrow-out (Bout).
ABBinDBout
00000
00111
01011
01101
10010
10100
11000
11111
Expressions (from K-map):
D = A ⊕ B ⊕ Bin
Bout = A'·B + A'·Bin + B·Bin
Or: Bout = A'·B + Bin·(A ⊕ B)'

Adder-Subtractor Circuit

A 4-bit Adder-Subtractor performs both A+B and A−B using the same hardware. It exploits the fact that subtraction can be done as: A − B = A + 2's complement of B = A + B̄ + 1.
Design using mode control M:
— When M=0: XOR gates pass B unchanged → performs A + B
— When M=1: XOR gates invert B (B⊕1 = B̄) + M=1 fed to Cin of first FA = +1 → performs A + B̄ + 1 = A − B (2's complement subtraction)
Each B bit goes through XOR with M: Bᵢ ⊕ M
Overflow detection: In signed arithmetic, overflow occurs when the carry into the MSB ≠ carry out of the MSB. Overflow = Cₙ ⊕ Cₙ₋₁. This can be detected with an XOR gate on the last two carries.

Adder (M=0)

  • B bits pass through XOR unchanged
  • Cin = 0
  • Result = A + B

Subtractor (M=1)

  • B bits inverted by XOR
  • Cin = 1 (adds the +1)
  • Result = A − B

§10 Serial & Parallel Adders

Parallel Adder (Ripple Carry Adder)

A Parallel (Ripple Carry) Adder adds two n-bit numbers simultaneously by cascading n Full Adders. All input bits are presented at the same time, but the carry must ripple through from LSB to MSB. Also called Ripple Carry Adder (RCA).
4-bit Parallel Adder (IC 7483 / 74283):
4 Full Adders connected in cascade. C₀ = 0 (no initial carry)
Each stage: Sᵢ = Aᵢ ⊕ Bᵢ ⊕ Cᵢ and Cᵢ₊₁ = AᵢBᵢ + Cᵢ(Aᵢ ⊕ Bᵢ)
Carry ripples: C₀ → C₁ → C₂ → C₃ → C₄ (final carry out)
Propagation delay problem: Each FA introduces ~2 gate delays for carry. For n-bit adder, worst-case delay = 2n gate delays (carry ripples through all n stages). For n=32 bits, this is 64 gate delays! This motivates the Carry Lookahead Adder.

Carry Lookahead Adder (CLA)

In a CLA, all carry bits are computed simultaneously (in parallel) using Generate (G) and Propagate (P) signals, eliminating the ripple delay. Carry for any bit position is calculated directly from the original inputs.
CLA formulas:
Gᵢ = AᵢBᵢ (Generate), Pᵢ = Aᵢ⊕Bᵢ (Propagate)
C₁ = G₀ + P₀·C₀
C₂ = G₁ + P₁·G₀ + P₁·P₀·C₀
C₃ = G₂ + P₂·G₁ + P₂·P₁·G₀ + P₂·P₁·P₀·C₀
All carries computed with only 2 logic gate delays!

Serial Adder

A Serial Adder adds two n-bit numbers one bit at a time over n clock cycles, using a single Full Adder and shift registers to supply the bits sequentially. The carry is stored in a D flip-flop between cycles.
Serial Adder components:
1. Shift Registers
Two shift registers hold operands A and B. They shift right one bit per clock, feeding the LSB first to the Full Adder.
2. Full Adder
Single FA receives one bit from each register and the carry from the D flip-flop.
3. D Flip-Flop (Carry Register)
Stores the carry output, feeds it as carry-in for the next clock cycle.
4. Sum Register
Collects sum bits serially, shifting in the sum output each clock cycle.

Serial Adder

  • 1 Full Adder only
  • n clock cycles for n-bit add
  • Low hardware, slow speed
  • Needs shift registers + D FF

Parallel Adder

  • n Full Adders required
  • Result in 1 clock cycle
  • High hardware, fast speed
  • No shift registers needed
Trade-off: Serial adder trades speed for hardware savings. Parallel adder trades hardware for speed. In modern CPUs, parallel (CLA) adders are used. Serial adders appear in low-power embedded systems and communication protocols.

§11 BCD Adder

A BCD Adder adds two BCD (Binary Coded Decimal) digits. Each BCD digit is a 4-bit binary number (0–9). When two BCD digits are added, the result must remain a valid BCD digit — if it exceeds 9 (or generates a carry), a correction of +6 (0110) must be added.

Why Add 6 as Correction?

BCD uses only 0000–1001 (0–9). Standard 4-bit binary addition gives 0000–1111 (0–15). The "gap" between valid BCD (max = 9) and binary (max = 15) has 6 invalid states (1010–1111). Adding 6 (0110) "skips over" this invalid zone:
If result > 9: it's in the invalid zone → add 0110 to correct
If carry generated: result exceeded 15 → also add 0110

BCD Addition Algorithm

STEP 1 — Binary Addition
Add the two 4-bit BCD digits using a standard 4-bit binary adder. Get a 5-bit result (4 sum bits + carry C₄).
STEP 2 — Check for Correction Need
Correction needed if: C₄ = 1 (carry out), OR if result S₃S₂S₁S₀ > 1001 (i.e., sum > 9)
Detection: K = C₄ + S₃·S₂ + S₃·S₁
STEP 3 — Add Correction
If K=1, add 0110 to the sum using a second 4-bit adder. K also becomes the carry out to the next BCD digit position.

BCD Adder — Correction Logic

Correction factor detection:
K = C₄ + S₃·S₂ + S₃·S₁

This is because result > 9 when:
— C₄=1 (exceeded 4-bit range), OR
— S₃=1 and S₂=1 (result ≥ 12), OR
— S₃=1 and S₁=1 (result = 10 or 11)

If K=1: add 0110 to S₃S₂S₁S₀ using second adder
K = carry out of the BCD digit
🧪 SOLVED EXAMPLE — BCD Addition: 6 + 7
BCD representation: 6 = 0110, 7 = 0111
Step 1 — Binary add: 0110 + 0111 = 1101 (= 13, no carry C₄=0 but result > 9)
Step 2 — Check K: S₃=1, S₂=1 → K = S₃·S₂ = 1 → Correction needed!
Step 3 — Add 0110: 1101 + 0110 = 10011
Lower nibble: 0011 = 3; Upper carry: 1 → means next digit gets carry 1
BCD result: 1 | 0011 = 13 ✓ (6+7=13, tens digit=1, units digit=3)
🧪 SOLVED EXAMPLE — BCD Addition: 9 + 8
BCD: 9 = 1001, 8 = 1000
Step 1: 1001 + 1000 = 10001 → C₄=1, S=0001
Step 2: C₄=1 → K=1, correction needed
Step 3: 0001 + 0110 = 0111
BCD result: 1 | 0111 = 17 ✓ (9+8=17)
K detection formula is the most asked part. Derive it by listing all cases where binary sum > 9 and using K-map. The formula K = C₄ + S₃S₂ + S₃S₁ covers all cases!
