§4 Decoders

A Decoder is a combinational circuit that converts an n-bit binary code into a unique output for each valid input combination. It has n input lines and up to 2ⁿ output lines (one for each minterm). It performs a code-to-output mapping — each output represents a specific minterm.

2:4 Decoder (2-to-4 Line Decoder)

2 inputs A₁, A₀; 4 outputs D₀–D₃. Enable input E (active HIGH).

EA₁A₀D₀D₁D₂D₃
0XX0000
1001000
1010100
1100010
1110001
Logic equations (E=1):
D₀ = A₁'·A₀' = m₀
D₁ = A₁'·A₀ = m₁
D₂ = A₁·A₀' = m₂
D₃ = A₁·A₀ = m₃

Each output = one minterm!

3:8 Decoder (IC 74138)

  • 3 inputs A₂A₁A₀ → 8 outputs Y₀–Y₇
  • Has 3 enable inputs: G1 (active HIGH), G̅2A, G̅2B (both active LOW)
  • Active only when: G1=1, G̅2A=0, G̅2B=0
  • Outputs are active LOW: selected output goes LOW, others HIGH
  • Yᵢ = (active condition) · mᵢ(A₂,A₁,A₀) — inverted minterm
Exam Tip: IC 74138 outputs are active-LOW (inverted). So if A₂A₁A₀ = 101 (= 5), output Y₅ goes LOW while all others stay HIGH. This is the opposite of a textbook decoder where selected output goes HIGH. Don't mix up the two!

Decoder as Universal Logic Element

An n:2ⁿ decoder generates ALL minterms of n variables. Any Boolean function = sum of selected minterms → connect appropriate decoder outputs to an OR gate.

Procedure — Implement f(A,B,C) using 3:8 Decoder + OR gate:
  1. Express function as sum of minterms: f = Σm(i₁, i₂, ...)
  2. Connect A→A₂, B→A₁, C→A₀ (or any consistent assignment)
  3. Connect outputs Yᵢ₁, Yᵢ₂, ... to an OR gate (outputs of active-HIGH decoder)
  4. OR gate output = f(A,B,C)
  5. For active-LOW decoder (74138): use NAND gate instead of OR gate (De Morgan)
EXAMPLE — Implement f(A,B,C) = Σ(1,3,5,7) using 3:8 Decoder
f = m₁ + m₃ + m₅ + m₇ = B (all odd minterms — C is always 1 here).
Connect Y₁, Y₃, Y₅, Y₇ to a 4-input OR gate.
OR gate output = f(A,B,C) = Σ(1,3,5,7) ✔
Observation: This equals variable C (the LSB) — so f = C. The decoder reveals this automatically.

Cascading Decoders — Building 4:16 from 3:8

4:16 DECODER FROM TWO 3:8 DECODERS (74138)
  1. MSB input bit A₃ → enables one of the two 74138 ICs (via G1/enable).
  2. When A₃=0: first 74138 enabled → produces outputs D₀–D₇.
  3. When A₃=1: second 74138 enabled → produces outputs D₈–D₁₅.
  4. Lower 3 bits A₂A₁A₀ fed to both ICs (both receive same address lines).
  5. Result: 16 mutually exclusive outputs — full 4:16 decoding.

BCD-to-7-Segment Decoder

The BCD-to-7-Segment Decoder converts a 4-bit BCD input (0–9) to the 7 segment control signals (a–g) needed to drive a 7-segment LED display. IC 7447 (active-LOW outputs for common-anode) and IC 7448 (active-HIGH outputs for common-cathode) are the standard ICs.

A 7-segment display has segments labeled a (top) through g (middle), plus dp (decimal point).

§5 Encoders & Priority Encoders

An Encoder is a combinational circuit that converts 2ⁿ input lines (only one active at a time) into an n-bit binary code. It performs the inverse of a decoder — many-to-binary conversion. The input is the "name" and the output is the "binary address."

4:2 Encoder (4-to-2 Line)

D₃D₂D₁D₀A₁A₀
000100
001001
010010
100011
Boolean Equations:

A₁ = D₃ + D₂
A₀ = D₃ + D₁

Implemented with just 2 OR gates!

Limitation: Only one input can be active at a time — no two inputs HIGH simultaneously.

8:3 Encoder (Octal-to-Binary, IC 74148)

  • 8 inputs I₀–I₇ (active LOW), 3 outputs A₂A₁A₀ (active LOW)
  • Enable Input EI (active LOW) + Enable Output EO + Group Signal GS
  • Standard encoder equations: A₂ = I₄'+I₅'+I₆'+I₇' (inverted active-LOW logic)
Problem with basic encoder: If two inputs are active simultaneously (e.g., D₃=1 and D₁=1), the output is incorrect (A₁A₀ = 11 which represents D₃, but D₁ is also active). This is solved by a Priority Encoder.

Priority Encoder

A Priority Encoder resolves the problem of simultaneous active inputs by assigning a priority level to each input. If multiple inputs are active simultaneously, the output corresponds to the highest-priority (highest-numbered) input that is active.

4:2 Priority Encoder truth table (D₃ has highest priority, D₀ lowest):

D₃D₂D₁D₀A₁A₀V (valid)
0000XX0
0001001
001X011
01XX101
1XXX111
Boolean equations for 4:2 Priority Encoder:
A₁ = D₃ + D₂    A₀ = D₃ + D₁·D₂'    V = D₃+D₂+D₁+D₀ (valid output indicator)
Basic Encoder
  • Only one input active at a time
  • Simple OR logic
  • No priority handling
  • Ambiguous when multiple inputs active
Priority Encoder
  • Multiple inputs can be active
  • More complex logic
  • Highest input wins
  • Valid (V) output indicates any input active

§6 Multiplexed Display

7-Segment LED Display

A 7-Segment Display consists of 7 LED segments (labeled a–g) arranged to form digits 0–9 and some letters. Two types: Common Anode (all anodes tied together to Vcc — segments lit by active-LOW signals) and Common Cathode (all cathodes tied to GND — segments lit by active-HIGH signals).
Segment Layout:
  _a_
f|   |b
 |_g_|
e|   |c
 |_d_|  dp
Common Anode (CA):
Segment ON when output = LOW (0)
Use IC 7447

Common Cathode (CC):
Segment ON when output = HIGH (1)
Use IC 7448

Multiplexed Display Concept

Multiplexed Display: When multiple 7-segment displays are used, instead of using a separate decoder for each display (expensive), they share a single decoder. Each display is turned ON briefly in sequence (time-division), so fast that the human eye perceives all digits lit simultaneously. This exploits the persistence of vision (~50Hz refresh minimum).
How Multiplexed Display Works:
  1. A counter/multiplexer selects one digit at a time (digit scanning).
  2. The correct BCD data for that digit is sent to the shared BCD-to-7-segment decoder.
  3. Only the selected digit's common anode/cathode is enabled (via transistor switch).
  4. After a brief ON time (~2ms per digit), move to the next digit.
  5. Full cycle repeats at >50 Hz — all digits appear ON simultaneously to the eye.

Advantages of Multiplexed Display

Non-Multiplexed (Static)
  • One decoder per digit
  • High component count
  • High power consumption
  • PCB space intensive
  • Simple wiring logic
Multiplexed (Dynamic)
  • Single shared decoder
  • Fewer ICs needed
  • Lower average power
  • Compact design
  • Requires timing/scan logic

System Block Diagram

4-DIGIT MULTIPLEXED DISPLAY SYSTEM
  1. Display Data Register: Stores BCD values for all 4 digits (D₃D₂D₁D₀)
  2. 2-bit Counter: Cycles through 00→01→10→11 at scan rate (~2kHz)
  3. 4:1 MUX: Selects correct BCD data for current digit based on counter output
  4. BCD-to-7-Segment Decoder (IC 7447/7448): Converts selected BCD to a–g signals
  5. 1:4 DEMUX (or transistor array): Enables only the currently selected digit's common terminal
  6. Result: Each digit lit for 25% of time but appears continuously ON
Duty Cycle: Each digit is ON for 1/N of the total time (N = number of digits). For 4 digits, duty cycle = 25%. Peak brightness must be 4× static brightness to appear equally bright — achieved by driving LEDs with higher peak current during their ON period.
Exam Tip: The key components in a multiplexed display system are MUX (data selection) + DEMUX (digit enable) + BCD-7-seg decoder + Counter (scan generator). You may be asked to draw the block diagram — memorize this chain. Refresh rate <50Hz causes flickering (human eye can detect it).
