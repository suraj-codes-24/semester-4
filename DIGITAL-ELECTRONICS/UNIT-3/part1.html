§1 Magnitude Comparator

A Magnitude Comparator is a combinational circuit that compares two binary numbers A and B and produces three outputs indicating whether A > B, A = B, or A < B. It is an MSI device widely used in arithmetic and control circuits.

1-Bit Comparator — Truth Table & Equations

ABA>BA=BA<B
00010
01001
10100
11010
Boolean Expressions:

A > B = A·B'
A = B = (A⊕B)' = A'B'+AB
A < B = A'·B

Note: (A>B) + (A=B) + (A<B) = 1 always

2-Bit Comparator Logic

Let A = A₁A₀, B = B₁B₀ where A₁, B₁ are MSBs.

Define equality bits: x₁ = (A₁⊕B₁)' and x₀ = (A₀⊕B₀)'

(A = B)  = x₁ · x₀   (both bits equal)
(A > B) = A₁·B₁' + x₁·A₀·B₀'   (MSB decides, or if equal then LSB)
(A < B) = A₁'·B₁ + x₁·A₀'·B₀   (symmetric to A > B)

Exam Tip: Comparison always starts from the MSB. If A₁ ≠ B₁, result is decided immediately. Only when A₁ = B₁ do we look at A₀ vs B₀. This cascading logic extends to any number of bits.

4-Bit Magnitude Comparator — IC 7485

IC 7485 is the standard MSI 4-bit comparator. Inputs: A₃–A₀, B₃–B₀ (data), plus 3 cascade inputs (IA>B, IA=B, IA<B). Outputs: OA>B, OA=B, OA<B.

Logic Equations (4-bit)

Define: pᵢ = Aᵢ·Bᵢ + Aᵢ'·Bᵢ' (pᵢ = 1 when Aᵢ = Bᵢ)

(A=B) = p₃·p₂·p₁·p₀·IA=B

(A>B) = A₃B₃' + p₃A₂B₂' + p₃p₂A₁B₁' + p₃p₂p₁A₀B₀' + p₃p₂p₁p₀·IA>B

(A<B) = A₃'B₃ + p₃A₂'B₂ + p₃p₂A₁'B₁ + p₃p₂p₁A₀'B₀ + p₃p₂p₁p₀·IA<B

Cascade Inputs (for single-chip use): Set IA=B=1, IA>B=0, IA<B=0 — these are the "neutral" starting conditions meaning "assume equal so far."
Cascading two 7485s for 8-bit comparison:
  1. Lower nibble IC (bits 3–0): Set cascade inputs IA=B=1, IA>B=0, IA<B=0
  2. Connect OA>B, OA=B, OA<B of lower IC → IA>B, IA=B, IA<B of upper IC
  3. Upper nibble IC (bits 7–4): Produces final comparison result
  4. Final outputs OA>B, OA=B, OA<B of upper IC = result for full 8-bit numbers

Applications

  • Sorting and ranking digital data
  • Digital alarm/threshold detection (compare sensor value vs setpoint)
  • Address range checking in memory decoders
  • Priority arbitration circuits
  • Overflow detection in ALUs

Solved Examples

EXAMPLE 1 — 1-Bit Comparator Design
Design a 1-bit magnitude comparator. List all Boolean expressions and the gates needed.
From truth table:
A > B = A·B' → 1 AND gate + 1 NOT gate
A = B = (A⊕B)' → 1 XOR + 1 NOT = 1 XNOR gate
A < B = A'·B → 1 NOT gate + 1 AND gate
Total: 2 NOT gates, 2 AND gates, 1 XNOR gate = 5 gates minimum.
EXAMPLE 2 — Cascaded 7485 (8-bit)
Compare A = 10110111₂ and B = 10110100₂ using two IC 7485s. Find result.
Lower nibble: A₃₋₀ = 0111 (7), B₃₋₀ = 0100 (4) → A > B → OA>B=1, rest 0.
Upper nibble: A₇₋₄ = 1011, B₇₋₄ = 1011 → Equal at upper nibble, so cascade inputs decide.
Cascade input from lower stage: IA>B=1 → Final result: A > B ✔

§2 Multiplexers (MUX)

A Multiplexer (MUX) is a combinational circuit that selects one of 2ⁿ data input lines and transmits it to a single output, controlled by n selection lines. It acts as a many-to-one switch or data selector.

2:1 MUX

SOutput Y
0I₀
1I₁

Y = S'·I₀ + S·I₁

Analogy:
Like a 2-way junction: you press selector S=0 to route I₀ to output, S=1 to route I₁. Think of it as a digital traffic policeman.

4:1 MUX

4 inputs (I₀–I₃), 2 select lines (S₁, S₀), 1 output Y.

S₁S₀YSelected
00I₀Minterm m₀
01I₁Minterm m₁
10I₂Minterm m₂
11I₃Minterm m₃

Y = S₁'S₀'·I₀ + S₁'S₀·I₁ + S₁S₀'·I₂ + S₁S₀·I₃

8:1 MUX (IC 74151)

  • 8 data inputs I₀–I₇, 3 select lines S₂S₁S₀, outputs Y and Y'
  • Has active-LOW enable G̅ — when G̅=0 circuit operates, G̅=1 disables output
  • Y = Σᵢ (mᵢ(S₂,S₁,S₀) · Iᵢ) for i = 0 to 7
Key Insight: Selection lines act like a binary address. S₁S₀ = 10 selects I₂ (binary 10 = decimal 2). Always decode the select lines as a binary number to find which input is chosen.

MUX as Universal Logic Element

A 2ⁿ:1 MUX can implement any Boolean function of n+1 variables. Use n variables as select lines, and connect 0/1/variable/complement to data inputs.

Procedure — Implement f(A,B,C) using 4:1 MUX:
  1. Assign S₁ = A, S₀ = B (two MSBs as select lines). C will appear on data inputs.
  2. For each combination of A,B (=00,01,10,11), look at what f equals for C=0 and C=1.
  3. Determine data input: If f=0,0 → Iᵢ=0 | f=1,1 → Iᵢ=1 | f=0,1 → Iᵢ=C | f=1,0 → Iᵢ=C'
  4. Connect the appropriate signal to each data input Iᵢ.
WORKED EXAMPLE — f(A,B,C) = Σ(1,2,6,7) using 4:1 MUX
Map S₁=A, S₀=B. For each AB pair, check f at C=0 and C=1 using minterm list:
AB=00: minterms 0,1 → f(C=0)=0, f(C=1)=1 → I₀ = C
AB=01: minterms 2,3 → f(C=0)=1, f(C=1)=0 → I₁ = C'
AB=10: minterms 4,5 → f(C=0)=0, f(C=1)=0 → I₂ = 0
AB=11: minterms 6,7 → f(C=0)=1, f(C=1)=1 → I₃ = 1
Connections: I₀=C, I₁=C', I₂=0, I₃=1; S₁=A, S₀=B
Output Y = f(A,B,C) = Σ(1,2,6,7) ✔ — verified by substitution.

Cascading MUX — Building Larger from Smaller

BUILD 16:1 MUX USING 4:1 MUX ICs
  1. Take 4 units of 4:1 MUX — each handles 4 of the 16 inputs. Select with S₁S₀.
  2. Take 1 more 4:1 MUX as "output selector" — select with S₃S₂.
  3. Outputs of first four MUXes → data inputs I₀–I₃ of the final MUX.
  4. Total = 5 units of 4:1 MUX. Final output = selected 1 of 16.

Applications of MUX

  • Data selection & routing — route one of many buses to a single output
  • Boolean function implementation — any logic function without K-map minimization
  • Parallel-to-serial conversion — shift register alternative
  • Time-division multiplexing (TDM) — share a single wire between multiple signals
  • Address decoding in microprocessor systems
  • Data compression and waveform generation
MUX
  • 2ⁿ inputs → 1 output
  • n selection lines
  • Data selector
  • Sender side in TDM
  • IC: 74151 (8:1), 74150 (16:1)
DEMUX
  • 1 input → 2ⁿ outputs
  • n selection lines
  • Data distributor
  • Receiver side in TDM
  • IC: 74138 (1:8), 74154 (1:16)
Common Exam Error: When using 4:1 MUX for a 3-variable function, students often try to put all 3 variables on select lines. Remember: a 4:1 MUX has only 2 select lines! The third variable must go on data inputs. Use 8:1 MUX if you want all 3 variables as select lines (with constant 0 or 1 on data inputs).

§3 Demultiplexers (DEMUX)

A Demultiplexer (DEMUX) receives a single input and routes it to one of 2ⁿ output lines based on n select lines. It is the inverse of a MUX — performs one-to-many distribution. Also called a data distributor.

1:2 DEMUX

SD₀D₁
0I (data)0
10I (data)

D₀ = I·S'    D₁ = I·S

Real-world analogy:
A railway switch point — the single incoming train (data) is directed to one of two tracks (output lines) depending on switch position (select line S).

1:4 DEMUX

1 input I, 2 select lines S₁S₀, 4 outputs D₀–D₃.

S₁S₀ActiveExpression
00D₀D₀ = I·S₁'·S₀'
01D₁D₁ = I·S₁'·S₀
10D₂D₂ = I·S₁·S₀'
11D₃D₃ = I·S₁·S₀
Key Observation: When input I = 1 (logic HIGH), DEMUX acts exactly like a Decoder. Each output Dᵢ = mᵢ(S₁,S₀) — the corresponding minterm. This is why Decoders and DEMUXes share the same hardware structure.

1:8 DEMUX and Standard ICs

  • 1 input, 3 select lines, 8 outputs
  • Dᵢ = I · mᵢ(S₂, S₁, S₀) — each output is input ANDed with a 3-variable minterm
  • Example: D₅ = I·S₂·S₁'·S₀ (since 101₂ = 5)

CASCADING: BUILD 1:16 DEMUX FROM 1:4 DEMUX ICs
  1. First stage: 1 unit of 1:4 DEMUX — uses MSB select bits S₃,S₂. Input I feeds this stage.
  2. Four outputs of first stage → each becomes the input of a second-stage 1:4 DEMUX.
  3. Second stage: 4 units of 1:4 DEMUX — each uses LSB select bits S₁,S₀.
  4. Total = 1 + 4 = 5 units of 1:4 DEMUX → 16 outputs.

DEMUX for Boolean Function Implementation

EXAMPLE — Implement F(A,B,C) = Σ(0,2,5,7) using 1:8 DEMUX
Set input I = 1. Connect A→S₂, B→S₁, C→S₀. Each output Dᵢ = mᵢ.
Outputs D₀, D₂, D₅, D₇ will be HIGH for minterms 0,2,5,7.
Connect these 4 outputs to an OR gate → F = D₀ + D₂ + D₅ + D₇ ✔
This is exactly how decoders implement sum-of-minterms functions!
