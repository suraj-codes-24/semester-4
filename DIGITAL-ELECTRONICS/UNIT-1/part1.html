§1 Digital Systems — Analog vs Digital

1. Analog Signal vs Digital Signal

Analog Signal
Signal in which information is processed continuously.
Output is a continuous waveform (sine wave).
E.g. — sound wave, temperature sensor output
Digital Signal
Signal in which information is processed in a finite sequence of discrete values.
Output is a square wave (0s and 1s).
E.g. — computer data, digital clock

2. Digital System

A Digital System represents symbols (information) with discrete symbols. It operates on binary values (0 and 1) to process, store and transmit information.

3. Advantages of Digital Systems

  • Easier to design — binary logic simplifies circuit design
  • Storage is easy — digital information is easy to store and retrieve
  • Accuracy and precision are greater — no signal degradation
  • Digital circuits are less affected by noise — immune to small voltage fluctuations

4. Representation of a Number System

A Number System is a system of writing to express numbers.

General representation:  Nᵣ = Σ aᵢ · rⁱ  (i from -n to n-1)

where:  r = base (radix),  aᵢ = digit at position i,  N = number with base r
★ Key Insight: Every number system is defined by its base (radix) r. The digits used are from 0 to (r-1). The positional value of each digit increases by a power of r as we move left.

3. Positional Notation

Any number can be expressed as a weighted sum of powers of its base:
(d₃d₂d₁d₀.d₋₁d₋₂)ᵣ = d₃×r³ + d₂×r² + d₁×r¹ + d₀×r⁰ + d₋₁×r⁻¹ + d₋₂×r⁻²
Example: (426)₇ → Decimal using positional notation
(426)₇ = 4×7² + 2×7¹ + 6×7⁰
= 4×49 + 2×7 + 6×1
= 196 + 14 + 6
= (216)₁₀
Example: (426.44)₇ → Decimal (with fractional part)
= 4×7² + 2×7¹ + 6×7⁰ + 4×7⁻¹ + 4×7⁻²
= 196 + 14 + 6 + 4/7 + 4/49
= 216 + 0.571 + 0.082
≈ (216.714)₁₀

4. Signed Binary Numbers

Signed binary numbers represent both positive and negative values. The MSB (Most Significant Bit) is used as the sign bit:
• MSB = 0 → Positive number
• MSB = 1 → Negative number

Representations:
① Sign-Magnitude: MSB = sign, remaining bits = magnitude
② 1's Complement: Flip all bits for negative
③ 2's Complement: Most commonly used — flip all bits and add 1
★ Example: +7 = 0111 (4-bit)  |  -7 = 1111 (sign-magnitude)  |  -7 = 1001 (2's complement of +7 = 0111 → flip → 1000 → add 1 → 1001)

§3 Number System Conversions

1. Decimal → Binary (Division Rule)

Integer part: Repeatedly divide by 2 (base). Write remainders bottom to top (LSB to MSB).
Fractional part: Repeatedly multiply by 2. Write integer part top to bottom.
Q: (41)₁₀ → Binary
41 ÷ 2 = 20 rem 1 ↑
20 ÷ 2 = 10 rem 0 ↑
10 ÷ 2 = 5  rem 0 ↑
5 ÷ 2  = 2  rem 1 ↑
2 ÷ 2  = 1  rem 0 ↑
1 ÷ 2  = 0  rem 1 ↑
= (101001)₂
Q: (0.6875)₁₀ → Binary
0.6875 × 2 = 1.375 ↓
0.375 × 2  = 0.75  ↓
0.75 × 2   = 1.5   ↓
0.5 × 2    = 1.0   ↓
= (0.1011)₂

2. Any Base → Decimal (Multiplication Rule)

Multiply each digit by its positional weight (power of base) and sum all.
N₁₀ = Σ dᵢ × rⁱ  (already covered in §2)

3. Any Base → Any Other Base

General Method:
Source Base → Decimal (multiplication rule) → Target Base (division rule)

Shortcut (Binary ↔ Octal/Hex):
• Binary ↔ Octal: Group binary digits in sets of 3 from decimal point (2³=8)
• Binary ↔ Hex: Group binary digits in sets of 4 from decimal point (2⁴=16)

4. Binary → Octal (Group of 3)

Convert (010110001101011.111100000110)₂ → Octal
Group bits in sets of 3 from decimal point, left and right:
Integer: 010 | 110 | 001 | 101 | 011
           2    6    1    5    3
Fraction: 111 | 100 | 000 | 110
            7    4    0    6
= (26153.7406)₈

5. Binary → Hexadecimal (Group of 4)

Convert (0010110000110101011.11110010)₂ → Hex
Group bits in sets of 4 from decimal point:
Integer: 0010 | 1100 | 0011 | 0101 | 011 (pad → 0011)
            2     C     3     5      3... wait
0010 | 1100 | 0110 | 1011 . 1111 | 0010
  2    |   C   |   6   |   B    .   F   |   2
= (2C6B.F2)₁₆
Hex letter mapping: 10=A, 11=B, 12=C, 13=D, 14=E, 15=F

6. Octal → Binary and Hex → Binary

Octal → Binary: Replace each octal digit with its 3-bit binary equivalent.
Hex → Binary: Replace each hex digit with its 4-bit binary equivalent.

Example: (8421)₈ → 000 | 100 | 010 | 001 = (000100010001)₂

§4 Arithmetic: Complements & Subtraction

1. Complements

Complements are used to simplify the subtraction operation (turn subtraction into addition).

Two types:
① r's complement (radix complement) — also called 2's complement, 10's complement
② (r-1)'s complement (diminished radix complement) — also called 1's complement, 9's complement

2. r's Complement (Radix Complement)

Formula:  r's complement = rⁿ − N
where r = base, n = number of positive digits, N = given number
10's complement of (7634)₁₀
rⁿ - N = 10⁴ - 7634
= 10000 - 7634
= 2366
8's complement of (7432)₈
8⁴ - 7432 = 10000₈ - 7432₈
(in decimal: 4096 - 3866)
= 0346₈
★ For fractional numbers: n = number of digits BEFORE the decimal point. E.g. for 0.7634 → n = 0 (before decimal) → 10⁰ - 0.7634 = 1 - 0.7634 = 0.2366

3. (r-1)'s Complement (Diminished Radix Complement)

Formula:  (r-1)'s complement = (rⁿ - 1) − N
9's complement of (76.32)₁₀
(10² - 1) - 76.32
= 99.99 - 76.32
= 23.67
7's complement of (76.32)₈
(8² - 1) - 76.32
= 77 - 76.32
= 00.46

4. Subtraction Using r's Complement (Steps)

Step 1
Equate number of digits (pad with leading zeros if needed)
Step 2
A − B: Take r's complement of the SUBTRAHEND (B)
Step 3
Add minuend (A) + r's complement of subtrahend
Step 4a
If sum produces a CARRY → discard carry → result is POSITIVE
Step 4b
If sum does NOT produce carry → take r's complement of sum → result is NEGATIVE

6. Subtraction Using (r-1)'s Complement (Steps)

Step 1
Equate number of digits
Step 2
Take (r-1)'s complement of subtrahend
Step 3
Add minuend + (r-1)'s complement of subtrahend
Step 4a (carry)
If sum produces carry: add carry to LSB (end-around carry) → result is POSITIVE
Step 4b (no carry)
No carry → take (r-1)'s complement of sum → place (−) sign → result is NEGATIVE

7. Example: 76532 − 04250 using 9's Complement

9's complement of 04250 = 99999 − 04250 = 95749

+ 95749

Result = +72282 ✓

8. Hexadecimal Subtraction

Q: (AB6)₁₆ − (7FF)₁₆
When subtraction requires borrowing: add base (16) to the digit, borrow from next column.
  A  B  6
−  7  F  F
——————————
Col 1 (rightmost): 6 − F(15): borrow → 6+16=22; 22−15=7
Col 2: B(11)−1(borrow)=10; 10−F(15): borrow → 10+16=26; 26−15=11(B)
Col 3: A(10)−1(borrow)=9; 9−7=2
= (2B7)₁₆

9. Summary — Complement Types

Systemr's Complement(r-1)'s Complement
Decimal (base 10)10's complement: 10ⁿ − N9's complement: (10ⁿ−1) − N
Binary (base 2)2's complement: 2ⁿ − N1's complement: (2ⁿ−1) − N = flip all bits
Octal (base 8)8's complement: 8ⁿ − N7's complement: (8ⁿ−1) − N
Hex (base 16)16's complement: 16ⁿ − N15's complement: (16ⁿ−1) − N
★ 1's complement shortcut: Simply flip all bits.  |  ★ 2's complement shortcut: Flip all bits then add 1.

Q1 — Subtract (1010)₂ − (0111)₂ using 2's complement
2's complement of 0111: flip=1000, add 1=1001
2's complement of (0111): 1000 + 1 = 1001
1010 + 1001 = 10011 → carry generated → discard carry
Result = 0011 = (+3)₁₀ ✓ (10-7=3)

0.9999 + 0.5433 = 1.5432 → carry → discard → +0.5432
