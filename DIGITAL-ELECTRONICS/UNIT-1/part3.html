§9 Karnaugh Map (K-Map)

1. What is a K-Map?

A Karnaugh Map (K-Map) is another way of representing the given information in a truth table. It is a graphical method for minimizing Boolean expressions.

For n variables → K-Map has 2ⁿ cells.
K-Map cells are arranged in Gray code order (adjacent cells differ by only 1 bit).

2. Grouping Rules in K-Map

  • Groups must be powers of 2: 1 (single), 2 (pair), 4 (quad), 8 (octet)
  • Groups can wrap around edges (left-right and top-bottom)
  • Always make the LARGEST possible group first
  • Each 1 must be covered by at least one group
  • A 1 can be in multiple groups (if it helps create larger groups)
  • Minimize the total number of groups for minimum expression

3. Two-Variable K-Map (2² = 4 cells)

A\BB=0B=1
A=0m₀ (00)m₁ (01)
A=1m₂ (10)m₃ (11)

4. Three-Variable K-Map (2³ = 8 cells)

A\BC00011110
A=0m₀m₁m₃m₂
A=1m₄m₅m₇m₆
★ IMPORTANT: Columns are ordered 00, 01, 11, 10 (Gray code order) — NOT 00, 01, 10, 11. This ensures adjacent cells differ by 1 bit only.

8. Don't Care Conditions

Don't Care (X or d): Conditions where the output value doesn't matter (either input combination never occurs, or the output doesn't matter).

Rule: Don't cares can be treated as 0 OR 1 — choose whichever helps form larger groups to simplify the expression more. Always try to use min. as possible.

10. How to Read Group Result

After grouping, for each group:
① List all variables — check which ones CHANGE within the group and which stay CONSTANT.
② Variables that CHANGE → eliminated (not in the product term).
③ Variables that stay CONSTANT:
   • If constant at 1 → include variable uncomplemented (e.g., A)
   • If constant at 0 → include variable complemented (e.g., Ā)

§10 NAND & NOR Implementation

1. Two-Level NAND-NAND = SOP

Any SOP expression can be implemented using only NAND gates in two levels:
• Level 1: Each product term → 1 NAND gate (with inputs = literals)
• Level 2: Final NAND gate (with Level 1 outputs as inputs)

Reason: Double complement of NAND = AND, and NAND of NAND = OR of complements by De Morgan = AND-OR = SOP ✓
Steps for NAND Implementation:
① Express function in minimal SOP using K-Map
② Draw 2-level NAND circuit (product terms at Level 1, OR at Level 2 → both become NAND)
③ If a literal appears directly (single-variable product): connect it to the output NAND with a NOT-NAND bubble

2. Two-Level NOR-NOR = POS

Any POS expression can be implemented using only NOR gates in two levels:
• Level 1: Each sum term → 1 NOR gate
• Level 2: Final NOR gate (with Level 1 outputs)

Reason: NOR-NOR is equivalent to OR-AND = POS ✓
Steps for NOR Implementation:
① Express function in minimal POS using K-Map (group 0s!)
② Draw 2-level NOR circuit (sum terms at Level 1, AND at Level 2 → both become NOR)
③ Complement any variable that appears uncomplemented in POS (invert inputs)

DE — Unit 1 · §10NAND/NOR

3. NAND Implementation — Example

Implement F = x̄y + xȳ + z using NAND gates (from K-Map result)
This is already in SOP form. Convert to NAND-NAND:
F = x̄y + xȳ + z

Level 1 NAND gates (produce complement of each product):
NAND₁: inputs = x̄, y → output = (x̄y)̄
NAND₂: inputs = x, ȳ → output = (xȳ)̄
NAND₃: input = z (single input NAND = NOT) → output = z̄

Level 2 NAND gate:
NAND₄: inputs = (x̄y)̄, (xȳ)̄, z̄ → output = ((x̄y)̄ · (xȳ)̄ · z̄)̄
= x̄y + xȳ + z = F ✓ (by De Morgan applied twice)

4. POS Simplification and NOR Implementation

POS Simplification using K-Map:
★ Group the ZEROS (0s) in the K-Map (instead of 1s)
★ Each group of 0s gives one sum term in POS
★ In sum terms: constant 0 → variable uncomplemented; constant 1 → variable complemented
(This is opposite to SOP convention!)

5. Summary: NAND vs NOR Implementation

PropertyNAND-NANDNOR-NOR
Equivalent toAND-OR = SOPOR-AND = POS
K-Map groupingGroup 1sGroup 0s
Result formMinimal SOPMinimal POS
Level 1 gatesProduct termsSum terms
Level 2 gate1 NAND (OR function)1 NOR (AND function)

6. Practice Questions — §10

Q1 — Implement F = AB + BC + AC using NAND gates only
Level 1 NAND gates:
G₁: (AB)̄ — inputs A, B
G₂: (BC)̄ — inputs B, C
G₃: (AC)̄ — inputs A, C
Level 2 NAND gate:
G₄: ((AB)̄ · (BC)̄ · (AC)̄)̄ = AB + BC + AC = F ✓
Q2 — For POS F = (A+B)(B+C), implement using NOR gates only
Level 1 NOR gates:
G₁: (A+B)̄ — inputs A, B → NOR output
G₂: (B+C)̄ — inputs B, C → NOR output
Level 2 NOR gate:
G₃: ((A+B)̄ + (B+C)̄)̄ = (A+B)(B+C) = F ✓

§11 Quine-McClusky (Tabular) Method

1. What is QMC / Tabulation Method?

Drawback of K-Map: Not suitable when number of variables exceeds 4-5 (becomes complex).

To avoid this complexity → use Quine-McClusky (QMC) method = Tabulation Method.

QMC is a systematic, algorithmic method for minimizing Boolean functions — can handle any number of variables.

2. Steps of QMC Method

Step 1
List all minterms. Convert each to binary. Group by number of 1s (weight/count of 1s).
Step 2
Compare minterms in adjacent groups (groups differing by 1 in count of 1s). If two minterms differ in only 1 bit → combine them. Replace the differing bit with dash (–). Mark both original minterms with ✓.
Step 3
Repeat Step 2 for the new group of combined terms. Only minterms with dashes in the SAME positions can be combined.
Step 4
Unchecked terms (not combined with any other) = Prime Implicants.
Step 5
Create a Prime Implicant Chart. Find minimum cover: select Essential Prime Implicants (columns with only one ✓) first. Then cover remaining minterms.
★ Rule: Only minterms that differ by exactly 1 bit can combine. Combined terms must have dashes in the SAME positions (this is important — different dash positions → cannot combine!).

4. Prime Implicant Chart

After finding all prime implicants, create a PI Chart:
• Rows = Prime Implicants
• Columns = Original minterms
• Place ✓ where a PI covers a minterm

Essential Prime Implicant (EPI): A PI that is the ONLY one covering a particular minterm. EPIs MUST be included in the final expression.

After selecting all EPIs, cover remaining uncovered minterms with fewest additional PIs.

Q1 — Apply QMC to minimize: F(A,B,C,D) = Σm(0,1,2,5,6,7,8,9,10,14)
Group by number of 1s and combine. Find prime implicants.

Q2 — What are Essential Prime Implicants? When are they used?
An Essential Prime Implicant (EPI) is a prime implicant that is the ONLY one that covers at least one particular minterm (that column in the PI chart has only ONE checkmark).
EPIs MUST be included in any minimal cover.
After selecting all EPIs, check if all minterms are covered. If not, select additional PIs to cover remaining minterms (choose PIs that cover the most uncovered minterms).

Complements
r's complement = rⁿ − N
(r-1)'s complement = (rⁿ−1)−N

2's complement: flip + add 1
Carry→discard; no carry→negative
Binary Codes
BCD: 4-bit code per decimal digit
BCD invalid: 1010–1111 (10–15)
XS-3: BCD + 3
Gray: 1-bit change between consecutive
XOR = AB̄ + ĀB (Gray conversion)

AND: Y=A·B (all 1→1)
OR: Y=A+B (any 1→1)
NAND: Y=(AB)̄ = universal
NOR: Y=(A+B)̄ = universal
XOR: Y=A⊕B (different→1)
XNOR: Y=A⊙B (same→1)
Boolean Laws
A+Ā=1; A·Ā=0; Ā̄=A
De Morgan 1: (A+B)̄=ĀB̄
De Morgan 2: (AB)̄=Ā+B̄
Absorption: A+AB=A
Distributive: A+(BC)=(A+B)(A+C)
SOP and POS
SOP: sum of minterms (group 1s)
POS: product of maxterms (group 0s)
Minterm: all vars, 1→A, 0→Ā
Maxterm: all vars, 0→A, 1→Ā
Canonical: add X+X̄ for missing vars
K-Map Rules
2ⁿ cells for n variables

Gray code column order: 00,01,11,10
Don't care: use to enlarge groups
5-var: 2 separate 4-var maps
NAND/NOR Implementation
NAND-NAND = SOP
NOR-NOR = POS
NAND: group 1s in K-Map first
NOR: group 0s in K-Map first
QMC Method
Group minterms by number of 1s

Uncombined = Prime Implicants

PI chart → minimum cover

Common Exam Mistakes

  • K-Map column order MUST be Gray code: 00,01,11,10 (NOT 00,01,10,11)
  • In r's complement subtraction: if carry → POSITIVE; if no carry → take complement of sum = NEGATIVE
  • In (r-1)'s complement: if carry → add 1 to LSB (end-around carry); no carry → take (r-1)'s complement of sum
  • Gray code MSB is always copied as-is (not XORed)
  • BCD addition: if sum > 9 OR generates carry → add 6 (0110)
  • Excess-3 addition: carry → add 3; no carry → subtract 3
  • In POS form: 0→uncomplemented, 1→complemented (OPPOSITE to SOP!)
  • Don't cares in K-Map: try to use them, but don't HAVE to use them
★ Canonical Forms Summary:
SOP Canonical: multiply each term by (x+x̄) for missing variable x
POS Canonical: add (x·x̄) to each sum term for missing variable x  [using A+BC=(A+B)(A+C)]
