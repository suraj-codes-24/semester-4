§1 Introduction to Combinational Logic

📖 Combinational Circuit A combinational circuit is a circuit which does not require any feedback element to process its circuit. The output depends only on the current inputs — not on past inputs or outputs. These circuits have a combination of n inputs and m outputs.
n inputs m outputs ────────►│ │────────► ────────►│ Combinational │────────► ────────►│ Circuit │────────► └──────────────────────┘
⭐ Key Property Combinational circuits are memoryless — they have no storage elements (no flip-flops, no latches). The output is a pure Boolean function of the inputs.

🔨 Design Procedure

For designing any combinational circuit, the following 5 steps must be followed:

Step 1 — Analyze the Problem
Analyze the given problem and identify the number of inputs and outputs required.
Step 2 — Assign Letter Symbols
Assign letter symbols to the input and output lines (e.g. A, B, C for inputs; Y, S, C for outputs).
Step 3 — Write the Truth Table
Write the truth table based on the given specification — list all 2n input combinations.
Step 4 — Simplify with K-Map
Determine the simplified Boolean expression for each output using Karnaugh Map.
Step 5 — Draw the Logic Circuit
Draw the logic circuit for the obtained simplified output expression.
⚠️ Exam Tip In exam questions: if the problem says "design a combinational circuit that…" — follow all 5 steps above. Always draw the K-map before drawing the circuit.

🔨 Truth Table Construction Steps

To obtain a truth table from a logic diagram:

  • Step 1: Determine the number of input variables n; list 2n binary combinations from 0 to 2n−1
  • Step 2: Label the output of each selected gate with an arbitrary symbol
  • Step 3: Obtain the truth table for the outputs of those gates that are a function of the input variables
  • Step 4: Proceed to obtain the truth table for the output of gates that are functions of previously defined values, until columns of all outputs are determined

🔥 Worked Example — Design Problem

Design a combinational circuit with 4 inputs and 1 output. The output is 1 when the input is greater than 1000 (binary).

🖊️ Solution
Input greater than 1000₂ (= 8 in decimal) means inputs 9–15 give output Y = 1.
From K-map (4-variable): group the minterms 9,10,11,12,13,14,15
K-map yields groups → AB, AD, AC
Simplified Expression: Y = AB + AD + AC
Logic circuit: Three AND gates (AB, AD, AC) feeding one OR gate → Y

✅ Combinational Circuits

  • No memory elements
  • Output = f(current inputs)
  • Examples: Adder, MUX, Decoder
  • No feedback paths

❌ Sequential Circuits

  • Has memory (flip-flops)
  • Output = f(inputs + past state)
  • Examples: Counter, Register
  • Has feedback paths

§2 Magnitude Comparator

📖 Magnitude Comparator A Magnitude Comparator is a combinational circuit designed primarily to compare the relative magnitude of two binary numbers A and B. The result of the comparison is specified by three binary output variables that indicate whether A > B, A < B, or A = B.
┌──────────────────┐ 2 inputs ──►│ Comparator │──► A > B │ │──► A < B └──────────────────┘──► A = B

⚡ Single-Bit Comparator

Compares two single bits A and B:

ABA > BA < BA = B
00001
01010
10100
11001
📐 Boolean Expressions — Single Bit A = B → Ā·B̄ + A·B = A ⊙ B (XNOR)
A > B → A·B̄
A < B → Ā·B
⚠️ Exam Tip A = B uses XNOR gate. Remember: XNOR gives 1 when both inputs are same. A > B needs A=1, B=0 (so AND with B̄). A < B needs A=0, B=1 (so AND with Ā).

Logic Circuit — Single Bit Comparator

A ──┬──────────────────────────────► A·B̄ ─► (A>B) │ ┌─── XNOR ──────────────────────► A⊙B ─► (A=B) │ │ B ──┴──┬──┘─ NOT(A) ─► Ā·B ─────────────► (A<B) └──── NOT(B) ─► A·B̄ ─────────────► (A>B)

🎯 Simplified Expressions — 2-Bit Comparator

📐 From K-Map Simplification A > B: A₂B̄₂ + A₁B̄₂B̄₁ + A₂A₁B̄₁

A < B: Ā₂B₂ + Ā₂Ā₁B₁ + Ā₁B₂B₁

A = B: A₂⊙A₁⊙B₂⊙B₁  (i.e., XNOR of corresponding bits)
        = Ā₂Ā₁B̄₂B̄₁ + Ā₂A₁B̄₂B₁ + A₂A₁B₂B₁ + A₂Ā₁B₂B̄₁
⭐ Shortcut Formula for A = B (any n-bit) For A = B in an n-bit comparator:
Each bit pair (Aᵢ, Bᵢ) must satisfy Aᵢ = Bᵢ.
So: A=B = (A₂⊙B₂) · (A₁⊙B₁)
Using XNOR for each bit pair, then AND them all together.
🖊️ Quick Check — A > B expression
A > B means A₂A₁ > B₂B₁ (as 2-bit numbers)
If A₂=1, B₂=0 → definitely A > B regardless of lower bits → term: A₂B̄₂
If A₂=B₂=0, then check lower bits: A₁=1, B₁=0 → term: Ā₂A₁B̄₁
If A₂=B₂=1, then check lower bits: A₁=1, B₁=0 → term: A₂A₁B̄₁
Final: A > B = A₂B̄₂ + A₁B̄₂B̄₁ + A₂A₁B̄₁ ✓
⚠️ Common Mistake Students often confuse the MSB and LSB subscripts. Remember: A₂ is the Most Significant Bit, A₁ is LSB. Always check MSB first when evaluating A > B.

§3 Multiplexer (MUX)

📖 Multiplexer A Multiplexer (MUX) is a combinational circuit designed to construct many input lines to one outgoing line. It selects one of 2n input lines and routes it to the output, depending upon the condition of the n selection lines.
I₀ ───┐ I₁ ───┤ I₂ ───┤ ┌─────────┐ I₃ ───┤ │ MUX ├──► Y (output) . ───┤ │ │ . ───┤ └────┬────┘ I₂ₙ──┘ │ S₀ S₁ … Sₙ₋₁ (n selection lines) Rule: 2ⁿ inputs → n selection lines

⚡ Types of MUX

📐 2:1 MUX Inputs: 2 (I₀, I₁)
Selection lines: 1 (S₀)
2¹ = 2 inputs
📐 4:1 MUX Inputs: 4 (I₀…I₃)
Selection lines: 2 (S₁, S₀)
2² = 4 inputs
📐 8:1 MUX Inputs: 8 (I₀…I₇)
Selection lines: 3 (S₂, S₁, S₀)
2³ = 8 inputs
📐 16:1 MUX Inputs: 16 (I₀…I₁₅)
Selection lines: 4 (S₃…S₀)
2⁴ = 16 inputs

⚡ 2:1 MUX — Detailed

Has Enable (E), one selection line (S₀):

ES₀Y (Output)
0×× (disabled)
10I₀
11I₁
📐 Expression: 2:1 MUX Y = S̄·I₀ + S·I₁
When S=0: Y = I₀   |   When S=1: Y = I₁

⚡ 4:1 MUX — Detailed

2 selection lines (S₁, S₀), 4 inputs I₀…I₃:

ES₁S₀Y
0××× (disabled)
100I₀
101I₁
110I₂
111I₃
📐 Expression: 4:1 MUX Y = S̄₁S̄₀·I₀ + S̄₁S₀·I₁ + S₁S̄₀·I₂ + S₁S₀·I₃

⚡ 8:1 MUX

📐 Expression: 8:1 MUX Y = S̄₂S̄₁S̄₀·I₀ + S̄₂S̄₁S₀·I₁ + S̄₂S₁S̄₀·I₂ + S̄₂S₁S₀·I₃
    + S₂S̄₁S̄₀·I₄ + S₂S̄₁S₀·I₅ + S₂S₁S̄₀·I₆ + S₂S₁S₀·I₇

⚡ Cascading MUX (Building Larger from Smaller)

🔧 Design 8:1 MUX using 4:1 MUX Use two 4:1 MUX blocks, feed their outputs into a 2:1 MUX:
→ MUX1 (4:1): handles I₀–I₃ with selection S₀, S₁
→ MUX2 (4:1): handles I₄–I₇ with selection S₀, S₁
→ Final 2:1 MUX: selects between MUX1 and MUX2 output using S₂
This is called the "Cascading Technique"
I₀─┐ I₄─┐ I₁─┤ 4:1 ─┐ I₅─┤ 4:1 ─┐ I₂─┤ MUX │ I₆─┤ MUX ├─── 2:1 ───► Y I₃─┘ │ I₇─┘ │ MUX S₁,S₀ S₁,S₀ │ S₂
⚠️ Exam Formula — MUX Sizes To design 16:1 from 4:1 MUX: need 5 blocks of 4:1 (4 in first stage + 1 in second stage)
To design 32:1 from 8:1 MUX: need 8:1, 4:1, 2:1 combined
General: Use cascading technique — split inputs into groups.

🔥 Boolean Function Implementation using MUX

Any Boolean function with n variables can be implemented using a 2n:1 MUX directly (or a smaller MUX with one variable as input).

🔧 Steps for MUX Implementation Step 1: Determine the multiplexer size based on input variables.
Step 2: Depending on the minterm given in the function, circle those numbers in the implementation table.
Step 3: If two minterms in a column are NOT circled → place 0 at corresponding MUX input.
Step 4: If two minterms in a column ARE circled → place 1 at corresponding MUX input.
Step 5: If minterm in 2nd row circled but 1st row NOT → apply 2nd row variable to MUX input.
Step 6: If minterm in 1st row circled but 2nd row NOT → apply complement of 2nd row variable.
🖊️ Example: F(A,B,C,D) = Σm(0,1,3,4,8,9,15) — Using 16:1 MUX
Since 4 variables → 2⁴ = 16:1 MUX (if not given in exam, use this)
Connect A,B,C,D to selection lines S₃,S₂,S₁,S₀
Minterms 0,1,3,4,8,9,15 → I₀=1, I₁=1, I₂=0, I₃=1, I₄=1, I₅=0, ..., I₁₅=1
Connect minterms to logic 1 (Vcc) and non-minterms to logic 0 (GND) → that's it!
🖊️ Example: F(A,B,C,D) = Σm(0,1,3,4,7,8,9,11,14,15) — Using 8:1 MUX
8:1 MUX has 3 selection lines → use 3 variables (B,C,D) as selection, A as data
Make table with A=0 (row Ā) and A=1 (row A) for each group of B,C,D values
For each MUX input Iₖ: check minterms 2k (when A=0) and 2k+1 (when A=1)
Both circled → Iₖ = 1  |  Neither → Iₖ = 0  |  Only A=0 → Iₖ = Ā  |  Only A=1 → Iₖ = A
This approach uses one less variable as selection line — saves hardware!

⭐ The "One Less Variable" Trick To implement an n-variable function using a MUX with (n-1) selection lines (2n-1:1):
→ Use (n-1) variables as selection lines
→ Use the nth variable (or its complement) as the data input
→ This halves the MUX size needed!
⚠️ Exam Tip — Which case to use? If exam says "8:1 MUX, 4 variables" → use one variable (A) as data input (4 cases: A, Ā, 0, 1). The remaining 3 variables B,C,D are the selection lines. Make a table with Ā row and A row for each group.

§PQ Practice Questions — Part 1

📝 Section §1 — Combinational Logic

Q1 Design a combinational circuit with 3 inputs and 1 output where the output is 1 when the majority of inputs are 1. Draw the logic circuit.
Hint: Output = 1 for minterms where two or more inputs are 1. Simplify using K-map → AC + BC + AB
Truth Table: Y=1 for minterms 3,5,6,7 (two or more 1s)
K-map simplification: Groups: {3,7}=AC, {5,7}=AB, {6,7}=BC
Expression: Y = AC + BC + AB
Circuit: Three AND gates → one 3-input OR gate
Q2 List the 5 steps of combinational circuit design procedure and explain with an example of a 4-input BCD-to-excess-3 code converter concept.
Hint: Follow: (1) Analyze (2) Assign symbols (3) Truth table (4) K-map (5) Logic circuit
5 Steps: (1) Analyze → 4 BCD inputs (A,B,C,D), 4 outputs (W,X,Y,Z)
(2) Assign symbols → inputs ABCD, outputs WXYZ
(3) Truth table → add 3 (0011) to each BCD digit
(4) K-map → simplify each output
(5) Draw circuit for each simplified expression

📝 Section §2 — Magnitude Comparator

Q3 Design a single-bit magnitude comparator. Write Boolean expressions for A>B, A<B, A=B and draw the logic circuit using basic gates only.
Hint: A=B uses XNOR; A>B = AB̄; A<B = ĀB
A=B: ĀB̄ + AB = A⊙B (XNOR)
A>B: A·B̄ (AND gate with B complemented)
A<B: Ā·B (AND gate with A complemented)
Circuit: Two NOT gates + two AND gates + one XNOR gate
Q4 For a 2-bit comparator, derive the expression for A>B where A=A₂A₁ and B=B₂B₁. Verify with at least two cases.
Hint: MSB has higher priority. Check: (i) A₂=1,B₂=0 always A>B; (ii) A₂=B₂, then check A₁ vs B₁
A>B = A₂B̄₂ + A₁B̄₂B̄₁ + A₂A₁B̄₁
Verify (A=10, B=01): A₂=1,B₂=0 → A₂B̄₂=1 → A>B ✓
Verify (A=11, B=10): A₂=B₂=1, A₁=1,B₁=0 → A₂A₁B̄₁=1 → A>B ✓

  • Design steps: Analyze → Symbol → Truth Table → K-map → Circuit
