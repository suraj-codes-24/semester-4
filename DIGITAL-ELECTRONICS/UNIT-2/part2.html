§4 Demultiplexer (DEMUX)

📖 Demultiplexer (DEMUX) A Demultiplexer is the reverse of a Multiplexer. It routes a single input line to one of 2n output lines, determined by the n selection lines. Also called a "one-to-many" circuit.
┌──────────────────┐ 1 input ──────► │ DEMUX ├──► Y₀ │ ├──► Y₁ n selection ──► │ ├──► Y₂ lines │ │ … └──────────────────┘──► Y₂ₙ₋₁ Rule: 1 input → 2ⁿ outputs, n selection lines

✅ MUX

  • Many → One
  • 2ⁿ inputs, 1 output
  • n selection lines
  • Data selector

✅ DEMUX

  • One → Many
  • 1 input, 2ⁿ outputs
  • n selection lines
  • Data distributor

⚡ 1:2 DEMUX

1 input, 2 outputs, 1 selection line:

SY₀Y₁
0I0
10I
📐 Expressions Y₀ = S̄ · I    Y₁ = S · I

⚡ 1:4 DEMUX

S₁S₀Y₀Y₁Y₂Y₃
00I000
010I00
1000I0
11000I
📐 Expressions — 1:4 DEMUX Y₀ = S̄₁·S̄₀·I   Y₁ = S̄₁·S₀·I   Y₂ = S₁·S̄₀·I   Y₃ = S₁·S₀·I
⚠️ Key Insight A decoder with an enable line works exactly as a DEMUX! The enable = single input I, address inputs = selection lines, outputs = DEMUX outputs. Decoder = DEMUX when enable is used as data input.

⚡ 1:8 DEMUX

3 selection lines (S₂,S₁,S₀), 8 outputs. Each output is active only when corresponding binary code is on selection lines:

S₂S₁S₀Active Output
000Y₀ = I
001Y₁ = I
010Y₂ = I
011Y₃ = I
100Y₄ = I
101Y₅ = I
110Y₆ = I
111Y₇ = I
⭐ General Expression Pattern For a 1:2ⁿ DEMUX, output Yₖ = I · (binary decode of k using selection lines)
Each output = I AND (the corresponding minterm of selection lines)
Only ONE output is active (= I) at any time; all others are 0.
🖊️ Worked Example — 1:4 DEMUX Circuit
Input I, selection S₁, S₀
Need: NOT gates for S̄₁ and S̄₀, then 4 AND gates (3-input each)
Y₀ = S̄₁ · S̄₀ · I → AND(NOT(S₁), NOT(S₀), I)
Circuit: 2 NOT gates + 4 AND gates (3-input). Total: 6 gates.

§5 Decoder

📖 Decoder A Decoder is a multi-input, multi-output combinational logic circuit that converts coded input to coded output. It has n input lines and 2n output lines. At any given time, exactly one output is active (HIGH) corresponding to the input binary code.
┌──────────────┐ 2ⁿ outputs n inputs ──►│ Decoder ├──► Y₀ │ ├──► Y₁ │ │ … └──────────────┘──► Y₂ₙ₋₁

⚡ 1:2 Decoder (1-to-2 Line)

1 input I, 2 outputs Y₀, Y₁:

IY₀Y₁
010
101
📐 Expressions Y₀ = Ī (output 0 active when I=0)
Y₁ = I (output 1 active when I=1)

⚡ 2:4 Decoder (2-to-4 Line)

2 inputs (I₀, I₁), 4 outputs Y₀…Y₃:

I₀I₁Y₀Y₁Y₂Y₃
001000
010100
100010
110001
📐 Expressions — 2:4 Decoder Y₀ = Ī₀·Ī₁   Y₁ = Ī₀·I₁
Y₂ = I₀·Ī₁   Y₃ = I₀·I₁
⚠️ Important Pattern Each output of an n:2ⁿ decoder represents exactly one minterm of the input variables. Y₀=m₀, Y₁=m₁, ..., Y₃=m₃ etc.

📐 Expressions — 3:8 Decoder Y₀=Ī₂Ī₁Ī₀   Y₁=Ī₂Ī₁I₀   Y₂=Ī₂I₁Ī₀   Y₃=Ī₂I₁I₀

⭐ Decoder as Minterm Generator A 3:8 decoder generates all 8 minterms of 3 variables. To implement any function using decoder:
→ Write function as sum of minterms
→ OR the corresponding decoder outputs
Example: F(A,B,C) = Σm(1,3,5,7) → F = Y₁ + Y₃ + Y₅ + Y₇
This means: connect decoder outputs 1,3,5,7 to an OR gate!

🔥 Expanding Decoders (Cascading)

🔧 Building 3:8 Decoder from Two 2:4 Decoders Use the MSB (I₂) to enable one decoder at a time:
→ Decoder 1 (2:4): enabled when I₂=0 → handles Y₀–Y₃ (minterms 0-3)
→ Decoder 2 (2:4): enabled when I₂=1 → handles Y₄–Y₇ (minterms 4-7)
Inputs I₁,I₀ feed both decoders. Enable = NOT(I₂) for Dec1, I₂ for Dec2.

🔥 Implementing Boolean Functions using Decoder

🖊️ Example: F(A,B,C) = Σm(1,2,4,7) using 3:8 decoder
Connect A,B,C to decoder inputs I₂,I₁,I₀ respectively
F requires minterms 1,2,4,7 → connect Y₁,Y₂,Y₄,Y₇ to an OR gate
F = Y₁ + Y₂ + Y₄ + Y₇ → one 4-input OR gate at output
🖊️ Example: Implement full adder sum using 3:8 decoder
Sum S = A⊕B⊕Cin = Σm(1,2,4,7) in A,B,Cin
Carry C = Σm(3,5,6,7) in A,B,Cin
S = Y₁+Y₂+Y₄+Y₇ (4-input OR gate)
C = Y₃+Y₅+Y₆+Y₇ (4-input OR gate)
⚠️ Exam Tip When using a decoder to implement functions with NAND gates instead of OR gates, use the fact that (Ā·B̄) = (A+B)̄. A NAND decoder uses active-LOW outputs — in that case, connect to NAND gate instead of OR.

§6 Encoder & Priority Encoder

📖 Encoder An encoder is the reverse of a decoder. It has 2n inputs and n output lines. At any time, exactly ONE input is HIGH (active), and the encoder generates the binary code corresponding to that input.
2ⁿ inputs n outputs Y₀ ────┐ Y₁ ────┤ ┌──────────┐ Y₂ ────┤ │ Encoder ├──► A₀ . ────┤ │ ├──► A₁ . ────┤ │ │ … Y₂ₙ───┘ └──────────┘──► Aₙ₋₁

⚡ 2:1 Encoder (2-to-1)

D₁D₀Y
010
101
📐 Expression Y = D₁

⚡ 4:2 Encoder (4-to-2)

D₃D₂D₁D₀Y₂Y₁
000100
001001
010010
100011
📐 Expressions — 4:2 Encoder Y₂ = D₂ + D₃
Y₁ = D₁ + D₃
⚠️ Limitation of Basic Encoder A basic encoder assumes only ONE input is active at a time. If two inputs are active simultaneously, the output is ambiguous/incorrect. → Solution: Priority Encoder

🔥 Priority Encoder

📖 Priority Encoder In a priority encoder, if two or more inputs are active simultaneously, the input with the highest priority takes precedence. It also has an additional output called the Valid Output Indicator (V): V=0 when all inputs are 0; V=1 when at least one input is 1.
📐 4:2 Priority Encoder (D₃ highest priority) Y₁ = D₂ + D₃
Y₀ = D₃ + D̄₂·D₁
V = D₀ + D₁ + D₂ + D₃

⚡ 4:2 Priority Encoder Truth Table

D₃D₂D₁D₀Y₁Y₀V
0000××0
0001001
001×011
01××101
1×××111

(× means don't care — could be 0 or 1, doesn't affect output)

⭐ Role of Valid Output Indicator (V) V = 0: All inputs are zero — output is invalid/meaningless
V = 1: At least one input is 1 — the binary output Y₁Y₀ is valid
Without V, we can't distinguish "D₀=1 → output 00" from "all inputs = 0 → output 00"
⚠️ Exam: Which encoder to use? If not specified in exam → use 4:2 Priority Encoder. It has 2 outputs + 1 valid bit. D₃ has highest priority, D₀ has lowest.
🖊️ Quick Derivation — Y₀ of 4:2 Priority Encoder
Y₀ = 1 when: D₃=1 (→11) OR D₂=0, D₁=1 (→01)
D₃=1 → Y₀=1 (regardless of D₂,D₁,D₀)
D₃=0, D₂=0, D₁=1 → Y₀=1
Y₀ = D₃ + D̄₂·D₁

⚡ BCD to 7-Segment Decoder

📖 BCD to 7-Segment Decoder A BCD to 7-segment decoder takes a 4-bit BCD input (0000–1001) and drives the corresponding segments of a 7-segment display to show the digit 0–9. Inputs 1010–1111 are "don't care" states.
🔧 Design Steps 1. Create truth table with BCD inputs ABCD → 7 outputs a,b,c,d,e,f,g
2. Treat minterms 10–15 as don't cares (d)
3. Apply K-map for each segment separately
4. Simplify and draw circuit

🔥 Multiplexed Display System

📖 Multiplexed Display Instead of using separate decoder+driver for each digit, a multiplexed display shares a single decoder/driver across multiple 7-segment displays. Each display is turned ON for a brief moment in turn — too fast for human eye to notice — creating the illusion of all digits being on simultaneously. This is called time-division multiplexing.
📐 How Multiplexed Display Works 1. A counter cycles through digit positions (display 0, 1, 2, 3…)
2. A MUX selects the BCD data for the currently active digit
3. A BCD-to-7-segment decoder converts it
4. A DEMUX enables the corresponding display
5. The process repeats at ~1 kHz or higher (persistence of vision)
BCD Data MUX Decoder DEMUX Displays Digit 0 ──►┐ ┌──────┐ ┌─► [ 0 ] Digit 1 ──►┤→ 4:1 → │BCD→7 │→ 1:4 → ├─► [ 1 ] Digit 2 ──►┤ MUX │ Seg │ DEMUX ├─► [ 2 ] Digit 3 ──►┘ └──────┘ └─► [ 3 ] ▲ ▲ └──── Counter ─────┘ (cycles 00→01→10→11)
⭐ Advantages of Multiplexed Display
  • Fewer components — one decoder shared
  • Lower power consumption
  • Simpler wiring for multi-digit displays
  • Used in calculators, clocks, meters
⚠️ Exam Tip The refresh rate must be ≥ 60 Hz per digit to avoid flicker. For 4-digit display, counter runs at ≥ 240 Hz. The key components are: Counter + MUX + Decoder + DEMUX (or transistor switches).

§PQ Practice Questions — Part 2

📝 Section §4 — DEMUX & §5 — Decoder

Q1 Explain with truth table: why is a decoder with enable equivalent to a DEMUX? Use 2:4 decoder as example.
Hint: Treat enable E as the single data input I of DEMUX. When E=0, all outputs=0. When E=1, selected output = 1.
2:4 Decoder with Enable E: Y₀ = E·Ī₀·Ī₁, Y₁ = E·Ī₀·I₁, Y₂ = E·I₀·Ī₁, Y₃ = E·I₀·I₁
Compare with 1:4 DEMUX: Y₀ = I·S̄₁·S̄₀, Y₁ = I·S̄₁·S₀, etc.
If E→I (data), I₀→S₀, I₁→S₁: Structures are identical!
∴ A decoder with enable = DEMUX when enable is used as the data input.
Q2 Implement the Boolean function F(A,B,C) = Σm(0,1,3,6) using a 3:8 decoder and external gates. What gate(s) are needed?
Hint: Each output of 3:8 decoder = one minterm. Just OR the required outputs.
Connect A,B,C to decoder inputs.
F = m₀ + m₁ + m₃ + m₆ = Y₀ + Y₁ + Y₃ + Y₆
Connect Y₀, Y₁, Y₃, Y₆ to a 4-input OR gate.
That's it — one 3:8 decoder + one 4-input OR gate.

📝 Section §6 — Encoder & §7 — Display

Q3 Design a 4-input priority encoder with D₃ having highest priority. Write the truth table, derive Boolean expressions, and explain the role of the valid bit V.
Hint: V = D₀+D₁+D₂+D₃. Y₁ = D₂+D₃. Y₀ = D₃ + D̄₂·D₁
Truth Table: (×= don't care)
D₃=1,×,×,× → Y₁Y₀=11, V=1
D₃=0,D₂=1,×,× → Y₁Y₀=10, V=1
D₃=0,D₂=0,D₁=1,× → Y₁Y₀=01, V=1
D₃=0,D₂=0,D₁=0,D₀=1 → Y₁Y₀=00, V=1
All 0 → Y₁Y₀=×× V=0
V: Without V, can't tell if output 00 means D₀=1 or all inputs=0.
Q4 What is a multiplexed display? Draw a block diagram showing how a 4-digit multiplexed display works with a counter, MUX, BCD decoder, and DEMUX.
Hint: 2-bit counter → cycles 00,01,10,11 → selects which digit's BCD data and which display to enable.
Block diagram: 2-bit counter → (i) drives 4:1 MUX selection (to pick correct BCD digit data) AND (ii) drives 1:4 DEMUX to enable correct display
MUX output (BCD data) → BCD-to-7-segment decoder → segment lines (a–g)
DEMUX output → enables one of 4 displays
Counter cycles fast (≥60 Hz per display) → appears all digits ON simultaneously
Advantage: saves hardware — only one decoder needed!

Q5 Differentiate between encoder and decoder. Give the truth table for a 4:2 encoder and derive the output expressions.
Hint: Decoder: n inputs → 2ⁿ outputs (one active). Encoder: 2ⁿ inputs → n outputs (binary code).
Decoder: coded input → coded output (n→2ⁿ), one output active at a time
Encoder: one active input → binary code (2ⁿ→n)
4:2 Encoder truth table: D₀→00, D₁→01, D₂→10, D₃→11
Y₂ = D₂ + D₃ (MSB), Y₁ = D₁ + D₃ (LSB)
Q6 Design a 2:4 decoder using basic gates (NOT and AND). Write all output expressions and draw the complete logic diagram.
Hint: Need 2 NOT gates (for Ī₀ and Ī₁) + 4 two-input AND gates.
Y₀ = Ī₁·Ī₀ (AND of both NOTs)
Y₁ = Ī₁·I₀ (AND of NOT(I₁) and I₀)
Y₂ = I₁·Ī₀ (AND of I₁ and NOT(I₀))
Y₃ = I₁·I₀ (AND of both)
Gates: 2 NOT + 4 AND = 6 gates total
⚠️ Last-Minute Revision Points — Part 2
  • DEMUX: 1 input → 2ⁿ outputs; MUX reverse = DEMUX
  • Decoder: n→2ⁿ; each output = one minterm; use OR gate to implement any function
  • Encoder: 2ⁿ→n; basic encoder: only one input active at a time
  • Priority Encoder: multiple inputs → highest priority wins; needs valid bit V
  • V=0 (all inputs zero), V=1 (at least one input active)
  • 7-segment: digits stored in truth table; 1010–1111 are don't cares
  • Multiplexed display: one decoder, counter+MUX+DEMUX, time-division
