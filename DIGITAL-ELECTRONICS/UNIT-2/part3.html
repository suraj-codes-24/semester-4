§8 Half Adder & Full Adder

📖 Adder Adders are logic circuits used to add binary bits. Arithmetic circuits are designed by the help of combinational circuits to perform addition and subtraction.

⚡ Half Adder (HA)

📖 Half Adder A Half Adder is a circuit containing two inputs (A and B) and two outputs — Sum (S) and Carry (C). It adds two single binary bits.
ABSum (S)Carry (C)
0000
0110
1010
1101
📐 Half Adder Expressions S = A ⊕ B = ĀB + AB̄
C = A · B

🔧 Half Adder Implementations

Using XOR Gate

A ──►┐
      XOR ──► S
B ──►┘
A ──►┐
      AND ──► C
B ──►┘

2 gates total

Using Basic Gates

Need: 2 NOT + 2 AND + 1 OR for Sum
1 AND for Carry
Total: 6 gates
(Less efficient — prefer XOR)
⚠️ Why called HALF adder? Called "half" because it can add only two bits — it cannot add a carry from a previous stage. To add multi-bit numbers, we need the Full Adder.

⚡ Full Adder (FA)

📖 Full Adder A Full Adder contains 3 inputs (A, B, Carry-in Cin) and 2 outputs (Sum S and Carry-out C). It adds three binary bits — the two operand bits plus the carry from the previous stage.
ABCᵢₙSum (S)Carry (C)
00000
00110
01010
01101
10010
10101
11001
11111
📐 Full Adder Expressions Sum: S = A ⊕ B ⊕ Cᵢₙ
   Also: S = ĀB̄Cᵢₙ + ĀBCᵢₙ̄ + AB̄Cᵢₙ̄ + ABCᵢₙ

Carry: C = AB + (A⊕B)·Cᵢₙ
   Also: C = AB + BCᵢₙ + ACᵢₙ
   (Simplified from K-map: C = BCᵢₙ + ACᵢₙ + AB)
⭐ XOR Observation — Sum S = A⊕B⊕Cᵢₙ: XOR gives 1 when odd number of inputs are 1.
Check truth table: S=1 for rows 1,2,4,7 — each has odd number of 1s ✓

🔧 Full Adder using Two Half Adders

🔧 Construction: FA = 2 × HA + 1 OR gate HA1: inputs A,B → outputs S₁ = A⊕B, C₁ = A·B
HA2: inputs S₁, Cᵢₙ → outputs S = S₁⊕Cᵢₙ = A⊕B⊕Cᵢₙ, C₂ = S₁·Cᵢₙ
Final Carry: C = C₁ + C₂ = AB + (A⊕B)·Cᵢₙ
Result: Sum = A⊕B⊕Cᵢₙ, Carry = AB + (A⊕B)Cᵢₙ
A ──►┐ ┌───►┐ HA₁ ─► S₁=A⊕B ───► HA₂ ─► S = A⊕B⊕Cᵢₙ B ──►┘ C₁=AB ──►│ ├───► Cᵢₙ ──────────────────►│ └─► C₂ OR ──► C (carry) │ ▲ └─────── C₁ ────┘

🔧 Full Adder — Carry Simplification

🖊️ Deriving Carry C from canonical form
C is 1 for minterms: 3(011), 5(101), 6(110), 7(111)
K-map (3-variable A,B,Cin):
Groups: {6,7}=AB, {5,7}=ACin, {3,7}=BCin → wait, also {5,7}+{3,7}+{6,7}
C = AB + ACᵢₙ + BCᵢₙ
This can also be written as: C = AB + (A⊕B)·Cᵢₙ
⚠️ Exam: NAND/NOR Implementation To implement HA using NAND gates only:
S = A⊕B requires 4 NAND gates. C = A·B requires 2 NAND gates (NAND then NOT)
Total: 5 NAND gates for complete Half Adder

§9 Half Subtractor & Full Subtractor

⚡ Half Subtractor

📖 Half Subtractor A Half Subtractor has two inputs A (minuend) and B (subtrahend) and two outputs D (difference) and Br (borrow). It performs A - B. D → 0 when A=B; D → 1 when A≠B
ABDifference (D)Borrow (Br)
0000
0111
1010
1100
📐 Half Subtractor Expressions D = A ⊕ B = ĀB + AB̄
Br = Ā · B

Half Adder

S = A⊕B
C = A·B
"Carry" generated when both 1

Half Subtractor

D = A⊕B
Br = Ā·B
"Borrow" when A=0, B=1

⚡ Full Subtractor

📖 Full Subtractor A Full Subtractor performs three-bit subtraction: A - B - Bᵢₙ (where Bᵢₙ = borrow in from previous stage). Outputs: Difference D and Borrow-out Br.
ABBᵢₙDBr
00000
00111
01011
01101
10010
10100
11000
11111
📐 Full Subtractor Expressions (from K-map) Difference D: D = A⊕B⊕Bᵢₙ
 (Same form as Full Adder Sum!)

Borrow Br: Br = ĀC + B·Bᵢₙ + Ā·B
Simplified: Br = Ā·C + B·Bᵢₙ + Ā·B

📐 Full Subtractor — K-map Results

📐 From K-map (from teacher's notes) Difference:
D = ĀB̄Bᵢₙ + ĀBBᵢₙ̄ + AB̄Bᵢₙ̄ + ABBᵢₙ = A⊕B⊕Bᵢₙ

Borrow:
Br = ĀB̄C + ĀBBᵢₙ̄ + ĀBᵢₙ + ABᵢₙ → simplified: ĀC + BBᵢₙ + ĀB
🔧 Borrow Simplification (from teacher's derivation) From Br canonical: Ā B̄ Bᵢₙ + Ā B Bᵢₙ̄ + Ā B Bᵢₙ + A B Bᵢₙ
K-map groups:
  → {ĀB̄Bᵢₙ, ĀBBᵢₙ} = Ā·Bᵢₙ
  → {ĀBBᵢₙ̄, ĀBBᵢₙ} = Ā·B
  → {ĀBBᵢₙ, ABBᵢₙ} = B·Bᵢₙ
Final: Br = Ā·Bᵢₙ + Ā·B + B·Bᵢₙ
🖊️ Binary Subtraction Example using Full Subtractor
Problem: 1101 - 1011 (subtract B from A)
Bit by bit (LSB first): 1-1=0, 0-1=1(borrow), 1-0-borrow=0, 1-1=0
1101 - 1011 = 0010 ✓ (verified: 13 - 11 = 2)
⚠️ Common Confusion Full Adder and Full Subtractor have the SAME expression for Sum/Difference: both = A⊕B⊕Cin/Bin. The difference is in the Carry/Borrow expression — adder: C = AB + (A⊕B)Cin. Subtractor: Br = ĀBin + ĀB + BBin.

§10 Serial & Parallel Adders

✅ Serial Adder

  • Adds one bit at a time
  • Uses shift registers
  • Slower (n clock cycles for n bits)
  • Less hardware
  • One Full Adder + flip-flop

✅ Parallel Adder

  • Adds all bits simultaneously
  • No shift registers needed
  • Faster (one clock cycle)
  • More hardware (n Full Adders)
  • n Full Adders connected in chain

⚡ 4-bit Binary Parallel Adder

📖 4-bit Binary Parallel Adder A 4-bit binary parallel adder uses 4 Full Adders connected in a chain (ripple carry). Augend bits A₃A₂A₁A₀ are added with Addend bits B₃B₂B₁B₀ simultaneously. Carries ripple from FA₀ to FA₃.
A₃B₃ A₂B₂ A₁B₁ A₀B₀ │ │ │ │ │ │ │ │ ┌▼─▼─┐ ┌▼─▼─┐ ┌▼─▼─┐ ┌▼─▼──────── C₀=0 │ FA │◄─C₃─ │ FA │◄─C₂─ │ FA │◄─C₁─ │ FA │ └─┬──┘ └─┬──┘ └─┬──┘ └─┬──┘ │ C₄ │ │ │ ▼ ▼ ▼ ▼ C_out S₃ S₂ S₁ S₀ Note: C₀ = 0 (no initial carry for addition)
📐 4-bit Parallel Adder — Example A = 1101 (13) + B = 1111 (15):
FA₀: 1+1+0 → S₀=0, C₁=1
FA₁: 0+1+1 → S₁=0, C₂=1
FA₂: 1+1+1 → S₂=1, C₃=1
FA₃: 1+1+1 → S₃=1, C₄=1
Result: C₄ S₃S₂S₁S₀ = 1 1100 = 11100₂ = 28 ✓

⚡ 4-bit Binary Adder/Subtractor

📖 Binary Adder/Subtractor A combined circuit that can perform both addition and subtraction on 4-bit numbers using a mode control input M (Mode line). Uses XOR gates on B inputs.
🔧 How Adder/Subtractor Works Each B bit passes through an XOR gate with control line M:
M = 0 (Addition): B ⊕ 0 = B → normal addition (A + B)
M = 1 (Subtraction): B ⊕ 1 = B̄ → B is complemented; M also feeds C₀=1
→ A + B̄ + 1 = A + 2's complement of B = A - B
A₃ B₃ M A₂ B₂ M A₁ B₁ M A₀ B₀ M │ │ │ │ │ │ │ │ │ │ │ │ │ XOR─┘ │ XOR─┘ │ XOR─┘ │ XOR─┘ │ │ │ │ │ │ │ │ ┌▼──▼┐ ┌▼──▼┐ ┌▼──▼┐ ┌▼──▼┐ ◄── C₀=M │ FA │←C₃ │ FA │←C₂ │ FA │←C₁ │ FA │ └─┬──┘ └─┬──┘ └─┬──┘ └─┬──┘ ▼ ▼ ▼ ▼ C_out S₃ S₂ S₁ S₀
⭐ Key Insight When M=1: all B bits inverted (B̄) + C₀=1 → This computes 2's complement of B.
A + (2's complement of B) = A - B
This is why XOR gates are used — they act as controlled inverters.
⚠️ Exam — Subtraction Example Subtract: B=1101 from A=1011 (B-A, so M=1):
B inputs XORed with 1: 0010, plus Cin=1
1011 + 0010 + 1 = 1110 (but note sign interpretation in exam context)

§11 BCD Adder

📖 BCD Adder Circuit A BCD adder circuit adds two BCD digits in parallel and produces a sum digit which is also BCD. BCD (Binary Coded Decimal) uses 4 bits to represent digits 0–9 only (0000–1001). States 1010–1111 are invalid BCD.
🔧 BCD Addition Rules Step 1: Add two 4-bit BCD numbers using straight binary addition.
Step 2: If the 4-bit sum ≤ 9 (0–1001): sum is valid BCD, no correction needed.
Step 3: If 4-bit sum > 9 (1010–1111) OR a carry is generated: sum is invalid. Add 6 (0110) to correct it. This also generates a carry to the next BCD digit.
🖊️ Example 1: BCD 6 + 7 = 13
6 = 0110, 7 = 0111
0110 + 0111 = 1101 (13) → invalid BCD (> 9)
Add 6: 1101 + 0110 = 1 0011
Result: carry=1, sum=0011 → BCD 13 ✓
🖊️ Example 2: BCD 8 + 8 = 16
8 = 1000, 8 = 1000
1000 + 1000 = 1 0000 (carry generated) → invalid BCD
Carry present → add 6: 0000 + 0110 = 0110, carry still = 1
Result: carry=1, sum=0110 → BCD 16 ✓

🔥 BCD Adder — Correction Logic

When is correction (add 6) needed? When output Y (correction) = 1:

📐 Correction Logic Expression (from K-map) Y = AB + AC
Where A,B,C,D are the sum bits (S₃,S₂,S₁,S₀) from the first 4-bit adder.
So: Y = S₃S₂ + S₃S₁
🔧 BCD Adder — Complete Block Diagram Stage 1: 4-bit parallel adder adds the two BCD inputs → gives S₃S₂S₁S₀ + Cout
Correction logic: Y = S₃S₂ + S₃S₁ OR if Cout=1 → correction needed
Stage 2: 4-bit adder adds 0110 (when Y or Cout = 1) to the first sum → final BCD output
Final carry: Cout from Stage 1 OR Cout from Stage 2
⚠️ Why add 6 (0110)? Because BCD skips 6 codes (1010–1111). When a binary result falls in this invalid range, adding 6 "jumps over" the invalid codes and arrives at the correct BCD representation with a carry.

§12 Quick Revision — All Key Formulas

📌 All Formulas at a Glance

🔲 Half Adder
S = A ⊕ B
C = A · B
🔲 Full Adder
S = A⊕B⊕Cᵢₙ
C = AB + ACᵢₙ + BCᵢₙ
🔲 Half Subtractor
D = A ⊕ B
Br = Ā · B
🔲 Full Subtractor
D = A⊕B⊕Bᵢₙ
Br = ĀBᵢₙ + ĀB + BBᵢₙ
🔲 2:1 MUX
Y = S̄·I₀ + S·I₁
🔲 4:1 MUX
Y = S̄₁S̄₀I₀+S̄₁S₀I₁
+S₁S̄₀I₂+S₁S₀I₃
🔲 Comparator
A=B: A⊙B (XNOR)
A>B: A·B̄
A<B: Ā·B
🔲 BCD Correction
Y = S₃S₂ + S₃S₁
(add 0110 if Y=1)

🎯 Key Relationships to Remember

  • MUX: 2ⁿ inputs, n selection lines | DEMUX: reverse
  • Decoder: n→2ⁿ (one output active) | Encoder: 2ⁿ→n
  • Priority Encoder: V=D₃+D₂+D₁+D₀ (valid bit)
  • FA = 2 Half Adders + 1 OR gate
  • 4-bit Parallel Adder = 4 Full Adders in ripple-carry chain
  • Adder/Subtractor: M=0→Add, M=1→Subtract (uses XOR + 2's complement)
  • BCD Adder: Add 6 if sum > 9 or carry generated

⭐ Don't Forget for Exam
  • Combinational circuit: no feedback, no memory, output = f(current input)
  • Design steps: Analyze → Symbol → Truth Table → K-map → Circuit
  • XOR: output 1 when odd number of inputs are 1
  • XNOR: output 1 when inputs are same (used for equality check)
  • BCD valid range: 0000–1001 only; 1010–1111 are invalid/don't care
  • Priority encoder: highest priority D₃; V=0 means all inputs zero
  • Multiplexed display: time-division; counter + MUX + decoder + DEMUX

§PQ Practice Questions — Part 3

📝 Section §8 — Adders

Q1 Design a Full Adder using two Half Adders and one OR gate. Show the complete truth table, derive the expressions for Sum and Carry, and draw the block diagram.
Hint: HA1(A,B)→S₁=A⊕B, C₁=AB. HA2(S₁,Cin)→S=S₁⊕Cin, C₂=S₁·Cin. Final C = C₁+C₂.
HA1: S₁ = A⊕B, C₁ = AB
HA2: S = S₁⊕Cin = A⊕B⊕Cin, C₂ = S₁·Cin = (A⊕B)·Cin
Final Carry: C = C₁ + C₂ = AB + (A⊕B)·Cin
Block diagram: [A,B]→HA1→[S₁,C₁]; [S₁,Cin]→HA2→[S,C₂]; C₁+C₂→C_out
Total: 2 Half Adders + 1 OR gate
Q2 Implement a Half Adder using only NAND gates. How many NAND gates are required? Draw the circuit and verify for inputs A=1, B=1.
Hint: XOR using NAND: 4 gates needed. AND(then NOT) for carry: 2 gates. Total: 5 NAND gates.
S = A⊕B = NAND(NAND(A,NAND(A,B)), NAND(B,NAND(A,B))) → 4 NAND gates
C = A·B = NOT(NAND(A,B)) = NAND(NAND(A,B),NAND(A,B)) → 1 extra NAND = total 5
Verification (A=1, B=1): NAND(1,1)=0; NAND(1,0)=1; NAND(1,0)=1; NAND(1,1)=0 → S=0 ✓
C = NAND(0,0) = 1 → Carry=1 ✓ (1+1=10 in binary)

📝 Section §9 — Subtractors

Q3 Design a Half Subtractor and compare it with a Half Adder. Write the truth table for subtraction A-B and derive Boolean expressions for Difference and Borrow.
Hint: D = A⊕B (same as sum!). Borrow Br = Ā·B (borrow occurs when A=0 and B=1)
Truth Table: A=0,B=0→D=0,Br=0 | A=0,B=1→D=1,Br=1 | A=1,B=0→D=1,Br=0 | A=1,B=1→D=0,Br=0
D = A⊕B (XOR gate, same as Sum in HA)
Br = Ā·B (AND gate with A complemented)
Comparison: HA Sum S=A⊕B = D; but Carry C=A·B ≠ Borrow Br=Ā·B
Q4 Show that the Difference output of a Full Subtractor has the same expression as Sum of a Full Adder. What is the difference between the two circuits?
Hint: Both use A⊕B⊕Cᵢₙ (or Bᵢₙ). The difference is in carry/borrow logic only.
Full Adder Sum: S = A⊕B⊕Cᵢₙ
Full Subtractor Diff: D = A⊕B⊕Bᵢₙ
Both are IDENTICAL in form! XOR of all 3 inputs.
Difference: Carry C = AB + ACᵢₙ + BCᵢₙ vs Borrow Br = ĀBᵢₙ + ĀB + BBᵢₙ
→ Same Sum/Difference logic, different Carry/Borrow logic.

📝 Section §10 — Parallel Adder & §11 — BCD Adder

Q5 Explain the working of a 4-bit binary parallel adder. Add A=1101 and B=1011 using it, showing carry propagation at each Full Adder stage.
Hint: Start from LSB (bit 0). Cin of FA₀ = 0. Each stage's Cout feeds next stage as Cin.
A=1101 (13), B=1011 (11), Expected: 11000 (24)
FA₀: A₀=1, B₀=1, Cin=0 → S₀=0, C₁=1
FA₁: A₁=0, B₁=1, Cin=1 → S₁=0, C₂=1
FA₂: A₂=1, B₂=0, Cin=1 → S₂=0, C₃=1
FA₃: A₃=1, B₃=1, Cin=1 → S₃=1, C₄=1
Result: C₄S₃S₂S₁S₀ = 11000₂ = 24 ✓
Q6 Add BCD digits 7 and 8 using a BCD adder. Show all steps including correction. When and why is the correction needed?
Hint: 7=0111, 8=1000. Binary sum=1111=15 > 9 → correction needed. Add 0110.
Step 1: 7=0111, 8=1000
Step 2: Binary add: 0111+1000 = 1111 (=15 decimal)
Step 3: 15 > 9 → invalid BCD, correction needed
Step 4: Add 6: 1111+0110 = 1 0101
Result: Carry=1, Sum=0101 → BCD 15 ✓
Why? BCD skips hex codes A–F (6 states), so adding 6 jumps over them.
