Section 5 — Finite Automata with Output: Moore & Mealy

Finite Automata with output (also called Finite State Transducers) produce an output for each input symbol or state — unlike DFA/NFA which only accept/reject. Two models: Moore Machine and Mealy Machine.

1. Moore Machine

A Moore Machine is a 6-tuple: M = (Q, Σ, Δ, δ, λ, q₀)

where:
• Q = finite set of states  |  Σ = input alphabet  |  Δ = output alphabet
• δ : Q × Σ → Q = transition function
• λ : Q → Δ = output function (output depends on CURRENT STATE only)
• q₀ = initial state

★ In a Moore machine: output is associated with states. The output for a string = sequence of outputs of states visited (including start state).

★ For input string of length n → output string of length n+1

2. Mealy Machine

A Mealy Machine is a 6-tuple: M = (Q, Σ, Δ, δ, λ, q₀)

Same structure BUT:
• λ : Q × Σ → Δ = output function (output depends on CURRENT STATE and INPUT SYMBOL)
• Output is associated with transitions, not states

★ In a Mealy machine: output is associated with transitions. For input string of length n → output string of length n

Moore Machine
Output on states
Output length = n+1
λ : Q → Δ
Simpler to design
Output written inside state circle
Mealy Machine
Output on transitions
Output length = n
λ : Q × Σ → Δ
More compact (fewer states)
Output written on edge (input/output)

5. Equivalence of Moore and Mealy Machines

Theorem: For every Moore machine, there exists an equivalent Mealy machine and vice versa — both produce the same output for the same input (with possible difference of one leading character in Moore output due to initial state).

Moore → Mealy Conversion

Method: For each transition δ(q, a) = p in Moore machine, create Mealy transition with output = λ_Moore(p) (output of DESTINATION state).
i.e., if Moore has state p with output y, and transition q→p on input a, then in Mealy: edge q→p labelled a/y.
Moore to Mealy — worked example
Moore machine table:
q₀ (out=0): on a → q₁, on b → q₀
q₁ (out=1): on a → q₁, on b → q₂
q₂ (out=0): on a → q₀, on b → q₂
Mealy equivalent: Transfer output of DESTINATION to the edge label:
q₀ →a/λ(q₁)=1→ q₁  |  q₀ →b/λ(q₀)=0→ q₀
q₁ →a/λ(q₁)=1→ q₁  |  q₁ →b/λ(q₂)=0→ q₂
q₂ →a/λ(q₀)=0→ q₀  |  q₂ →b/λ(q₂)=0→ q₂

Method: Each Mealy state that produces different outputs for different transitions may need to be split into multiple Moore states — one for each distinct output value.
New Moore state qᵢ_y: transition to state q produces output y → create state qᵢ_y with λ(qᵢ_y) = y.

Section 6 — Minimization of Finite Automata

1. Why Minimization?

A DFA may have redundant (equivalent) states — states that behave identically for all future inputs. Minimization removes these to produce the smallest equivalent DFA (minimum number of states) that accepts the same language.
  • Unique: The minimised DFA is unique (up to state renaming)
  • Efficient: Fewer states = faster simulation, less memory
  • Canonical form: Used to test if two DFAs are equivalent

2. Key Definitions

Distinguishable States

Two states p and q are distinguishable if there exists a string w such that:
δ̂(p, w) ∈ F  but  δ̂(q, w) ∉ F   (or vice versa)
i.e., one accepts w and the other rejects w.

Indistinguishable / Equivalent States

Two states p and q are indistinguishable (equivalent) if for every string w:
δ̂(p, w) ∈ F  iff  δ̂(q, w) ∈ F   (they always agree on acceptance)
Such states can be merged into a single state.

3. Table-Filling (Myhill-Nerode) Algorithm

Step 1 — Remove unreachable states
Find all states reachable from q₀. Remove any unreachable states — they cannot affect the language.
Step 2 — Base: mark (F, non-F) pairs
Create a triangular table for all pairs (p,q). Mark all pairs where one is final (∈F) and other is non-final (∉F) — these are immediately distinguishable.
Step 3 — Iterate: propagate distinguishability
For each UNMARKED pair (p,q): for each symbol a, check if (δ(p,a), δ(q,a)) is already MARKED. If yes → mark (p,q) too. Repeat until no new marks.
Step 4 — Merge unmarked pairs
All still-UNMARKED pairs (p,q) are indistinguishable → merge them into a single state. Build minimised DFA from merged states.

5. Myhill-Nerode Theorem

Define the Myhill-Nerode equivalence relation ~_L on Σ* as:
x ~_L y  iff  for all z ∈ Σ*: xz ∈ L ↔ yz ∈ L

Theorem: A language L is Regular if and only if ~_L has a finite number of equivalence classes.
Furthermore, the number of equivalence classes = number of states in the minimised DFA.
Q2 — Another minimization example. States: {q₀,q₁,q₂,q₃,q₄}, Σ={a,b}, F={q₃,q₄}
Transition: δ(q₀,a)=q₁, δ(q₀,b)=q₂, δ(q₁,a)=q₃, δ(q₁,b)=q₂, δ(q₂,a)=q₁, δ(q₂,b)=q₄, δ(q₃,a)=q₃, δ(q₃,b)=q₃, δ(q₄,a)=q₄, δ(q₄,b)=q₄
Step 1: All states reachable from q₀. No removal needed.
Step 2: Mark (final, non-final) pairs:
F={q₃,q₄}, non-F={q₀,q₁,q₂}
Mark: (q₀,q₃), (q₀,q₄), (q₁,q₃), (q₁,q₄), (q₂,q₃), (q₂,q₄)
Step 3: Check (q₀,q₁): δ(q₀,a)=q₁, δ(q₁,a)=q₃ → (q₁,q₃) is marked → mark (q₀,q₁) ✗
Check (q₀,q₂): δ(q₀,a)=q₁, δ(q₂,a)=q₁ → (q₁,q₁) same state, ok. δ(q₀,b)=q₂, δ(q₂,b)=q₄ → (q₂,q₄) is marked → mark (q₀,q₂) ✗
Check (q₁,q₂): δ(q₁,a)=q₃, δ(q₂,a)=q₁ → (q₁,q₃) marked → mark (q₁,q₂) ✗
Check (q₃,q₄): δ(q₃,a)=q₃, δ(q₄,a)=q₄ → (q₃,q₄) — same type (both final), need to check further. δ(q₃,b)=q₃, δ(q₄,b)=q₄ → (q₃,q₄) — same reasoning. NOT marked → (q₃,q₄) indistinguishable.
Step 4: Merge q₃ and q₄ → single state q₃₄. All others remain separate (all pairs marked).
Minimised DFA: 4 states {q₀, q₁, q₂, q₃₄}

6. Complete Summary

★ Key Points for Minimization:
  • Step 0: Remove all unreachable states first
  • Initially mark ALL pairs where exactly one is final
  • Iteratively mark (p,q) if δ(p,a) and δ(q,a) are already marked, for any symbol a
  • Stop when no new pairs are marked in a full pass
  • All unmarked pairs → indistinguishable → merge them
  • Result is the unique minimum-state DFA
  • Two DFAs are equivalent iff their minimised DFAs are isomorphic
★ Exam Tip: Always start by drawing the triangular pair table. Fill in the base marks first (final vs non-final = marked). Then work through each unmarked pair systematically for each input symbol. The process always terminates.
★ Complete Unit 1 Flow:
RE → ε-NFA (Thompson's) → NFA (remove ε) → DFA (Subset) → Min-DFA (Table-filling)

§1 — Introduction to Theory of Computation

Q1 (5 marks) — Classify each language below and justify with one property of each class.
Languages given:
(a) L₁ = { aⁿbⁿ | n ≥ 0 }   (b) L₂ = { w | w ∈ {a,b}*, w starts with 'a' }
Step 1 — L₁:
L₁ requires counting equal numbers of a's and b's. A finite automaton has finite memory (finite states), so it cannot "remember" an arbitrary count n.
→ L₁ is NOT regular. It is Context-Free (accepted by a PDA using a stack).
Step 2 — L₂:
L₂ needs only one bit of memory: "did the first symbol = a?" A DFA with 3 states handles this:
  q₀ (start) —a→ q₁ (accept) —{a,b}→ q₁
  q₀ —b→ q₂ (dead/trap) —{a,b}→ q₂
→ L₂ is regular.
Step 3 — Chomsky Hierarchy recap:
Type 3 (Regular) ⊂ Type 2 (CFL) ⊂ Type 1 (CSL) ⊂ Type 0 (RE)
FA recognises Type 3; PDA recognises Type 2.
∴ L₁ — Context-Free (not regular). L₂ — Regular (accepted by 3-state DFA).
Q2 (4 marks) — Define a Deterministic Finite Automaton formally. What is an "accepting computation"?
Formal Definition:
A DFA is a 5-tuple M = (Q, Σ, δ, q₀, F) where:
 • Q = finite, non-empty set of states
 • Σ = finite input alphabet
 • δ : Q × Σ → Q = total transition function
 • q₀ ∈ Q = initial/start state
 • F ⊆ Q = set of final (accepting) states
Extended transition δ*:
δ*(q, ε) = q
δ*(q, wa) = δ(δ*(q, w), a)  for w ∈ Σ*, a ∈ Σ
Accepting Computation:
A string w is accepted by M if δ*(q₀, w) ∈ F.
The language of M: L(M) = { w ∈ Σ* | δ*(q₀, w) ∈ F }
∴ DFA = (Q, Σ, δ, q₀, F). A word is accepted when processing it from q₀ ends in a final state.

Q6 (4 marks) — State and prove that every NFA has an equivalent DFA (key argument only).
Statement: For every NFA N = (Q, Σ, δ, q₀, F) there exists a DFA D = (2^Q, Σ, δ', {q₀}, F') with L(D) = L(N).
Construction:
δ'(S, a) = ∪_{q∈S} δ(q, a) for any subset S ⊆ Q
F' = { S ⊆ Q | S ∩ F ≠ ∅ }
Correctness argument (by induction on |w|):
Claim: δ'*({q₀}, w) = set of all NFA states reachable on w from q₀.
Base: |w|=0 → δ'*({q₀},ε) = {q₀} = NFA states reachable on ε ✓
Step: δ'*({q₀},wa) = δ'(δ'*({q₀},w), a) = ∪_{q∈S} δ(q,a) = NFA reachable states on wa ✓
→ w accepted by D iff δ'*({q₀},w) ∩ F ≠ ∅ iff w accepted by N.
∴ By subset construction, every NFA is equivalent to a DFA. Both recognise the same class — Regular Languages.

States: {A,B,C,D,E}, Σ={0,1}, Start=A, F={C,E}
δ: A—0→B, A—1→C, B—0→B, B—1→D, C—0→B, C—1→C, D—0→B, D—1→E, E—0→B, E—1→C

Pair (A,D): δ(A,0)=B, δ(D,0)=B → same; δ(A,1)=C, δ(D,1)=E → (C,E) both final, not yet marked → keep unmarked for now
Pair (C,E): δ(C,0)=B, δ(E,0)=B → same; δ(C,1)=C, δ(E,1)=C → same → NOT distinguishable → (C,E) stays unmarked
Pair (A,D): δ(A,1)=C, δ(D,1)=E → (C,E) unmarked → (A,D) stays unmarked
Step 3 — Equivalent classes:
Unmarked pairs: {C,E} and {A,D}
Equivalence classes: {A,D}, {C,E}, {B}
Rename: P={A,D}, Q={C,E}, R={B}
Step 4 — Minimized DFA transitions:
P—0→R, P—1→Q ; R—0→R, R—1→P (since δ(B,1)=D∈P) ; Q—0→R, Q—1→Q
Start=P, Accept={Q}
∴ Original 5-state DFA minimizes to 3 states: {P,Q,R}. States C and E are equivalent; states A and D are equivalent.
Q12 (4 marks) — What is a "dead state"? When is a DFA already minimal?
Dead / Trap State:
A state from which no final state is reachable on any input. It acts as a "reject sink". Once entered, the machine stays there forever.
Example: δ(d,0)=d, δ(d,1)=d, d ∉ F.
Condition for minimality:
A DFA M is minimal if:
 (1) It is reachable — every state is reachable from q₀.
 (2) It is distinguishable — no two distinct states are equivalent (i.e., no pair remains unmarked in table-filling).
Uniqueness:
The minimal DFA for a regular language is unique up to state renaming (isomorphism).
∴ A DFA is minimal iff all states are reachable and pairwise distinguishable. Merging equivalent states yields the unique minimal DFA.
