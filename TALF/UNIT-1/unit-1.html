Section 1 — Introduction to Theory of Computation

1. Theory of Computation — Overview

Theory of Computation is the branch of computer science that deals with how efficiently problems can be solved on a model of computation, using an algorithm. It is divided into three major areas:
Automata Theory
Study of abstract machines (automata) and computational problems solvable using them.

Deals with: DFA, NFA, PDA, Turing Machines
Computability Theory
Determines what problems CAN be solved (decidable) vs what CANNOT be solved (undecidable) by any algorithm.

Deals with: Halting Problem, Decidability
Complexity Theory
Classifies problems by the resources (time, space) needed to solve them.

Deals with: P, NP, NP-Complete, NP-Hard

2. Basic Terminology

Symbol (σ)

A symbol (also called a character) is the most basic, indivisible unit of a language. It is a primitive entity — a letter, digit, or any character.
Examples: a, b, 0, 1, #, $

Alphabet (Σ)

An alphabet is a finite, non-empty set of symbols. It is usually denoted by Σ (sigma).

Examples:Σ = {a, b}  |  Σ = {0, 1}  |  Σ = {a, b, c}  |  Σ = {0, 1, 2, ..., 9}
Note:Σ must be finite and non-empty.

String (w)

A string (also called a word) is a finite sequence of symbols chosen from an alphabet Σ.
The length of a string w is written as |w| = number of symbols in w.

Example:Σ = {a,b}  →  w = abba  →  |w| = 4
Empty string:ε (epsilon) — string with zero symbols, |ε| = 0

6. Automata

An automaton (plural: automata) is an abstract computational model — a mathematical machine that reads input and changes states according to transition rules. It is used to decide whether a string belongs to a language.

Components of an Automaton

  • Input Tape: A linear sequence of cells holding input symbols, read left to right
  • Read Head: Points to the current symbol being read
  • Finite Control: The current state — the "memory" of the machine
  • State Set (Q): Finite set of all possible states
  • Transition Function (δ): Rules for moving between states on input symbols
  • Initial State (q₀): The starting state
  • Final/Accept States (F): States that indicate acceptance

Section 2 — Deterministic Finite Automaton (DFA)

1. Definition of DFA

A Deterministic Finite Automaton (DFA) is a 5-tuple: M = (Q, Σ, δ, q₀, F)

where:
Q= Finite non-empty set of states
Σ= Finite non-empty input alphabet (set of input symbols)
δ= Transition function:  δ: Q × Σ → Q  (maps state+symbol to exactly ONE next state)
q₀= Initial state (start state),  q₀ ∈ Q
F= Set of final (accepting) states,  F ⊆ Q
★ The "Deterministic" property: In a DFA, for every state and every input symbol, there is exactly ONE transition. No ambiguity, no multiple choices, no missing transitions (in complete DFA, dead/trap state handles missing transitions).

2. Representations of DFA

A DFA can be represented in two ways:

① Transition Diagram (State Diagram)

  • States → circles
  • Initial state → arrow (→) pointing to it
  • Final/accepting state → double circle
  • Transitions → directed edges labelled with input symbol

② Transition Table (δ-table)

  • Rows = states  |  Columns = input symbols
  • Entry = next state: δ(current state, symbol)
  • Initial state marked with →
  • Final state marked with *

3. DFA Example — Strings ending in 'b'

L = { w ∈ {a,b}* | w ends with b }   e.g. {b, ab, aab, bab, abb, …}

4. Extended Transition Function δ̂

The extended transition function δ̂ (delta hat) extends δ to handle entire strings (not just one symbol):
δ̂ : Q × Σ* → Q

Base case: δ̂(q, ε) = q   (reading empty string keeps same state)
Recursive case: δ̂(q, wa) = δ(δ̂(q, w), a)   (process w first, then apply a)

5. Acceptability — String and Language

String Acceptance: A string w is accepted by DFA M if δ̂(q₀, w) ∈ F (final state).

Language of a DFA: L(M) = { w ∈ Σ* | δ̂(q₀, w) ∈ F } — the set of ALL strings accepted by M.

7. Language Acceptance by DFA

The language accepted by DFA M = (Q, Σ, δ, q₀, F) is:
L(M) = { w ∈ Σ* | δ̂(q₀, w) ∈ F }

A string w is accepted if and only if the computation starting from q₀ ends in a final state after reading all of w.

8. Dead / Trap State

A dead state (trap state) is a non-accepting state from which there is no way to reach any accepting state. Once entered, it can never exit. All transitions from trap state loop back to itself.
★ Used to make a DFA complete (total/defined for every state-symbol pair).

9. More DFA Examples

q₀ = even count of a's (accept). q₁ = odd count of a's (reject). b loops in same state (doesn't change count). Parity tracking — classic 2-state DFA.

Section 3 — NFA & Equivalence with DFA

1. Definition of NFA

A Non-Deterministic Finite Automaton (NFA) is a 5-tuple: M = (Q, Σ, δ, q₀, F)

Same components as DFA, BUT the transition function is different:
δ : Q × Σ → 2^Q   (maps state+symbol to a SET of states, possibly empty)
DFA
→ For each (state, symbol): exactly 1 next state
→ No ambiguity in transitions
→ Dead state needed if no transition
→ δ : Q × Σ → Q
→ Harder to design, easy to simulate
NFA
→ For each (state, symbol): 0, 1, or many next states
→ Multiple paths possible simultaneously
→ Missing transition = implicit dead state
→ δ : Q × Σ → 2^Q (power set)
→ Easier to design, harder to simulate
★ Acceptance in NFA: A string w is accepted by an NFA if there EXISTS AT LEAST ONE computation path that ends in a final state. Even if some paths reject, if any path accepts → string is accepted.

2. NFA Example — Strings ending in 'ab'

★ q₀ on 'a' goes to BOTH q₀ and q₁ simultaneously (non-determinism). If ANY path reaches q₂, string is accepted.

3. Equivalence of NFA and DFA

Theorem: For every NFA, there exists an equivalent DFA that accepts the same language.

i.e., NFA and DFA are equivalent in expressive power — both recognise exactly the class of Regular Languages. NFA does NOT recognise more languages than DFA.

4. Subset Construction — NFA to DFA Conversion

The key idea: each state in the DFA = a SET of NFA states (a subset of Q in NFA).

Step 1 — Start state
The start state of the DFA = {q₀} (the set containing only the NFA's start state)
Step 2 — Compute transitions
For each DFA state (set of NFA states S) and each symbol a: DFA δ(S, a) = ∪ δ_NFA(q, a) for all q in S
Step 3 — Add new states
Any new set produced by Step 2 that hasn't been seen yet → add as a new DFA state. Repeat Step 2 for each new state.
Step 4 — Mark final states
A DFA state (set S) is a final state if S contains at least one NFA final state.
Step 5 — Dead state
The empty set ∅ becomes the dead/trap state in the DFA (if reached, stays there, non-accepting).

5. Why NFA is useful despite being equivalent to DFA?

  • NFA is easier and more compact to design for many languages
  • NFA may have exponentially fewer states than equivalent DFA
  • DFA conversion can lead to 2ⁿ states (worst case) from an n-state NFA
  • NFA is used as intermediate step: RE → NFA → DFA
  • Both accept exactly the class of Regular Languages
★ State complexity: An NFA with n states may require up to 2ⁿ states in the equivalent DFA (subset construction). However, in practice many subsets are unreachable and the actual DFA is much smaller.

Section 4 — NFA with ε-Transition

1. NFA with ε-Transitions (ε-NFA)

An ε-NFA is an NFA that also allows ε-transitions — transitions that occur without consuming any input symbol. The machine can spontaneously move to another state for "free".

Transition function: δ : Q × (Σ ∪ {ε}) → 2^Q
ε-NFA Example: L = { aⁿ | n ≥ 0 } ∪ { bᵐ | m ≥ 0 } = strings of only a's OR only b's

2. ε-Closure

The ε-closure of a state q, written ε-closure(q), is the set of all states reachable from q using zero or more ε-transitions (including q itself).

ε-closure({q₁, q₂, ...}) = union of ε-closures of each individual state.
ε-closure Example
Given: q₀ →ε→ q₁ →ε→ q₂, and q₃ →ε→ q₃ (only itself)

ε-closure(q₀) = {q₀, q₁, q₂}  (q₀ reaches q₁ via ε, q₁ reaches q₂ via ε)
ε-closure(q₁) = {q₁, q₂}
ε-closure(q₂) = {q₂}
ε-closure(q₃) = {q₃}

3. Converting ε-NFA to DFA (ε-closure + Subset Construction)

Step 1 — Start state of DFA
DFA start state = ε-closure(q₀) of the ε-NFA
Step 2 — Transition computation
For DFA state S and symbol a:   DFA δ(S,a) = ε-closure( ∪ δ_NFA(q,a) for all q ∈ S )
Step 3 — Repeat until stable
Add each new set produced as a new DFA state. Compute its transitions. Repeat until no new states.
Step 4 — Final states
A DFA state (set S) is final if S contains at least one ε-NFA final state.

4. Equivalence of NFA with and without ε-Transitions

Theorem: For every ε-NFA, there exists an equivalent NFA (without ε-transitions) that accepts the same language.

Consequence: DFA = NFA = ε-NFA in expressive power — all three recognise exactly the Regular Languages.

Method to convert ε-NFA to NFA (remove ε-transitions)

  • Compute ε-closure for every state
  • For new NFA: δ'(q, a) = ε-closure( δ(ε-closure(q), a) )
  • New start state = ε-closure(old start state)
  • State q is final in new NFA if ε-closure(q) contains any old final state
Q2 — Full Worked Example: ε-NFA to NFA without ε
ε-NFA: q₀ →ε→ q₁, q₀ →ε→ q₂, q₁ →a→ q₁, q₂ →b→ q₂. F={q₁,q₂}.
ε-closures:
ε-closure(q₀) = {q₀, q₁, q₂}
ε-closure(q₁) = {q₁}
ε-closure(q₂) = {q₂}

New NFA transitions:
δ'(q₀, a) = ε-closure(δ({q₀,q₁,q₂}, a)) = ε-closure({q₁}) = {q₁}
δ'(q₀, b) = ε-closure(δ({q₀,q₁,q₂}, b)) = ε-closure({q₂}) = {q₂}
δ'(q₁, a) = ε-closure(δ({q₁}, a)) = ε-closure({q₁}) = {q₁}
δ'(q₁, b) = ε-closure(∅) = ∅
δ'(q₂, a) = ε-closure(∅) = ∅
δ'(q₂, b) = ε-closure({q₂}) = {q₂}

Final states in new NFA: q₀ (since ε-closure(q₀) contains q₁,q₂ which are final), q₁, q₂

5. Summary — Three-way Equivalence

DFA  ≡  NFA  ≡  ε-NFA   ⟺   Regular Languages
  • All three models have equal expressive power
  • DFA → NFA: trivial (DFA is a special case of NFA)
  • NFA → DFA: subset construction (may exponentially increase states)
  • ε-NFA → NFA: ε-closure computation + redefine transitions
  • ε-NFA → DFA: ε-closure + subset construction (combined)
★ Important Flow: RE → ε-NFA (Thompson's) → NFA → DFA (Subset) → Minimised DFA. Each conversion is well-defined and the resulting automaton accepts the same language.

Section 5 — Finite Automata with Output: Moore & Mealy

Finite Automata with output (also called Finite State Transducers) produce an output for each input symbol or state — unlike DFA/NFA which only accept/reject. Two models: Moore Machine and Mealy Machine.

1. Moore Machine

A Moore Machine is a 6-tuple: M = (Q, Σ, Δ, δ, λ, q₀)

where:
• Q = finite set of states  |  Σ = input alphabet  |  Δ = output alphabet
• δ : Q × Σ → Q = transition function
• λ : Q → Δ = output function (output depends on CURRENT STATE only)
• q₀ = initial state

★ In a Moore machine: output is associated with states. The output for a string = sequence of outputs of states visited (including start state).

★ For input string of length n → output string of length n+1

2. Mealy Machine

A Mealy Machine is a 6-tuple: M = (Q, Σ, Δ, δ, λ, q₀)

Same structure BUT:
• λ : Q × Σ → Δ = output function (output depends on CURRENT STATE and INPUT SYMBOL)
• Output is associated with transitions, not states

★ In a Mealy machine: output is associated with transitions. For input string of length n → output string of length n

Moore Machine
Output on states
Output length = n+1
λ : Q → Δ
Simpler to design
Output written inside state circle
Mealy Machine
Output on transitions
Output length = n
λ : Q × Σ → Δ
More compact (fewer states)
Output written on edge (input/output)

5. Equivalence of Moore and Mealy Machines

Theorem: For every Moore machine, there exists an equivalent Mealy machine and vice versa — both produce the same output for the same input (with possible difference of one leading character in Moore output due to initial state).

Moore → Mealy Conversion

Method: For each transition δ(q, a) = p in Moore machine, create Mealy transition with output = λ_Moore(p) (output of DESTINATION state).
i.e., if Moore has state p with output y, and transition q→p on input a, then in Mealy: edge q→p labelled a/y.
Moore to Mealy — worked example
Moore machine table:
q₀ (out=0): on a → q₁, on b → q₀
q₁ (out=1): on a → q₁, on b → q₂
q₂ (out=0): on a → q₀, on b → q₂
Mealy equivalent: Transfer output of DESTINATION to the edge label:

Section 6 — Minimization of Finite Automata

1. Why Minimization?

A DFA may have redundant (equivalent) states — states that behave identically for all future inputs. Minimization removes these to produce the smallest equivalent DFA (minimum number of states) that accepts the same language.
  • Unique: The minimised DFA is unique (up to state renaming)
  • Efficient: Fewer states = faster simulation, less memory
  • Canonical form: Used to test if two DFAs are equivalent

2. Key Definitions

Distinguishable States

Two states p and q are distinguishable if there exists a string w such that:
δ̂(p, w) ∈ F  but  δ̂(q, w) ∉ F   (or vice versa)
i.e., one accepts w and the other rejects w.

Indistinguishable / Equivalent States

Two states p and q are indistinguishable (equivalent) if for every string w:
δ̂(p, w) ∈ F  iff  δ̂(q, w) ∈ F   (they always agree on acceptance)
Such states can be merged into a single state.

3. Table-Filling (Myhill-Nerode) Algorithm

Step 1 — Remove unreachable states
Find all states reachable from q₀. Remove any unreachable states — they cannot affect the language.
Step 2 — Base: mark (F, non-F) pairs
Create a triangular table for all pairs (p,q). Mark all pairs where one is final (∈F) and other is non-final (∉F) — these are immediately distinguishable.
Step 3 — Iterate: propagate distinguishability
For each UNMARKED pair (p,q): for each symbol a, check if (δ(p,a), δ(q,a)) is already MARKED. If yes → mark (p,q) too. Repeat until no new marks.
Step 4 — Merge unmarked pairs
All still-UNMARKED pairs (p,q) are indistinguishable → merge them into a single state. Build minimised DFA from merged states.

5. Myhill-Nerode Theorem

Define the Myhill-Nerode equivalence relation ~_L on Σ* as:
x ~_L y  iff  for all z ∈ Σ*: xz ∈ L ↔ yz ∈ L

Theorem: A language L is Regular if and only if ~_L has a finite number of equivalence classes.
Furthermore, the number of equivalence classes = number of states in the minimised DFA.

6. Complete Summary

★ Key Points for Minimization:
  • Step 0: Remove all unreachable states first
  • Initially mark ALL pairs where exactly one is final
  • Iteratively mark (p,q) if δ(p,a) and δ(q,a) are already marked, for any symbol a
  • Stop when no new pairs are marked in a full pass
  • All unmarked pairs → indistinguishable → merge them
  • Result is the unique minimum-state DFA
  • Two DFAs are equivalent iff their minimised DFAs are isomorphic
★ Exam Tip: Always start by drawing the triangular pair table. Fill in the base marks first (final vs non-final = marked). Then work through each unmarked pair systematically for each input symbol. The process always terminates.

Q2 (4 marks) — Define a Deterministic Finite Automaton formally. What is an "accepting computation"?
Formal Definition:
A DFA is a 5-tuple M = (Q, Σ, δ, q₀, F) where:
 • Q = finite, non-empty set of states
 • Σ = finite input alphabet
 • δ : Q × Σ → Q = total transition function
 • q₀ ∈ Q = initial/start state
 • F ⊆ Q = set of final (accepting) states
Extended transition δ*:
δ*(q, ε) = q
δ*(q, wa) = δ(δ*(q, w), a)  for w ∈ Σ*, a ∈ Σ
Accepting Computation:
A string w is accepted by M if δ*(q₀, w) ∈ F.
The language of M: L(M) = { w ∈ Σ* | δ*(q₀, w) ∈ F }
∴ DFA = (Q, Σ, δ, q₀, F). A word is accepted when processing it from q₀ ends in a final state.

Q6 (4 marks) — State and prove that every NFA has an equivalent DFA (key argument only).
Statement: For every NFA N = (Q, Σ, δ, q₀, F) there exists a DFA D = (2^Q, Σ, δ', {q₀}, F') with L(D) = L(N).

F' = { S ⊆ Q | S ∩ F ≠ ∅ }

→ w accepted by D iff δ'*({q₀},w) ∩ F ≠ ∅ iff w accepted by N.
∴ By subset construction, every NFA is equivalent to a DFA. Both recognise the same class — Regular Languages.

Q12 (4 marks) — What is a "dead state"? When is a DFA already minimal?
Dead / Trap State:
A state from which no final state is reachable on any input. It acts as a "reject sink". Once entered, the machine stays there forever.
Example: δ(d,0)=d, δ(d,1)=d, d ∉ F.
Condition for minimality:
A DFA M is minimal if:
 (1) It is reachable — every state is reachable from q₀.
 (2) It is distinguishable — no two distinct states are equivalent (i.e., no pair remains unmarked in table-filling).
Uniqueness:
The minimal DFA for a regular language is unique up to state renaming (isomorphism).
∴ A DFA is minimal iff all states are reachable and pairwise distinguishable. Merging equivalent states yields the unique minimal DFA.
