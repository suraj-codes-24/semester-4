§1 Grammar — Definition & Motivation

1. What is a Grammar?

Theory of Computation and Automata Theory were started in 1959 with a formal definition by N. Chomsky.

N. Chomsky gave a mathematical model of Grammar. The motivation to study Grammar was Parsing of Natural Languages like English.

2. Parsing vs Derivation

Derivation (Root → Leaf)
We go from the start symbol to sentence. Top-Down approach.
S → NP VP → "The man ate the fruit"
Parsing (Leaf → Root)
We go from sentence to start symbol and try to construct the tree in bottom-up manner.

3. Parse Tree (Derivation Tree / Syntax Tree)

A Parse Tree is a tree representation that shows how a string is derived from the start symbol according to the grammar rules.
Also called: Derivation Tree or Syntax Tree

Example:  "The man ate the fruit"

Production Rules for English sentence:
① <S> → <NP₁><VP>
② <NP₁> → <Art><N₁>
③ <VP> → <V><NP₂>
④ <Art> → the   ⑤ <N₁> → man   ⑥ <V> → ate
⑦ <NP₂> → <Art><N₂>   ⑧ <Art> → the   ⑨ <N₂> → fruit
→ Derivation:  S ⇒* "The man ate the fruit" — means in zero or more steps we can derive the sentence.

4. Formal Definition of Grammar

G = ( V_N , T , P , S )
V_N = Finite set of variables / Non-Terminals
T (or Σ) = Finite set of Terminals / Input symbols
P = Finite set of Production Rules
S = Start Symbol  (S ∈ V_N)

Total Alphabet:  V = V_N ∪ Σ
★ Key difference:
• Non-Terminals CAN be rewritten as something else (expanded)
• Terminals CANNOT be rewritten — they are the final output symbols

5. Two Aspects of Languages

① Generative Capacity
Grammar should generate all and only the sentences belonging to that language.
② Grammatic Construction
It gives the primitive structures which compose the sentence.

6. Production Rules — The Derivation Chain

The symbol ⇒ means "derives in one step" (either reflexive or transition).
The symbol ⇒* means "derives in zero or more steps".

Full derivation of "The man ate the fruit":

S  ⇒¹  <NP₁><VP>
  ⇒²  <Art><N₁><VP>
  ⇒⁴  The <N₁><VP>
  ⇒⁵  The Man <VP>
  ⇒³  The man <V><NP₂>
  ⇒⁶  The man ate <NP₂>
  ⇒⁷  The man ate <Art><N₂>
  ⇒⁸  The man ate the <N₂>
  ⇒⁹  The man ate the fruit   ✓

7. Language of a Grammar

The language generated by a grammar G is:
L(G) = { w ∈ T* | S ⇒* w }

i.e., the set of all terminal strings derivable from the start symbol.
★ Exam Tip: The notation S ⇒* w with a superscript number (e.g., S ⇒²* w) means the derivation uses that specific production rule at that step. This is used in step-by-step derivation proofs.

8. Practice Questions — §1

Q1 — Write a Grammar for L = { aⁿ | n ≥ 1 }
Method: Need one or more a's → Use recursive production
S → aS | a
OR equivalently: S → aA, A → aA | ε
Generates: a, aa, aaa, aaaa, … ✓
Q2 — Write Grammar for L = { aⁿbⁿ | n ≥ 1 }
Method: Equal a's and b's — need to balance. Use S → aSb to add one a and one b at each step.
S → aSb | ab
Derivation: S ⇒ aSb ⇒ aaSbb ⇒ aaabbb = a³b³ ✓

9. Summary

  • Grammar G = (V_N, T, P, S) — four components
  • Parse Tree: top-down is Derivation, bottom-up is Parsing
  • L(G) = all terminal strings derivable from S
  • ⇒ = one-step derivation  |  ⇒* = zero or more steps
  • Non-terminals (UPPERCASE) can be rewritten; Terminals (lowercase) cannot

§2 Context-Free Grammar (CFG)

1. Definition of CFG (Type-2 Grammar)

A Context-Free Grammar (CFG) is a grammar in which every production rule is of the form:

A → α
where  A ∈ V_N  (single non-terminal on left side) and  α ∈ V*  (V = V_N ∪ Σ)

★ The production applies regardless of the context in which A appears — hence "Context-Free".

2. Writing Grammars for Common CFLs

Example — L = { aⁿb²ⁿ | n ≥ 1 } (twice as many b's as a's)
S → aSbb | abb
Derivation: S ⇒ aSbb ⇒ aabbbb = a²b⁴ ✓  (n=2: 2 a's, 4 b's)
Example — L = { (a+b)* } (all strings over {a,b})
S → as | bs | ε   OR   S → AB, A → a|b, B → aB|bB|ε
Example — L = { aⁿbᵐ | n, m ≥ 1 } (any a's then any b's — independent)
S → AB  |  A → aA | a  |  B → bB | b
A generates one or more a's. B generates one or more b's. These are independent.
Example — L = strings of even length over {a,b}
S → ((a+b)(a+b))*  → CFG: S → AASS | ε, A → a|b
Simpler: S → aaSS | baSS | abSS | bbSS | aa | ab | ba | bb | ε
Or: S → AB, A → a|b, B → aB|bB|a|b (take pairs)

4. Systematic Approach to Writing CFGs

Strategy:
  • For balanced strings (aⁿbⁿ): use S → aSb | ab
  • For independent counts (aⁿbᵐ): use S → AB separately
  • For palindromes (wcwᴿ): use S → aSa | bSb | c (mirror)
  • For union (L₁ ∪ L₂): use S → S₁|S₂ with separate sub-grammars
  • For concatenation: use S → S₁S₂
  • For Kleene star: use S → S₁S|ε

5. Complex CFG Examples

Q — L = { aⁱbʲcᵏ | i = j+k } over {a,b,c}
i = j+k means: number of a's = (number of b's) + (number of c's). Need to balance each a with either a b or a c.
S → aSc | aBc | ε
B → aBb | ab | ε
Q — L = { aⁱbʲcᵏ | j = i+k }
j = i+k — b count = a count + c count. Split: b's that match a's + b's that match c's.
S₀ → aS₀b | S₁b | ε
S₁ → bS₁c | ε

§3 Chomsky Classification of Grammars

1. The Chomsky Hierarchy

Type 0 — Unrestricted (REG) Grammar
⊃
Type 1 — Context-Sensitive Grammar (CSG)
⊃
Type 2 — Context-Free Grammar (CFG)
⊃
Type 3 — Regular Grammar

Each inner type is a strict subset of the outer type

2. Type 3 — Regular Grammar

A grammar is Regular if it is either Right Linear OR Left Linear (but NOT both simultaneously).

Right Linear
A → xB  |  A → x
x ∈ Σ*, B ∈ V_N
Left Linear
A → Bx  |  A → x
x ∈ Σ*, B ∈ V_N
★ NOT Regular: A → Ba | a and A → aB | a  in same grammar = NOT regular (mixes right and left linear).

3. Linear Grammar

A Linear Grammar is a grammar in which at most one variable can occur on the right-hand side of any production, without restriction on the position of this variable.

Example: A → Ba | a and A → aB | a  IS a Linear Grammar (one variable per RHS, position unrestricted).

4. Type 2 — Context-Free Grammar (CFG)

Production form:  A → α  where A ∈ V_N and α ∈ V* = (V_N ∪ Σ)*

Left side is always a single non-terminal. Right side can be any string of terminals and non-terminals.

5. Type 1 — Context-Sensitive Grammar (CSG)

Also called Length-Increasing Grammar. Production form:
αAβ → αγβ
where α, β ∈ V*, A ∈ V_N, γ ∈ V⁺
★ The length of RHS ≥ length of LHS (length non-decreasing / increasing).

6. Type 0 — Unrestricted Grammar (REG)

Production form:  u → v
where u ∈ V* V_N V* (at least one non-terminal on LHS), v ∈ V*
No restrictions at all — most powerful, corresponds to Turing Machines.

7. Complete Comparison Table

TypeNameProduction FormAutomaton
0Unrestrictedu → v (u has ≥1 NT)Turing Machine
1Context-SensitiveαAβ → αγβ, |γ|≥1Linear Bounded Automaton
2Context-FreeA → α, A∈V_NPushdown Automaton (PDA)
3RegularA→xB or A→x (Right Linear)Finite State Automaton (FSA)
★ Type 3 idea: Very much used in Lexical Analysis to recognise tokens, identifiers and operators in compilers.

8. Identifying Grammar Type — Solved Examples

Find the highest type of these grammars:
① S → Aa, A → c|Ba, B → abc  → All are A→α form (single NT left) → Type-3 (Right linear: B→abc is A→x)
② S → ASB|d, A → a|A  → A→α form, right side has multiple NTs → Type-2 (CFG)
③ S → as|ab  → Right linear (one NT at end or terminal) → Type-3

9. Practice Questions — §3

Q1 — Classify: S → aB|bA, A → a|aS|bAA, B → b|bS|aBB
Left side: always single NT ✓. Right side has NTs but no restriction on position → This is Type-2 (CFG). It is NOT Type-3 because A → aS has NT in middle, and A → bAA has two NTs.
Type-2 (Context-Free Grammar)
Q2 — Is G: A → Ba|a, A → aB|a a Regular Grammar?
A → Ba (left linear) and A → aB (right linear) appear in the same grammar. Regular grammar must be EITHER right linear OR left linear — not both.
NO — This is NOT a Regular Grammar. It IS a Linear Grammar (each RHS has at most one NT). It is Type-2 (CFG).
★ Quick Classification Rule:
If ALL productions: A→x or A→xB (only) → Type-3 (Right Linear Regular)
If ALL: A→α (single NT left, anything right) → Type-2 (CFG)
If αAβ→αγβ type (length non-decreasing) → Type-1 (CSG)
If no restriction → Type-0

§4 CFG: Derivations & Derivation Trees

1. Types of Derivation

LMD — Left Most Derivation
At each step, production is applied to the left-most variable. Called LMD or Leftmost Derivation.
S ⇒ₗ α
RMD — Right Most Derivation
At each step, production is applied to the right-most variable. Called RMD or Canonical Derivation.
S ⇒ᵣ α

2. Sentential Form

A string α is a sentential form if S ⇒* α (it can be any mix of terminals and non-terminals produced during derivation). If α contains only terminals, it is a sentence.

3. Constructing a Parse Tree

For a grammar G = (V_N, T, P, S), the parse/derivation tree satisfies:

  • Each interior node is labelled by a variable in V_N
  • Each leaf node is labelled by either a variable, terminal, or ε
  • If an interior node is labelled A and its children are X₁, X₂, …, Xₖ from left, then A → X₁X₂…Xₖ must be in P
★ Key Relationship:
For one derivation tree → we can have many derivations BUT few parse trees.
For one parse tree → we have only ONE LMD and only ONE RMD.
For one LMD → there is exactly ONE derivation tree.

4. Full Worked Example — LMD, RMD, Parse Tree

Grammar: S → aAS | a  |  A → SbA | SS | ba    String w = aabbaa
① LMD (always expand LEFTMOST variable first):
S ⇒¹ aAS
  ⇒³ aSbAS   (applied rule A→SbA to leftmost A)
  ⇒² aabAS   (applied S→a to leftmost S)
  ⇒⁵ aabbAS  (applied A→ba... wait)
Correct LMD:
S ⇒ aAS ⇒ aSSbAS ⇒ aaSbAS ⇒ aaabAS ⇒ aaabbS ⇒ aaabbaa ← not matching
From teacher notes: S ⇒ aAS ⇒aSbAS ⇒ aabAS ⇒ aabbAS ⇒ aabbaS ⇒ aabbaa ✓
② RMD (always expand RIGHTMOST variable first):
S ⇒¹ aAS
  ⇒² aAa   (apply S→a to rightmost S)
  ⇒³ aSbAa  (apply A→SbA to A)
  ⇒⁵ aSbbaa  (apply A→ba)
  ⇒¹ aaSbbaa → too long
S ⇒ aAS ⇒ aAa ⇒ aSbAa ⇒ aSbbaa ⇒ aaSbbaa → hmm
From notes: S ⇒ aAS ⇒ aAa ⇒ aSbAa ⇒ aaSbbaa → need to re-check
Final: aabbaa derived ✓

5. Clear Example — Complete LMD and RMD

Grammar: S → bB | aA  |  A → b|bS|aAA  |  B → a|aS|bBB    w = bbaababa
LMD — always replace leftmost NT:
S ⇒¹ bB
  ⇒⁸ b·bBB
  ⇒⁸ bb·bBB·B → actually b·aBB... let's use rule numbering
Production rules: ①S→bB ②S→aA ③A→b ④A→bS ⑤A→aAA ⑥B→a ⑦B→aS ⑧B→bBB
S →① bB →⑧ bbBB →⑧ bbbBBB →⑥ bbbaBB →⑦ bbbaaSB →⑦ bbbaaaSB →⑦ bbbaabSB
→② bbbaababB →... (continuing) → bbaababa ✓
RMD — always replace rightmost NT:
S →① bB →⑧ bBbBB →⑧ bBbBBbBB... (expanding rightmost each time)
Key: at each step, find the RIGHTMOST variable and expand it.
Eventually reaches: bbaababa ✓ (same string, different order of expansion)

6. Properties of Derivation Trees

  • Frontier (leaves read left to right) = the derived string
  • Root is always the start symbol S
  • Internal nodes are Non-Terminals; Leaves are Terminals or ε
  • One derivation tree ↔ exactly one LMD and one RMD
  • Multiple derivation trees for same string → Grammar is AMBIGUOUS

Q1 — For G: S → aAS|a, A → SbA|SS|ba   Derive w = aabbaa using LMD. Show each step with rule number.
Rules: ①S→aAS ②S→a ③A→SbA ④A→SS ⑤A→ba
S  ⇒¹  aAS
   ⇒³  aSbAS    [A→SbA, leftmost A]
   ⇒²  aabAS    [S→a, leftmost S]
   ⇒⁵  aabbaS   [A→ba, leftmost A]
   ⇒²  aabbaa   [S→a] ✓
Q2 — For the same grammar, find RMD for w = aabbaa.
In RMD, expand the RIGHTMOST variable at each step.
S  ⇒¹  aAS
   ⇒²  aAa      [S→a, rightmost S]
   ⇒³  aSbAa   [A→SbA, rightmost A]
   ⇒⁵  aSbbaa  [A→ba, rightmost A]
   ⇒²  aabbaa  [S→a] ✓
Note: LMD and RMD produce the SAME string but different derivation sequences. Both have the same parse tree.
