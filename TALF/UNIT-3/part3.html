§9 Simplification of CFG

1. Why Simplify?

A CFG may contain unnecessary productions that complicate analysis but don't affect the language. Simplification removes these, making the grammar cleaner for Normal Form conversion.

Three types of simplifications:
① Eliminating ε-productions    ② Eliminating Unit Productions    ③ Eliminating Useless Productions

2. Eliminating ε-Productions

A variable A is nullable if A ⇒* ε (can derive empty string).

An ε-production is a rule of the form A → ε.
Algorithm to remove ε-productions:
  • Step 1: Find all nullable variables (those that can derive ε)
  • Step 2: For each production, add new versions with each nullable variable either present or absent
  • Step 3: Remove all ε-productions (except S→ε if ε ∈ L(G))
Q — Remove ε-productions: S→ABC; A→BC|a; B→bAC|ε; C→CAB|ε
Step 1 — Find nullable variables: B→ε ✓, C→ε ✓, A→BC (B,C both nullable) → A nullable ✓, S→ABC (all nullable) → S nullable ✓
Step 2 — Generate new productions for S→ABC (A,B,C all nullable):
S → ABC | BC | AC | AB | A | B | C | ε
A → BC | a | C | B
B → bAC | bA | bC | b
C → CAB | CA | CB | AB | C | A | B
(Remove all X→ε entries from final grammar)

3. Eliminating Unit Productions

A unit production is a production of the form A → B where both A and B are non-terminals. These can complicate proofs but are sometimes useful in creating unambiguous grammars.
Algorithm to remove unit productions:
  • Step 1: Find all unit pairs (A, B) — where A ⇒* B through unit productions
  • Step 2: For each unit pair (A, B): add A → α for every B → α that is a NON-unit production
  • Step 3: Remove all unit productions A → B from the grammar
Q — Remove unit productions: S → Aa|B; B → A|bb; A → a|bc|B
Unit pairs: (S,B), (B,A), (A,B), (S,A) [transitively via S→B→A]
For each pair, add non-unit productions:
(S,B): add S→bb (from B→bb)
(B,A): add B→a|bc (from A→a|bc)
(A,B): add A→bb (from B→bb)
(S,A): add S→a|bc (from A→a|bc)

Final grammar (no unit productions):
S → Aa | bb | a | bc
B → bb | a | bc
A → a | bc | bb

TAFL — Unit 3 · §9USELESS PRODS

4. Eliminating Useless Productions

A symbol X is useful if:
① X is generating: X ⇒* w (derives some terminal string) AND
② X is reachable: S ⇒* αXβ (reachable from start symbol S)

A symbol that is NOT both generating AND reachable is useless.
Algorithm (4 steps):
  • Step 1: S₁ = set of all terminal symbols (they are generating by definition)
  • Step 2: S₂ = S₁ ∪ {NTs that directly derive terminals only} → "directly terminates"
  • Step 3: Add NTs whose ALL RHS symbols are in S₂ → repeat until stable
  • Step 4: Check reachability from S — remove any NT not reachable from S. Result = useful symbols only.
Q — Remove useless: S → AB|a; A → BC|b; B → aB|aC; C → abCA|abb; D → bD|ac
Step 1: S₁ = {a,b} (terminal symbols)
Step 2: Who directly derives terminals? A→b ✓, S→a ✓, D→ac ✓, C→abb ✓
Step 3: B→aC (A and C now in S₂? C yes, need A in S₂ first... A→b ✓ so A∈S₂). B→aB? B needs B... → Actually B derives: B→aB|aC. Since C∈S₂ and a is terminal: B→aC → B∈S₂ ✓
Generating: S, A, B, C (all derive terminals). D generates ac ✓ too.
Reachable from S: S→AB: A,B reachable. S→a. A→BC: B,C reachable. D not reachable from S!
Remove D (not reachable). Final grammar: S→AB|a; A→BC|b; B→aB|aC; C→abCA|abb

5. Practice Questions — §9

Q1 — Remove ε-productions: S → aSB|a|Ab; A → ε; B → bBB|ε
Nullable variables: A→ε ✓, B→ε ✓. Is S nullable? S→aSB (S needs itself, not directly). S is not nullable.
New productions (each nullable can be present or absent):
S → aSB | aS | aB | a | Ab | b   [A,B both nullable → include versions without each]
A → (removed — only had A→ε)
B → bBB | bB | bB | b   [B nullable: bBB→bBB|bB]
Simplified:
S → aSB | aS | aB | a | Ab | b
B → bBB | bB | b
Q2 — Remove unit productions and useless symbols: S → AB; A → a; B → C; C → D; D → b
Unit pairs: (B,C), (C,D). Chain: B→C→D→b. Also B⇒*b transitively.
Unit productions: B→C, C→D — remove these.
Add: B→b (from D→b via chain), C→b (from D→b)
Now: S→AB; A→a; B→b; C→b; D→b
Check useless: C and D reachable from S? S→AB, no path to C or D → useless!
Final grammar: S → AB; A → a; B → b
Language: {ab}

§10 Chomsky Normal Form (CNF)

1. Definition of CNF

A CFG is in Chomsky Normal Form (CNF) if every production is of the form:

NT → T   (single terminal)
NT → NT  NT   (exactly two non-terminals)
OR:  S → ε  if ε ∈ L(G)

where NT = Non-Terminal, T = Terminal.

2. Steps to Convert CFG to CNF

Step 1
Eliminate all ε-productions (find nullable variables, generate new rules)
Step 2
Eliminate all unit productions (A→B type)
Step 3
Eliminate all useless symbols and productions
Step 4
For each production A → a₁a₂…aₙ (n≥2): replace each terminal aᵢ with a new NT Rᵢ, add rule Rᵢ → aᵢ
Step 5
For each remaining production with 3+ NTs, break into binary: A→X₁X₂…Xₙ becomes A→X₁Y₁, Y₁→X₂Y₂, …, Yₙ₋₂→Xₙ₋₁Xₙ

3. CNF Conversion — Worked Example 1

Convert to CNF: S → aAD; A → aB; A → bAB; B → b; D → d
Step 1,2,3: No ε, unit, or useless productions here.
Step 4: Replace terminals in long productions with new NTs
S → aAD: 'a' is terminal in 3-symbol RHS → introduce R₁→a
A → aB: 'a' is terminal in 2-symbol RHS → introduce R₁→a
A → bAB: 'b' is terminal → introduce R₂→b

After step 4:
S → R₁AD   (still 3 NTs — need step 5)
A → R₁B    (2 NTs ✓)
A → R₂AB   (3 NTs — need step 5)
R₁ → a   R₂ → b   B → b   D → d
Step 5: Break 3-NT productions into binary
S → R₁AD → S → R₁X₁; X₁ → AD
A → R₂AB → A → R₂Y₁; Y₁ → AB

Final CNF:
S → R₁X₁  |  X₁ → AD  |  A → R₁B | R₂Y₁  |  Y₁ → AB
R₁ → a  |  R₂ → b  |  B → b  |  D → d

4. CNF Conversion — Worked Example 2

Convert to CNF: S → aAbB; A → aA|a; B → bB|b (no simplification needed)
S → aAbB: has terminals a,b in 4-symbol production. Replace terminals:
Introduce: Rₐ→a, R_b→b
S → Rₐ A R_b B (4 NTs — break into binary)

S → Rₐ X₁; X₁ → A X₂; X₂ → R_b B

A → aA: replace a → Rₐ A (already 2 NTs ✓ after: A → Rₐ A)
A → a: just one terminal ✓ keep: A → a... wait, that's NT→T ✓ (one terminal = CNF ✓)
B → bB: B → R_b B ✓
B → b: terminal only ✓

Final CNF:
S → Rₐ X₁      X₁ → A X₂      X₂ → R_b B
A → Rₐ A | a   B → R_b B | b
Rₐ → a         R_b → b

5. Practice Questions — §10

Q1 — Convert to CNF: S → aAb; A → Aa|ε
First: A→ε means A is nullable. Apply ε-removal, then convert.
Step 1 (ε-removal): A nullable. S→aAb → add S→ab (with A absent)
S → aAb | ab; A → Aa | a (removed A→ε)

Step 4 (replace terminals): Rₐ→a, R_b→b
S → Rₐ A R_b | Rₐ R_b; A → A Rₐ | a

Step 5 (break 3-NTs):
S → Rₐ X₁; X₁ → A R_b  |  S → Rₐ R_b ✓
A → A Rₐ ✓  | A → a ✓
CNF: S→Rₐ X₁|Rₐ R_b; X₁→AR_b; A→ARₐ|a; Rₐ→a; R_b→b
Q2 — Is this in CNF? S → AB; A → a; B → bB|b. If not, convert.
Check: S→AB ✓ (NT NT). A→a ✓ (single T). B→bB — 'b' is terminal in 2-symbol RHS → NOT CNF. B→b ✓.
NOT in CNF. Fix: Introduce R_b → b
B → R_b B | b ← now CNF ✓
CNF: S→AB; A→a; B→R_b B|b; R_b→b

6. CNF — Advanced Example (Full Pipeline)

Convert to CNF: S → aAbB; A → aA|a; B → bB|b (complete steps)
This grammar has no ε or unit productions, no useless symbols. Go directly to step 4.
S → aAbB: 4 symbols (mixed terminals+NTs)
Introduce: R₁→a, R₂→b
Step 4: S → R₁AR₂B (4 NTs)
A → aA → R₁A (2 NTs ✓); A → a ✓
B → bB → R₂B (2 NTs ✓); B → b ✓

Step 5: S → R₁AR₂B → S → R₁X; X → AR₂B → X → AY; Y → R₂B

Final CNF:
S → R₁X   X → AY   Y → R₂B
A → R₁A | a   B → R₂B | b
R₁ → a   R₂ → b
★ CNF Key Points: Every CFG can be converted to CNF. CNF is used in CYK parsing algorithm. In CNF: parse trees are always binary trees. Max string derivable in exactly n steps has length n (for NT→T rules).

§11 Greibach Normal Form (GNF)

1. Definition of GNF

A CFG is in Greibach Normal Form (GNF) if every production is of the form:

NT → a K  where a∈T, K∈(V_N)*
NT → T  (single terminal)
OR: S → ε if ε ∈ L(G)

★ Every production starts with exactly one terminal followed by zero or more non-terminals.

2. Removing Left Recursion (needed for GNF)

Left Recursion: A production A → Aα | β is left recursive (A appears at the start of RHS).

To remove: Replace with right recursion:
A → Aα | β   ⟺   A → βZ; Z → αZ | α
Remove left recursion: E → E+T | T
E → TZ    (β = T, move to front)
Z → +TZ | +T    (α = +T, create Z for right recursion)
E → Tz | T is equivalent: E → TZ; Z → +TZ | ε. Without ε: Z → +TZ | +T

3. Steps to Convert to GNF

Step 1
Rename variables as A₁, A₂, …, Aₙ in order
Step 2
For each Aᵢ, ensure that all productions Aᵢ → Aⱼα have j > i (no left recursion). If Aᵢ → Aⱼα and j ≤ i, substitute Aⱼ's productions into it.
Step 3
Remove direct left recursion (using the A→βZ trick)
Step 4
Work backwards: substitute to make every production start with a terminal

4. GNF Worked Example 1

Convert to GNF: S → aB|ab; A → aAB|a; B → ABb|b
Step 1: Rename: S→A₁, A→A₂, B→A₃
A₁ → aA₃ | ab
A₂ → aA₂A₃ | a
A₃ → A₂A₃b | b

A₃ starts with A₂ (j=2 < i=3) → substitute A₂:
A₃ → (aA₂A₃)A₃b | aA₃b | b
A₃ → aA₂A₃A₃b | aA₃b | b ✓ (all start with terminal now)

GNF (rename back):
S → aB | ab
A → aAB | a
B → aABBb | aBb | b ✓ all start with terminal

5. GNF Worked Example 2 — Left Recursion Removal

Convert to GNF: S → SoS|1o1
S → SoS is left recursive. Remove using Z trick:
S has: α = oS, β = 1o1
S → 1o1Z | 1o1   (β starts every production)
Z → oSZ | oS    (right recursive Z)

But 'o' and '1' are terminals. Check GNF form:
S → 1 Rₒ 1 Z | 1 Rₒ 1   (need to handle multi-char terminal sequences)
In simplified notation:
S → 01Z | 01
Z → 0SZ | 0S   ← GNF ✓ (starts with terminal 0)

6. Full GNF Conversion — Complete Pipeline

Convert to GNF: S → AA | a; A → SS | b
Step 1: S→A₁, A→A₂
Step 2: A₁→A₂A₂|a (A₂ has index 2>1, so OK). A₂→A₁A₁|b — A₁ has index 1 < 2! Substitute A₁:
A₂ → A₁A₁ | b
Substitute A₁ → A₂A₂|a into A₂→A₁A₁:
A₂ → (A₂A₂)A₁ | aA₁ | b
A₂ → A₂A₂A₁ | aA₁ | b

Step 3: Remove left recursion from A₂:
A₂ → aA₁Z | bZ | aA₁ | b   (using β = aA₁|b)
Z → A₂A₁Z | A₂A₁

Step 4: Back-substitute into A₁:
A₁ → A₂A₂ | a → substitute A₂:
A₁ → (aA₁Z)A₂ | (bZ)A₂ | (aA₁)A₂ | bA₂ | a

GNF:
A₁ → aA₁ZA₂ | bZA₂ | aA₁A₂ | bA₂ | a
A₂ → aA₁Z | bZ | aA₁ | b
Z → A₂A₁Z | A₂A₁ → (substitute A₂ here too for complete GNF)

8. Practice Questions — §11

Q1 — Convert to GNF: S → aB|ab; B → ABb|b; A → aAB|a
Check if already in GNF: S→aB ✓ starts with 'a'; S→ab — need A_b→b... S→aR_b? Actually S→ab means 2 terminals, but GNF allows NT→aK where K can be empty. ab = a followed by b but b must be wrapped as NT. Hmm — simplify:
S → aB ✓ (terminal a then NT B)
S → ab — introduce Rb→b: S → aRb ✓
A → aAB ✓   A → a ✓
B → ABb — A is NT first, not terminal. NOT GNF! Substitute A:
B → (aAB)Bb | aBb | b
B → aABBb | aBb | b ✓ (all start with terminal 'a' or 'b')
Q2 — Remove left recursion: E → E+T|T; T → T*F|F; F → (E)|id
E → E+T | T → remove E-left recursion:
E → TZ  |  Z → +TZ | +T

T → T*F | F → remove T-left recursion:
T → FW  |  W → *FW | *F

F → (E) | id → no left recursion ✓

Result (no left recursion, ready for GNF):
E → TZ | T; Z → +TZ | +T
T → FW | F; W → *FW | *F
F → (E) | id

§12 Quick Revision — All of Unit 3

Grammar G = (V_N, T, P, S)
V_N = Non-Terminals (can rewrite)
T = Terminals (cannot rewrite)
P = Production Rules
S = Start Symbol
V = V_N ∪ T (total alphabet)
Chomsky Types
Type 0: u→v (unrestricted)
Type 1: αAβ→αγβ (CSG)
Type 2: A→α (CFG)
Type 3: A→xB or A→x (Regular)
CFG Derivations
LMD: expand LEFTMOST NT always
RMD: expand RIGHTMOST NT always
Parse tree: unique structure
1 LMD ↔ 1 parse tree (bijection)
Ambiguity
Ambiguous: ≥2 distinct LMDs for some w
Equivalently: ≥2 parse trees for w
Inherently ambiguous: ALL grammars are ambiguous
Fix: add precedence/associativity
Regular Grammar
Right linear: A→xB or A→x
Left linear: A→Bx or A→x
Must be ONLY right OR only left
Linear = at most one NT per RHS
FA ↔ Grammar
FSA→Grammar: state = NT, δ(q,a)=p → q→aP (or q→aP|a if p is final)
Grammar→FSA: NT = state, create final state F for terminal-only rules
CFG Simplification
① ε-removal: find nullable NTs, expand all subsets
② Unit removal: find (A,B) pairs, add non-unit versions
③ Useless removal: check generating + reachable
CNF Rules
NT → T (single terminal)
NT → NT NT (exactly two NTs)
S → ε only if ε∈L(G)
Steps: simplify → replace T → binarise
GNF Rules
NT → a K (terminal then NTs)
NT → T (single terminal)
Every prod starts with terminal!
Remove left recursion first
Left Recursion Removal
A → Aα | β
becomes:
A → βZ | β
Z → αZ | α

Common Exam Mistakes to Avoid

  • Mixing right and left linear in SAME grammar → NOT Regular (it's Linear/CFG)
  • In LMD: always expand the LEFTMOST variable — don't skip any
  • In CNF: A→aB is NOT CNF (terminal in 2-symbol production needs Rₐ→a)
  • In GNF: productions must start with EXACTLY ONE terminal
  • Ambiguity requires two DISTINCT LMDs — not just two different strings
  • ε-removal: S→ε stays if ε∈L(G), otherwise remove it
  • Useless = NOT generating OR NOT reachable (either condition = useless)
