§1 Functional Interfaces

What is a Functional Interface?

A functional interface is an interface that contains exactly one abstract method (SAM — Single Abstract Method). It can have any number of default and static methods. Introduced in Java 8 as the foundation for lambda expressions.

Annotated (optionally) with @FunctionalInterface to enforce the SAM rule at compile time. 
@FunctionalInterface
interface Greeting {
    void greet(String name);        // ONE abstract method = functional

    // default and static methods are allowed
    default void greetFormal(String name) {
        System.out.println("Dear " + name);
    }
    static void info() { System.out.println("Greeting interface"); }
}

// Old way — anonymous class
Greeting g1 = new Greeting() {
    public void greet(String name) { System.out.println("Hello, " + name); }
};

// New way — lambda expression (Java 8+)
Greeting g2 = name -> System.out.println("Hello, " + name);

g2.greet("Alice");   // Hello, Alice

Built-in Functional Interfaces (java.util.function)

InterfaceMethodInput → OutputUse case
Predicate<T>boolean test(T t)T → booleanFilter, check conditions
Function<T,R>R apply(T t)T → RTransform/map a value
Consumer<T>void accept(T t)T → voidPerform action, no return
Supplier<T>T get()() → TProvide a value
BiFunction<T,U,R>R apply(T t, U u)T,U → RTwo-input transformation
UnaryOperator<T>T apply(T t)T → TTransform same type

@FunctionalInterface is optional but strongly recommended. It makes the compiler enforce the one-abstract-method rule, so if you accidentally add a second abstract method, you get a compile error immediately.

Using Built-in Functional Interfaces

import java.util.function.*;

// Predicate — test a condition
Predicate<Integer> isEven = n -> n % 2 == 0;
System.out.println(isEven.test(4));    // true
System.out.println(isEven.test(7));    // false

// Predicate composition
Predicate<Integer> isPositive = n -> n > 0;
Predicate<Integer> isEvenAndPositive = isEven.and(isPositive);  // &&
Predicate<Integer> isEvenOrPositive  = isEven.or(isPositive);   // ||
Predicate<Integer> isOdd             = isEven.negate();          // !

// Function — transform a value
Function<String, Integer> strLen = s -> s.length();
System.out.println(strLen.apply("Hello"));  // 5

// Function chaining with andThen / compose
Function<Integer, Integer> doubleIt = x -> x * 2;
Function<Integer, Integer> addTen   = x -> x + 10;
Function<Integer, Integer> doubleThenAdd = doubleIt.andThen(addTen);
System.out.println(doubleThenAdd.apply(5));  // (5*2)+10 = 20

// Consumer — action, no return
Consumer<String> print = s -> System.out.println(s);
print.accept("Java 8 rocks!");

// Supplier — provide a value
Supplier<Double> random = () -> Math.random();
System.out.println(random.get());  // some random double

// BiFunction
BiFunction<String, Integer, String> repeat = (s, n) -> s.repeat(n);
System.out.println(repeat.apply("Ha", 3));  // HaHaHa
Key insight: Functional interfaces are the target types for lambda expressions and method references. The compiler infers which functional interface is needed from context (variable type, method parameter type, etc.).
🧪 Which functional interface to use?
Need to check if a String is empty → Predicate<String> — returns boolean.
Need to convert int to String → Function<Integer, String> — takes T, returns R.
Need to print each element → Consumer<T> — takes T, returns nothing.
Need to generate a random number → Supplier<Double> — no input, returns T.
✓ Match the shape of your operation (inputs/outputs) to the right interface.

§2 Lambda Expressions

What is a Lambda?

A lambda expression is a concise, anonymous function — it has no name, no class, no access modifier. It represents an instance of a functional interface. Introduced in Java 8.

Syntax: (parameters) -> expression   or   (parameters) -> { statements; }
Lambda = anonymous function that can be passed around like a value. This brings functional programming style to Java — functions as first-class citizens.

Lambda Syntax — All Variations

// 1. No parameters, expression body
Runnable r = () -> System.out.println("Hello!");

// 2. One parameter — parentheses optional
Predicate<String> isEmpty = s -> s.isEmpty();
Predicate<String> isEmpty2 = (s) -> s.isEmpty();   // same thing
Predicate<String> isEmpty3 = (String s) -> s.isEmpty();  // explicit type

// 3. Multiple parameters
Comparator<Integer> cmp = (a, b) -> a - b;

// 4. Block body (multiple statements)
Function<Integer, String> classify = (n) -> {
    if (n > 0) return "positive";
    else if (n < 0) return "negative";
    else return "zero";
};

// 5. No return needed for expression body (implicitly returned)
Function<Integer, Integer> square = x -> x * x;
System.out.println(square.apply(5));  // 25

The Evolution — Before and After Lambda

❌ Before Java 8 Anonymous inner class.

5+ lines for simple logic.
→ Anonymous Class new Comparator() {

✅ Lambda (Java 8) (a, b) -> a - b

Variable Capture in Lambdas

Lambdas can capture (access) variables from the enclosing scope. However, they can only capture variables that are effectively final — variables that are never reassigned after initialisation. 
int factor = 3;             // effectively final (never reassigned)
Function<Integer, Integer> multiply = x -> x * factor;   // ✅ OK
System.out.println(multiply.apply(5));  // 15

factor = 5;                 // ❌ now factor is reassigned
// Function<Integer, Integer> bad = x -> x * factor;  ← compile error!
// "Variable used in lambda expression should be final or effectively final"

// Instance variables and static variables — no restriction
class MyClass {
    int multiplier = 10;
    void test() {
        Function<Integer, Integer> f = x -> x * multiplier;  // ✅ instance var — OK
        multiplier = 20;   // ✅ can change instance var after lambda creation
    }
}

Lambda vs Anonymous Class

Anonymous Class Creates a new class + object
Has its own this (refers to anon class instance)
Can have state (fields)
Works with any interface (not just SAM)
Verbose
Lambda Expression Just a function, no separate class
this refers to enclosing class
No state (variables must be effectively final)
Only for functional interfaces (SAM)
Concise

Practical Lambda Examples

import java.util.*;

List<String> names = Arrays.asList("Charlie", "Alice", "Bob");

// Sort with lambda
names.sort((a, b) -> a.compareTo(b));     // alphabetical
names.sort((a, b) -> b.compareTo(a));     // reverse alphabetical
names.sort((a, b) -> a.length() - b.length()); // by length

// forEach with lambda
names.forEach(name -> System.out.println(name));

// removeIf with lambda
names.removeIf(name -> name.length() < 4);  // removes "Bob"

// replaceAll with lambda
names.replaceAll(name -> name.toUpperCase());  // CHARLIE, ALICE

Returning Lambdas from Methods

// A method that RETURNS a lambda (higher-order function)
static Predicate<Integer> greaterThan(int threshold) {
    return n -> n > threshold;    // captures threshold (effectively final)
}

Predicate<Integer> gt10 = greaterThan(10);
Predicate<Integer> gt50 = greaterThan(50);
System.out.println(gt10.test(15));  // true
System.out.println(gt50.test(30));  // false

// Passing lambdas as method arguments (higher-order function)
static void processNames(List<String> names, Predicate<String> filter,
                            Consumer<String> action) {
    names.stream()
         .filter(filter)
         .forEach(action);
}

processNames(names, s -> s.startsWith("A"), System.out::println);
// prints all names starting with A
Lambda rules summary:
  • Target type must be a functional interface (exactly 1 abstract method).
  • Parameter types can be inferred — no need to write them explicitly.
  • Single-expression body: return is implicit, no braces needed.
  • Block body: must have explicit return (if non-void) and braces.
  • Captured local variables must be effectively final.
  • this inside lambda refers to the enclosing class, NOT the lambda itself.
🧪 Is this lambda valid?
int x = 10; Runnable r = () -> { x = 20; System.out.println(x); };
❌ Compile error — x is modified inside lambda (not effectively final).
Fix: final int x = 10; and remove the assignment inside lambda.
✓ Lambdas can READ effectively-final local vars but cannot MODIFY them.

§3 Method References

What is a Method Reference?

A method reference is a shorthand for a lambda that calls an existing method. Instead of writing x -> obj.method(x), you write obj::method. Uses the :: operator (double colon).

Four Types of Method References

TypeSyntaxLambda equivalentExample
Static methodClassName::staticMethodx -> ClassName.staticMethod(x)Integer::parseInt
Instance method of a particular objectinstance::methodx -> instance.method(x)System.out::println
Instance method of arbitrary object of a typeClassName::instanceMethod(obj, x) -> obj.method(x)String::toUpperCase
Constructor referenceClassName::newx -> new ClassName(x)ArrayList::new
// 1. Static method reference
Function<String, Integer> parse = Integer::parseInt;    // s -> Integer.parseInt(s)
System.out.println(parse.apply("42"));  // 42

// 2. Specific instance method reference
Consumer<String> printer = System.out::println;         // s -> System.out.println(s)
printer.accept("Hello from method ref!");

// 3. Arbitrary instance method reference (first arg becomes the receiver)
Function<String, String> toUpper = String::toUpperCase; // s -> s.toUpperCase()
Predicate<String> startA = s -> s.startsWith("A");     // with arg
BiPredicate<String, String> startsWith = String::startsWith; // (s, prefix) -> s.startsWith(prefix)

// 4. Constructor reference
Supplier<ArrayList<String>> listFactory = ArrayList::new;  // () -> new ArrayList()
ArrayList<String> myList = listFactory.get();
Method references are ONLY valid when the lambda does nothing except call an existing method — no extra logic. If you need to transform arguments or do additional work, use a full lambda instead.

Method References in Action

List<String> names = Arrays.asList("Alice", "Bob", "Charlie", "Anna");

// Using method references instead of lambdas
names.forEach(System.out::println);         // prints each name
names.sort(String::compareToIgnoreCase);    // sorts alphabetically
names.removeIf(String::isEmpty);            // removes empty strings

// Stream with method references
names.stream()
     .filter(s -> s.startsWith("A"))
     .map(String::toUpperCase)              // method ref instead of s->s.toUpperCase()
     .forEach(System.out::println);         // ALICE, ANNA

// Constructor reference for object creation
List<Integer> numbers = Arrays.asList(1, 2, 3);
numbers.stream()
       .map(n -> n * n)
       .collect(Collectors.toCollection(ArrayList::new));  // ArrayList constructor ref

// Custom class method reference
class MathUtils {
    static int doubleIt(int x) { return x * 2; }
}
Function<Integer, Integer> doubler = MathUtils::doubleIt;
System.out.println(doubler.apply(5));  // 10

Lambda vs Method Reference — Side by Side

LambdaMethod Reference (equivalent)
x -> System.out.println(x)System.out::println
s -> s.toUpperCase()String::toUpperCase
s -> Integer.parseInt(s)Integer::parseInt
() -> new ArrayList()ArrayList::new
(a, b) -> a.compareTo(b)String::compareTo
x -> Math.sqrt(x)Math::sqrt
When to prefer method reference over lambda: When the lambda body is just a method call with the same parameters. Method references are more readable and concise — they communicate "call this method" without ceremony. Let the IDE suggest them automatically!

§4 Stream API

What is a Stream?

A Stream (java.util.stream.Stream) represents a sequence of elements supporting sequential and parallel aggregate operations. It is not a data structure — it doesn't store elements; it processes them from a source (collection, array, I/O channel).
Stream pipeline = Source + Intermediate operations + Terminal operation
Think of it as an assembly line: raw materials (source) → processing steps (intermediate) → finished product (terminal).

Creating Streams

import java.util.stream.*;

// From Collection
List<String> list = List.of("A", "B", "C");
Stream<String> s1 = list.stream();
Stream<String> s2 = list.parallelStream();   // parallel processing

// From Array
Stream<String> s3 = Arrays.stream(new String[]{"X", "Y"});

// From Stream.of()
Stream<Integer> s4 = Stream.of(1, 2, 3, 4, 5);

// Infinite streams
Stream<Integer> naturals = Stream.iterate(1, n -> n + 1);     // 1,2,3,4,...
Stream<Double> randoms  = Stream.generate(Math::random);       // random doubles

// Primitive streams (avoid boxing overhead)
IntStream.range(1, 6);       // 1,2,3,4,5 (exclusive end)
IntStream.rangeClosed(1, 5); // 1,2,3,4,5 (inclusive end)
LongStream.of(1L, 2L, 3L);
DoubleStream.of(1.1, 2.2);
Streams are lazy! Intermediate operations are not executed until a terminal operation is called. This enables optimisations — e.g., findFirst() after a filter() stops processing once the first match is found.

Stream Examples — Real Pipelines

List<String> words = List.of("apple", "banana", "avocado", "cherry", "apricot", "blueberry");

// 1. Filter + map + collect
List<String> aWords = words.stream()
    .filter(w -> w.startsWith("a"))          // apple, avocado, apricot
    .map(String::toUpperCase)                   // APPLE, AVOCADO, APRICOT
    .sorted()                                   // alphabetical
    .collect(Collectors.toList());              // [APPLE, APRICOT, AVOCADO]

// 2. Count
long count = words.stream()
    .filter(w -> w.length() > 5)
    .count();                                   // 4 (banana, avocado, cherry, blueberry)

// 3. reduce — sum of lengths
int totalLen = words.stream()
    .mapToInt(String::length)
    .sum();                                     // sum all word lengths

// 4. reduce — product of numbers
int product = IntStream.rangeClosed(1, 5)
    .reduce(1, (a, b) -> a * b);             // 1*2*3*4*5 = 120 (factorial 5)

// 5. findFirst with Optional
Optional<String> first = words.stream()
    .filter(w -> w.startsWith("b"))
    .findFirst();
first.ifPresent(System.out::println);           // banana

// 6. Collect to Map (word → its length)
Map<String, Integer> wordLengths = words.stream()
    .collect(Collectors.toMap(w -> w, String::length));

// 7. Grouping
Map<Character, List<String>> grouped = words.stream()
    .collect(Collectors.groupingBy(w -> w.charAt(0)));
// {a=[apple, avocado, apricot], b=[banana, blueberry], c=[cherry]}

// 8. Joining
String joined = words.stream()
    .collect(Collectors.joining(", ", "[", "]"));
// [apple, banana, avocado, cherry, apricot, blueberry]

Optional — Null-Safe Container

Optional<T> is a container that may or may not contain a non-null value. It replaces returning null from methods — forces callers to handle the "no value" case explicitly. 
// Creating Optional
Optional<String> present = Optional.of("Hello");      // must be non-null
Optional<String> empty   = Optional.empty();
Optional<String> maybe   = Optional.ofNullable(getValue()); // null → empty

// Consuming Optional
present.isPresent();                    // true
present.get();                          // "Hello" (throws if empty!)
present.ifPresent(System.out::println); // prints if present
present.orElse("default");           // value or default
present.orElseGet(() -> compute());    // value or compute lazily
present.orElseThrow(NoSuchElementException::new); // value or throw

// Chaining with Optional
Optional<String> upper = present
    .filter(s -> s.length() > 3)
    .map(String::toUpperCase);         // Optional["HELLO"]

Stream — Key Rules to Remember

  • Streams are single-use. Once a terminal operation is called, the stream is consumed — cannot be reused.
  • Lazy evaluation. Intermediate ops don't run until a terminal op is invoked.
  • Non-interfering. Don't modify the source while the stream is processing.
  • Stateless operations (filter, map) are safe for parallel. Stateful (sorted, distinct) need more care.
  • flatMap = maps each element to a stream, then flattens all streams into one. Used for one-to-many transformations.
🧪 Stream pipeline — what is the output?
List.of(3,1,4,1,5,9,2,6).stream().filter(n -> n%2==0).sorted().limit(3).forEach(System.out::println);
filter even: [4,2,6] → sorted: [2,4,6] → limit 3: [2,4,6]
✓ Output: 2, 4, 6 (each on a new line)
