§7 Java I/O Overview & Stream Concept

What is a Stream?

A stream is an ordered sequence of data flowing from a source to a destination. Java I/O is based on streams — you read from an input stream and write to an output stream. Streams are one-directional.
Source → [Input Stream] → Program → [Output Stream] → Destination

Sources / Destinations: files, network sockets, memory (byte arrays), keyboard, console, pipes.

Two Types of Streams

🔵 Byte Streams Handle raw binary data (8-bit bytes).
Good for: images, audio, video, executables.
Base classes: InputStream / OutputStream
Package: java.io
🟢 Character Streams Handle text data (Unicode 16-bit chars).
Good for: text files, .java, .xml, .csv.
Base classes: Reader / Writer
Package: java.io

Byte Streams vs Character Streams

Byte Streams Work with 8-bit raw bytes
No character encoding/decoding
For binary data (images, audio)
read() returns int (0–255 or −1)
May corrupt text on multi-byte chars
Character Streams Work with 16-bit Unicode chars
Handle encoding automatically (UTF-8, UTF-16…)
For text data (.txt, .csv, .java)
read() returns int (0–65535 or −1)
Safe for international text

Stream Chaining (Decorator Pattern)

Java I/O uses the decorator pattern — you wrap one stream inside another to add functionality. This is called stream chaining. 
// Chain: File → Buffered → Data
FileOutputStream fos = new FileOutputStream("data.bin");
BufferedOutputStream bos = new BufferedOutputStream(fos);   // adds buffering
DataOutputStream dos = new DataOutputStream(bos);           // adds typed writes
dos.writeInt(42);
dos.writeDouble(3.14);
dos.writeUTF("Hello");
dos.close();   // closing outer stream closes all inner streams

// Chain for text: File → InputStreamReader (encoding) → BufferedReader (line reading)
BufferedReader br = new BufferedReader(
    new InputStreamReader(
        new FileInputStream("text.txt"), "UTF-8"));
Always close streams! Use try-with-resources (Java 7+) to guarantee closure:
try (BufferedReader br = new BufferedReader(...)) { ... }
Unclosed streams = resource leaks = file locks and memory waste.
🧪 Why buffer streams?
Without buffering: every read/write = one system call = very slow.
BufferedInputStream reads a chunk (e.g., 8KB) at once into memory buffer.
Subsequent reads come from buffer — no disk I/O until buffer is exhausted.
✓ Buffering can improve I/O performance by 10x–100x. Always buffer file I/O!

§8 Byte Streams

InputStream — Key Methods

java.io.InputStream is the abstract superclass for all byte input streams.
MethodDescription
int read()Reads one byte. Returns byte value (0–255) or −1 at end of stream.
int read(byte[] b)Reads bytes into array. Returns count read, or −1.
int read(byte[] b, int off, int len)Reads up to len bytes into b starting at off.
long skip(long n)Skips n bytes.
int available()Returns estimated available bytes without blocking.
void close()Closes stream and releases resources.
void mark(int limit)Marks current position (if markSupported()).
void reset()Resets to last mark.

OutputStream — Key Methods

MethodDescription
void write(int b)Writes one byte (low 8 bits of int).
void write(byte[] b)Writes all bytes in array.
void write(byte[] b, int off, int len)Writes len bytes from b starting at off.
void flush()Flushes internal buffer to underlying stream / file.
void close()Flushes and closes the stream.
return −1 means EOF: Always check for −1 when reading. The classic loop pattern is: int b; while((b = in.read()) != -1) { process(b); }

FileInputStream & FileOutputStream

// ── Writing bytes to a file ──
try (FileOutputStream fos = new FileOutputStream("output.dat")) {
    byte[] data = {72, 101, 108, 108, 111};   // "Hello" in bytes
    fos.write(data);
    System.out.println("Written successfully");
}

// ── Reading bytes from a file ──
try (FileInputStream fis = new FileInputStream("output.dat")) {
    int b;
    while ((b = fis.read()) != -1) {
        System.out.print((char) b);   // cast byte to char for display
    }
}

// ── Append mode (second param = true) ──
try (FileOutputStream fos = new FileOutputStream("output.dat", true)) {
    fos.write("World".getBytes());
}

BufferedInputStream & BufferedOutputStream

// ── Buffered copy — efficient file copying ──
try (BufferedInputStream bis = new BufferedInputStream(
                                   new FileInputStream("source.jpg"));
     BufferedOutputStream bos = new BufferedOutputStream(
                                   new FileOutputStream("dest.jpg"))) {
    byte[] buffer = new byte[8192];   // 8KB buffer
    int bytesRead;
    while ((bytesRead = bis.read(buffer)) != -1) {
        bos.write(buffer, 0, bytesRead);
    }
    System.out.println("File copied successfully!");
}
// Default buffer size = 8192 bytes. Can customize: new BufferedInputStream(fis, 16384)
Buffer size tip: Default buffer (8192B) works fine for most cases. For large binary files (images, videos), use 64KB (65536) buffers. Always use array read, not byte-by-byte, for performance.

DataInputStream & DataOutputStream

Allow reading/writing Java primitive types (int, double, boolean, etc.) and Strings in a machine-independent format. Data written with DataOutputStream can be reliably read back with DataInputStream. 
// ── Writing primitives ──
try (DataOutputStream dos = new DataOutputStream(
        new BufferedOutputStream(new FileOutputStream("data.bin")))) {
    dos.writeInt(42);
    dos.writeDouble(3.14159);
    dos.writeBoolean(true);
    dos.writeUTF("Hello Java");   // writes length + bytes
}

// ── Reading back in SAME ORDER ──
try (DataInputStream dis = new DataInputStream(
        new BufferedInputStream(new FileInputStream("data.bin")))) {
    System.out.println(dis.readInt());      // 42
    System.out.println(dis.readDouble());   // 3.14159
    System.out.println(dis.readBoolean());  // true
    System.out.println(dis.readUTF());      // Hello Java
}
Critical: Must read data in the EXACT SAME ORDER it was written. Reading int where double was written = corrupted data. DataInputStream/OutputStream are great for binary protocols and custom file formats.
🧪 Identify: Byte or Character stream?
Copying a .jpg image file → Byte stream (binary data)
Reading a .txt file to display on console → Character stream (text data)
Reading from keyboard (System.in) → Byte stream, but wrap in InputStreamReader for chars
✓ Rule: binary data → byte streams; text data → character streams.

§9 Character Streams

Reader — Key Methods

java.io.Reader is the abstract superclass for all character input streams. Works with chars (16-bit Unicode), not bytes.
MethodDescription
int read()Reads one char (0–65535) or −1 at EOF.
int read(char[] cbuf)Reads chars into array. Returns count or −1.
int read(char[] cbuf, int off, int len)Reads up to len chars into cbuf.
long skip(long n)Skips n chars.
boolean ready()True if stream ready to be read without blocking.
void close()Closes stream.

Writer — Key Methods

MethodDescription
void write(int c)Writes one char.
void write(char[] cbuf)Writes entire char array.
void write(String str)Writes a String directly.
void write(String str, int off, int len)Writes substring.
Writer append(CharSequence cs)Appends char sequence, returns this (fluent).
void flush()Flushes buffer.
void close()Flushes and closes.

FileReader & FileWriter

// ── Write text to file ──
try (FileWriter fw = new FileWriter("hello.txt")) {
    fw.write("Hello, World!\n");
    fw.write("Java I/O is fun!");
}

// ── Read text from file char by char ──
try (FileReader fr = new FileReader("hello.txt")) {
    int c;
    while ((c = fr.read()) != -1) {
        System.out.print((char) c);
    }
}

// ── Append mode ──
try (FileWriter fw = new FileWriter("hello.txt", true)) {
    fw.write("\nAppended text!");
}

BufferedReader & BufferedWriter

The most commonly used character stream pair for text file processing. BufferedReader.readLine() is the most convenient way to read text line by line. 
// ── Write lines efficiently ──
try (BufferedWriter bw = new BufferedWriter(new FileWriter("output.txt"))) {
    bw.write("Line 1");
    bw.newLine();          // platform-independent line separator (\n or \r\n)
    bw.write("Line 2");
    bw.newLine();
    bw.write("Line 3");
}

// ── Read line by line — the classic pattern ──
try (BufferedReader br = new BufferedReader(new FileReader("output.txt"))) {
    String line;
    while ((line = br.readLine()) != null) {   // null = EOF (not -1!)
        System.out.println(line);
    }
}
// readLine() returns null at end of file, NOT -1 (important!)

PrintWriter — Formatted Text Output

// PrintWriter wraps any Writer or OutputStream, adds print/println/printf
try (PrintWriter pw = new PrintWriter(new BufferedWriter(new FileWriter("report.txt")))) {
    pw.println("Student Report");
    pw.println("==================");
    pw.printf("%-10s %5s%n", "Name", "Score");   // formatted columns
    pw.printf("%-10s %5d%n", "Alice", 92);
    pw.printf("%-10s %5d%n", "Bob", 87);
    pw.flush();  // PrintWriter buffers — always flush!
}

// Auto-flush mode (slower but safer):
PrintWriter pw = new PrintWriter(new FileWriter("out.txt"), true); // autoFlush=true
PrintWriter checkError(): Unlike other streams, PrintWriter swallows exceptions (no IOException from print methods). Use pw.checkError() to check if an error occurred. This is a common gotcha!

InputStreamReader — Bridge Class

InputStreamReader converts a byte stream (InputStream) into a character stream (Reader), applying a character encoding. Essential for reading text from System.in or network sockets. 
// Read from keyboard (System.in is a byte stream) as text
BufferedReader keyboard = new BufferedReader(new InputStreamReader(System.in));
System.out.print("Enter name: ");
String name = keyboard.readLine();
System.out.println("Hello, " + name);

// Read file with specific encoding (e.g., UTF-8)
try (BufferedReader br = new BufferedReader(
        new InputStreamReader(new FileInputStream("arabic.txt"), "UTF-8"))) {
    String line;
    while ((line = br.readLine()) != null) System.out.println(line);
}

// OutputStreamWriter — reverse bridge
try (PrintWriter pw = new PrintWriter(
        new OutputStreamWriter(new FileOutputStream("out.txt"), "UTF-8"))) {
    pw.println("Unicode text: こんにちは");
}

Scanner — Easy User Input

import java.util.Scanner;

Scanner sc = new Scanner(System.in);     // reads from keyboard
System.out.print("Enter age: ");
int age = sc.nextInt();
System.out.print("Enter name: ");
sc.nextLine();                             // consume leftover newline
String name = sc.nextLine();

Scanner fileSc = new Scanner(new File("data.txt"));   // scan a file
while (fileSc.hasNextLine()) System.out.println(fileSc.nextLine());
fileSc.close();
Scanner trap: After nextInt() or nextDouble(), the newline character '\n' remains in the buffer. The next nextLine() will read the empty line. Fix: call sc.nextLine() once extra to consume the newline.

§10 Reading & Writing Files — Complete Patterns

The File Class

java.io.File represents a file or directory path. It does NOT read/write data — it's used for metadata operations: exists, create, delete, list, rename. 
File f = new File("notes.txt");
System.out.println(f.exists());       // true/false
System.out.println(f.length());       // file size in bytes
System.out.println(f.getName());      // "notes.txt"
System.out.println(f.getAbsolutePath()); // full path
System.out.println(f.isFile());       // true
System.out.println(f.isDirectory());  // false
System.out.println(f.canRead());      // true/false
f.createNewFile();                    // creates empty file (throws IOException)
f.delete();                           // deletes file
f.mkdir();                            // creates directory
String[] files = f.list();            // lists directory contents

Pattern 1: Write text to a file

try (BufferedWriter bw = new BufferedWriter(new FileWriter("student.txt"))) {
    bw.write("Roll: 101"); bw.newLine();
    bw.write("Name: Alice"); bw.newLine();
    bw.write("Marks: 92");
    System.out.println("File written.");
} catch (IOException e) {
    System.err.println("Write error: " + e.getMessage());
}

Pattern 2: Read text from a file line by line

try (BufferedReader br = new BufferedReader(new FileReader("student.txt"))) {
    String line;
    while ((line = br.readLine()) != null) {
        System.out.println(line);
    }
} catch (FileNotFoundException e) {
    System.err.println("File not found!");
} catch (IOException e) {
    System.err.println("Read error: " + e.getMessage());
}

Pattern 3: Copy a binary file (Byte Streams)

public static void copyFile(String src, String dest) throws IOException {
    try (BufferedInputStream in = new BufferedInputStream(new FileInputStream(src));
         BufferedOutputStream out = new BufferedOutputStream(new FileOutputStream(dest))) {
        byte[] buf = new byte[8192];
        int n;
        while ((n = in.read(buf)) != -1) out.write(buf, 0, n);
    }
    System.out.println("Copied: " + src + " → " + dest);
}

Pattern 4: Read all lines into a List (Java 8+ NIO)

import java.nio.file.*;
import java.util.List;

// Read all lines at once — convenient for small files
List<String> lines = Files.readAllLines(Paths.get("student.txt"));
lines.forEach(System.out::println);

// Write all lines at once
List<String> data = List.of("Alice", "Bob", "Charlie");
Files.write(Paths.get("names.txt"), data);

// Read entire file as a String (Java 11+)
String content = Files.readString(Paths.get("student.txt"));

// Write entire String to file (Java 11+)
Files.writeString(Paths.get("out.txt"), "Hello World");

// Stream lines lazily (good for large files)
try (java.util.stream.Stream<String> stream = Files.lines(Paths.get("big.txt"))) {
    stream.filter(l -> l.startsWith("ERROR")).forEach(System.out::println);
}
java.nio.file (New I/O — Java 7+): The Files and Paths classes in java.nio.file provide a much cleaner API than the old java.io.File. Prefer NIO for new code. Old File class is mainly used for compatibility.

Complete File Read/Write Program

import java.io.*;

public class FileDemo {

    public static void writeStudents(String[] students) throws IOException {
        try (PrintWriter pw = new PrintWriter(new BufferedWriter(
                              new FileWriter("students.txt")))) {
            for (String s : students) pw.println(s);
        }
    }

    public static void readStudents() throws IOException {
        try (BufferedReader br = new BufferedReader(
                                 new FileReader("students.txt"))) {
            String line;
            int i = 1;
            while ((line = br.readLine()) != null)
                System.out.println(i++ + ". " + line);
        }
    }

    public static void main(String[] args) {
        try {
            String[] names = {"Alice", "Bob", "Charlie"};
            writeStudents(names);
            System.out.println("--- Reading back ---");
            readStudents();
        } catch (IOException e) {
            System.err.println("I/O Error: " + e.getMessage());
        }
    }
}
🧪 Which to use: FileReader or InputStreamReader?
FileReader uses the default system charset — not portable across machines!
InputStreamReader lets you specify encoding: new InputStreamReader(fis, "UTF-8")
✓ Best practice: always specify encoding explicitly. Use InputStreamReader + UTF-8 for portability.
