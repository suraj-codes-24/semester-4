§1 Why Java? History & Overview

Why Java?

Java is the most widely used programming language for enterprise, Android, and web backend development. It solves a classic problem: write once, run anywhere (WORA).
  • Platform Independent: Java code compiles to bytecode, not native machine code. Any device with a JVM can run it.
  • Object-Oriented: Everything is modelled as objects — mirrors the real world.
  • Robust & Secure: Strong type checking, exception handling, no pointer arithmetic, sandbox execution.
  • Multithreaded: Built-in support for concurrent programming.
  • Rich API & Ecosystem: Huge standard library, frameworks (Spring, Hibernate), tools.
  • Open Source: OpenJDK is freely available.

§2 JVM, JRE & Java Environment

The Big Three — JDK, JRE, JVM

Think of it as nested boxes: JDK ⊇ JRE ⊇ JVM
JVM — Java Virtual Machine
An abstract machine that provides a runtime environment to execute Java bytecode. It is platform-specific (different JVMs for Windows, Linux, Mac) but the bytecode it runs is platform-independent.
JRE — Java Runtime Environment
JVM + Java class libraries + supporting files. Needed to run Java programs. End users need JRE, not JDK.
JDK — Java Development Kit
JRE + compiler (javac) + debugger + javadoc + other tools. Needed to develop Java programs.

JVM Architecture

Three main subsystems inside the JVM:
1
Class Loader Subsystem — Loads .class files into memory. Three loaders: Bootstrap → Extension → Application.
2
Memory Areas — Method Area (class data), Heap (objects), Stack (frames/local vars), PC Register, Native Method Stack.
3
Execution Engine — Interpreter (executes bytecode line by line) + JIT Compiler (compiles hot code to native for speed) + Garbage Collector.
JIT vs Interpreter: Interpreter translates bytecode line-by-line (slow). JIT compiles frequently executed code to native machine code at runtime (fast). This is why Java is much faster than pure interpreted languages.

Garbage Collection

Garbage Collector (GC) — Automatic memory management. JVM automatically reclaims memory occupied by objects that are no longer referenced. Programmers don't need to manually free memory (unlike C/C++).
  • GC runs in the background as a daemon thread.
  • You can suggest (not force) GC with System.gc() — not guaranteed.
  • Common GC algorithms: Serial, Parallel, G1 (Garbage First), ZGC.

Java Environment Setup

Steps to set up Java development environment:
1
Download and install JDK from oracle.com or adoptium.net.
2
Set JAVA_HOME environment variable → path to JDK installation.
3
Add %JAVA_HOME%\bin (Windows) or $JAVA_HOME/bin (Linux/Mac) to PATH.
4
Verify: open terminal → java -version and javac -version.
Common exam question: "What is the difference between JDK, JRE, and JVM?" — Memorise: JDK = develop + run; JRE = run only; JVM = virtual machine inside JRE that executes bytecode.

§3 Java Source File Structure & Compilation

Rules for a Java Source File

A Java source file follows strict structural rules:
  • File extension must be .java
  • File name must exactly match the public class name (case-sensitive).
  • At most one public class per file.
  • Can have multiple non-public classes in the same file.
  • Package statement (if present) must be the very first line.
  • Import statements come after package, before class declaration.

Structure of a Java Source File

// 1. Package declaration (optional, must be first)
package com.example.myapp;

// 2. Import statements (optional)
import java.util.Scanner;
import java.util.*;          // wildcard import
import static java.lang.Math.PI;  // static import

// 3. Class declaration (filename must match public class name)
public class HelloWorld {

    // 4. Main method — entry point of the program
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

The main() Method Signature — Explained

public static void main(String[] args)
  • public — JVM must be able to call it from outside the class.
  • static — JVM calls it without creating an object of the class.
  • void — returns nothing to the JVM.
  • main — fixed name recognised by JVM as entry point.
  • String[] args — command-line arguments passed as an array of strings.
Common mistake: Writing Main or MAIN instead of main — Java is case-sensitive! Also, String args[] is also valid syntax (C-style array declaration).

Variables

A variable is a named memory location that holds a value of a specific data type.
Syntax: dataType variableName = value;

Types of Variables

1
Local Variable: Declared inside a method/block. Must be initialised before use. No default value. Stored in stack. Scope = that block only.
2
Instance Variable (Field): Declared inside a class but outside methods. Each object gets its own copy. Has default value. Stored in heap.
3
Static Variable (Class Variable): Declared with static keyword. Shared by all instances of the class. Stored in Method Area. Exists for lifetime of class.
public class Variables {
    static int count = 0;        // static variable — shared
    String name;                   // instance variable — per object

    void display() {
        int x = 10;              // local variable
        System.out.println(x + name + count);
    }
}

Type Casting

Widening (Implicit): smaller type → larger type. Done automatically. No data loss.
byte → short → int → long → float → double

Narrowing (Explicit): larger type → smaller type. Must be done manually. May lose data.
double d = 9.7; int i = (int) d; // i = 9 (truncated)
🧪 Example: Type casting
int i = 300; byte b = (byte) i; — What is b?
300 in binary is 100101100. byte only keeps last 8 bits = 00101100 = 44.
✓ b = 44 (data loss due to narrowing)

§5 Control Flow & Arrays

Control Flow Statements

Decision Making

// if-else
if (score >= 90) { grade = 'A'; }
else if (score >= 75) { grade = 'B'; }
else { grade = 'C'; }

// switch — works with int, char, String (Java 7+), enum
switch (day) {
    case 1: System.out.println("Monday"); break;
    case 2: System.out.println("Tuesday"); break;
    default: System.out.println("Other");
}
Forgetting break in switch causes fall-through — execution continues into the next case. This is a very common bug and a frequent exam trick question!

Loops

// for loop
for (int i = 0; i < 5; i++) { System.out.println(i); }

// while loop — condition checked first (may not execute)
while (i < 5) { System.out.println(i); i++; }

// do-while — executes at least once, then checks condition
do { System.out.println(i); i++; } while (i < 5);

// enhanced for (for-each) — Java 5+, for arrays and collections
int[] arr = {1, 2, 3};
for (int x : arr) { System.out.println(x); }

Jump Statements

  • break — exits the innermost loop or switch block.
  • continue — skips the rest of the current loop iteration, goes to next iteration.
  • return — exits the current method, optionally returning a value.
  • break label / continue label — jumps to a labelled outer loop.

Arrays

An array is a fixed-size, ordered collection of elements of the same data type. Arrays in Java are objects stored in the heap. 
// 1D Array Declaration + Allocation + Initialization
int[] marks = new int[5];           // creates array of 5 ints, default=0
marks[0] = 85; marks[1] = 90;      // manual assignment

int[] scores = {85, 90, 78, 92, 88}; // array literal

// Length property
System.out.println(scores.length);   // 5 (not a method, it's a field!)

// 2D Array
int[][] matrix = new int[3][4];     // 3 rows, 4 columns
int[][] grid = {{1,2},{3,4},{5,6}}; // 3×2 array literal

// Jagged Array — rows of different lengths
int[][] jagged = new int[3][];
jagged[0] = new int[2];
jagged[1] = new int[4];
Key Array Facts:
  • Array index starts at 0. Last element = arr[arr.length - 1]
  • Accessing out-of-bounds index throws ArrayIndexOutOfBoundsException
  • Arrays are objects — int[] x = new int[5] allocates on heap.
  • Arrays.sort(arr) sorts in ascending order (java.util.Arrays).
  • Arrays.copyOf(arr, len) creates a new array copy.
🧪 Find max element in an array
int max = arr[0];
Loop: for(int x : arr) { if(x > max) max = x; }
✓ O(n) time — must examine every element once.
Array vs ArrayList: Arrays have fixed size. ArrayList (java.util) is dynamic — grows automatically. Exams may ask when to use which.

§6 Strings in Java

String Basics

String is a class in java.lang that represents a sequence of characters. Strings in Java are immutable — once created, their value cannot be changed. 
// Two ways to create a String
String s1 = "Hello";              // String literal — uses String Pool
String s2 = new String("Hello"); // Always creates new object in heap

// String Pool: JVM keeps a pool of unique string literals
String a = "Java";
String b = "Java";
System.out.println(a == b);        // true — same object from pool!
System.out.println(a.equals(b));   // true — same content

String Immutability & StringBuilder

Because Strings are immutable, operations like concatenation create new String objects. In a loop, this is very inefficient. Use StringBuilder (or StringBuffer for thread-safety) instead. 
// Inefficient — creates many intermediate String objects
String result = "";
for(int i = 0; i < 1000; i++) result += i;

// Efficient — StringBuilder modifies in-place
StringBuilder sb = new StringBuilder();
for(int i = 0; i < 1000; i++) sb.append(i);
String result = sb.toString();
String (immutable) Thread-safe (no modification)
Stored in String Pool
+ operator creates new object
Use for constants / keys
StringBuilder (mutable) NOT thread-safe
Stored in heap
.append() modifies in-place
Use in loops / heavy concatenation

String Conversion

// int to String
String s = String.valueOf(42);           // "42"
String s = Integer.toString(42);         // "42"
String s = "" + 42;                      // "42" (concatenation trick)

// String to int
int n = Integer.parseInt("42");          // 42

// String to char array
char[] arr = "hello".toCharArray();

// char array to String
String s = new String(arr);
🧪 Reverse a String (classic exam question)
Use StringBuilder: new StringBuilder(s).reverse().toString()
Or manually: loop from s.length()-1 to 0, append each char.
✓ StringBuilder.reverse() is the clean, efficient one-liner answer.
StringBuffer vs StringBuilder: Both are mutable. StringBuffer is thread-safe (synchronized methods) but slower. StringBuilder is faster but NOT thread-safe. For single-threaded use → StringBuilder.
